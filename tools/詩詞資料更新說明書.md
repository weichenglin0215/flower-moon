# 詩詞資料更新與轉換說明書

本文件說明如何將 `唐詩三百首佳句.xlsx` 的原始資料，轉換為本專案所使用的三種格式：`poems.js`、`詩詞朗讀AI生成語音.txt` 以及 `calendar_assignments.js`。

## 準備工作
1. 確保電腦已安裝 **Python 3.x**。
2. 準備好最新的 [data/唐詩三百首佳句.xlsx](../data/唐詩三百首佳句.xlsx)。

---

## 步驟一：使用 Excel 轉換器產生核心資料

1. **開啟 Excel 檔案**：
   - 開啟 `data/唐詩三百首佳句.xlsx`。
   - 選取您要更新的列數範圍（注意：每一首詩通常包含「資料行」與「評分行」兩行）。
   - 按下 `Ctrl + C` 複製資料。

2. **執行轉換工具**：
   - 在瀏覽器中開啟 [converter.html](converter.html)。
   - 將 Excel 複製的內容貼入上方的文字框中。
   - 按下 **「EXCEL資料轉換成程式碼格式」** 按鈕。

3. **更新 poems.js**：
   - 在「詩詞轉換成json格式」區域點選 **「複製」**。
   - 開啟 [data/poems.js](../data/poems.js)。
   - 將複製的內容替換到檔案中的 `const POEMS = [...]` 陣列內。就是全部置換。

4. **更新 author_biography.js**：
   - 在「作者生平與小傳轉換成json格式」區域點選 **「複製」**。
   - 開啟 [data/author_biography.js](../data/author_biography.js)。
   - 將複製的內容替換到檔案中，更新所有的作者資料。

5. **更新朗誦文字 詩詞朗讀AI生成語音.txt**：
   - 在「轉換成單行式朗誦文本」區域點選 **「複製」**。
   - 開啟 [data/詩詞朗讀AI生成語音.txt](../data/詩詞朗讀AI生成語音.txt)。
   - 將內容貼上並存檔。

---

## 步驟二：更新日曆詩詞分配表

當 `poems.js` 更新後，必須重新計算未來三年的日曆分配，以確保 290 天內不重複。

1. **開啟檔案總管**。
2. **切換至專案目錄FlowerMoon_web\tools**。
3. **執行更新腳本**：
   雙擊"產生三年份的日曆詩詞資料_refresh_assignments.bat"檔案執行。
   或
   在檔案總管的網址列輸入 cmd，按下 Enter。
   python refresh_assignments.py
4. **確認結果**：
   - 看到「成功！已重新產生 .../data/calendar_assignments.js」字樣即代表更新完成。
   - 本專案會自動載入 [data/calendar_assignments.js](../data/calendar_assignments.js) 並顯示於日曆中。

---

## 注意事項
- **Excel 格式**：請確保 Excel 的欄位順序與 `converter.html` 說明中一致，否則轉換可能失敗。
- **品質過濾**：日曆分配表 (`refresh_assignments.py`) 僅會挑選評分為 4 (含) 以上的詩詞。如果新加入的詩詞評分過低，將不會出現在日曆中。
- **重複率**：目前的分配機制確保了至少連續 290 天不會出現重複的詩詞。

## 詳細程式說明
converter.html 的核心功能與演算法
這個網頁的本質是一個 自定義的 TSV (Tab-Separated Values) 數據解析器與數據處理引擎。當使用者從 Excel 複製表格貼上到網頁時，資料會帶有 Tab (縮排) 與 Enter (換行) 字元。程式透過 parseTSV() 函數將這串純文字轉換為可程式化操作的二維陣列（Row 與 Column）。

1. **詩詞轉換成 JSON 格式 (轉為程式碼)**
- **資料配對邏輯**： 程式會一行一行掃描轉換後的陣列。當它讀取到一首詩的「主要資料行」（包含年代、作者、各種資訊與詩句）時，建立一個 poem 物件。為了取得各句詩的評分，它會「往前看一行 (Lookahead)」：檢查下一行是否只有評分數字而沒有「年代、作者、詩名」，如果是的話，就會把下一行的評分對應到上面那行的詩句 (line_ratings)。
- **添加「大陸拼音」與「同音字」(注音優化演算法)**： 針對「注音」這一個欄位，程式會執行 enhanceZhuyin() 函式。
- **正則表示式比對**： 使用 Regex /([\u4e00-\u9fff]+)\s*(\([ㄅ-ㄩˊˇˋ˙\s　]+\))/g 來精準抓出 漢字(注音) 的格式。
- **串接 pinyin-pro API**： 把抓出來的漢字傳入第三方的 pinyin-pro 函式庫，並設定提取帶有聲調符號的拼音格式，藉此自動獲得大陸拼音。
- **查表得出同音字**： 程式內部內建了一個非常大的字典陣列 COMMON_CHARS，將注音與我們最常見的「常用字」互相映射。解析出注音後，會去這個字典裡查找最常用的同音字。
- **重組字串**： 最終將原始字元、括號拼音、同音注釋重新組合，例如將 酌(ㄓㄨㄛˊ) 變成 酌(ㄓㄨㄛˊ)(zhuó)發音同"琢"。
輸出 JSON： 最後使用 JSON.stringify，將整理好的 poems 陣列轉換為 JavaScript 宣告語法 (const POEMS = [...]) 提供複製。

2. **作者生平與小傳轉換成 JSON 格式 (轉為程式碼)**
- **資料配對邏輯**： 程式會一行一行掃描轉換後的陣列。當它讀取到一首詩的「主要資料行」（包含年代、作者、各種資訊與詩句）時，建立一個 poem 物件。為了取得各句詩的評分，它會「往前看一行 (Lookahead)」：檢查下一行是否只有評分數字而沒有「年代、作者、詩名」，如果是的話，就會把下一行的評分對應到上面那行的詩句 (line_ratings)。
- **添加「大陸拼音」與「同音字」(注音優化演算法)**： 針對「注音」這一個欄位，程式會執行 enhanceZhuyin() 函式。
- **正則表示式比對**： 使用 Regex /([\u4e00-\u9fff]+)\s*(\([ㄅ-ㄩˊˇˋ˙\s　]+\))/g 來精準抓出 漢字(注音) 的格式。

- **查表得出同音字**： 程式內部內建了一個非常大的字典陣列 COMMON_CHARS，將注音與我們最常見的「常用字」互相映射。解析出注音後，會去這個字典裡查找最常用的同音字。

3. **轉換成單行式朗誦文本**
- **目的**： 為了方便將詩詞放入 AI 語音合成系統（TTS）生成語音，或是讓真人不用看排版就能流暢朗讀，演算法會把樹狀的 JSON 資料攤平成「一整段沒有換行的自然語言描述」。
- **本文重組**： 程式會執行 generateRecitationLine()。將基本資訊用逗號與文字連接，例如：{詩名}，{類型}，{朝代}，作者：{作者}。{詩句1}，{詩句2}。
- **發音說明智慧提取**： 若有填寫「注音」欄位，程式不僅加上「發音說明：」字樣，還會：
去找該「生難字」出現在哪一句詩裡，將那整句詩抓出來（提供上下文脈絡）。
將注音轉換為系統算出的「同音常用字」，而非直接唸注音符號（因為語音合成器不一定看得懂注音符號）。
例如：原本注音寫 蒹(ㄐㄧㄢ)，程式會自動把它轉成：蒹葭蒼蒼，蒹(間)，讓 AI 或真人可以直接看「間」的音來唸出正確讀音。
