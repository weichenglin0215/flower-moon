## 《花月》網頁遊戲：雲端存檔功能實作手冊

### 第一階段：Supabase 環境建置（使用者手動執行）

1.  **建立組織與專案：**
    * 登入 [Supabase](https://supabase.com/)。
    * 點擊 **Create a new organization**，名稱自訂（如 `HappyFlowerCreative`），方案選 **Free**。
    * 點擊 **New Project**，名稱設為 `FlowerMoon-DB`，設定資料庫密碼。區域選 **Tokyo** 或 **Singapore**。
2.  **執行 SQL 指令（SQL Editor）：**
    * 請在 Supabase 的 **SQL Editor** 貼入以下代碼：

```sql
-- 1. 建立 player_saves 資料表
CREATE TABLE player_saves (
    id TEXT PRIMARY KEY, -- 這裡存放你的「引繼碼」(例如：李白#A9F2)
    version TEXT DEFAULT '1.2',
    nickname TEXT DEFAULT '訪客',
    total_score INTEGER DEFAULT 0,
    global_rank TEXT DEFAULT '書僮',
    play_days INTEGER DEFAULT 1,
    last_played_date DATE DEFAULT CURRENT_DATE,
    games JSONB DEFAULT '{}'::jsonb,
    level_progress JSONB DEFAULT '{}'::jsonb,
    difficulty_counts JSONB DEFAULT '{"小學": 0, "中學": 0, "高中": 0, "大學": 0, "研究所": 0}'::jsonb,
    achievements JSONB DEFAULT '{"unlocked": [], "progress": {}, "claimed": []}'::jsonb,
    settings JSONB DEFAULT '{"bgm": true, "soundEffects": true}'::jsonb,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 2. 開啟 RLS
ALTER TABLE player_saves ENABLE ROW LEVEL SECURITY;

-- 3. 建立 RLS 政策 (允許持有效引繼碼的匿名使用者存取)
-- 由於不強制登入，我們設定只要知道 ID (引繼碼) 就能讀取與更新
CREATE POLICY "Enable access via ID" ON player_saves 
FOR ALL USING (true) WITH CHECK (true);

-- 4. 建立更新時間觸發器
CREATE OR REPLACE FUNCTION update_updated_at_column()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = NOW();
    RETURN NEW;
END;
$$ language 'plpgsql';

CREATE TRIGGER update_player_saves_updated_at
    BEFORE UPDATE ON player_saves
    FOR EACH ROW
    EXECUTE FUNCTION update_updated_at_column();
```

3.  **獲取 API 金鑰：**
    * 記下 `Project URL` 與 `Publishable key`。
    a. 左側選單最下方，點擊 Project Settings（專案設定，⚙️ 齒輪圖示）。
    b. 在左側第二層選單中，點擊 API Keys。
    c. Publishable and secret API keys分頁，Publishable key（前端用）
    d. 如果沒看到，按 + New publishable Key 建立 API Keys（取代舊版 anon public key）
    e. 把需要的金鑰複製下來。
    **切勿複製service_role key**（後端最高權限用，千萬不能放在前端的程式碼之中）

4.  **supabase 設定**
### 4.1 目前正確設定位置與步驟

#### **主要設定位置：Authentication > URL Configuration**

a. 登入 Supabase 後台（https://supabase.com/dashboard）
b. 選擇你的專案
c. 左側選單點擊 **Authentication**（驗證）
d. 在 Authentication 區塊下方，點擊 **URL Configuration**（URL 設定）
e. 填寫以下欄位：

   | 欄位名稱                  | 建議填寫內容                              | 說明 |
   |---------------------------|-------------------------------------------|------|
   | **Site URL**              | `https://你的帳號.github.io`             | 主要網站網址，登入後預設跳轉位置 |
   | **Additional Redirect URLs** | `https://你的帳號.github.io`<br>`https://你的帳號.github.io/**` | 多行加入，`**` 為 wildcard，涵蓋所有子路徑 |

f. 確認無誤後，點擊頁面最下方的 **Save** 按鈕。

---

### 4.2 CORS 相關說明（重要更新）

- **Settings > API > CORS Allowed Origins 欄位已移除**。
- Supabase 目前對 REST API、Auth、Storage 等服務 **預設允許所有來源**（`Access-Control-Allow-Origin: *`）。
- 因此一般情況下，**無需額外設定 CORS** 即可從 GitHub Pages 呼叫 Supabase。
- 若未來 Supabase 恢復此功能，將會在相同位置（Settings > API）出現。

### 4.3 常見問題排查

| 問題情境                     | 可能原因                          | 建議解決方式 |
|-----------------------------|-----------------------------------|-------------|
| 登入後無法跳轉               | 未設定 Site URL 或 Redirect URLs | 依照第 3 節完整設定 |
| CORS Error                  | 極少發生（預設允許）              | 檢查是否使用自訂域名或 Cloudflare 等 CDN |
| 開發環境（localhost）無法登入 | 未加入 localhost                  | 在 Additional Redirect URLs 加入 `http://localhost:3000`、`http://localhost:5173` 等 |
| 設定後仍無效                | 未點擊 Save                       | 務必點擊最下方 Save 按鈕 |

### 4.4 推薦最佳實務

- 同時加入以下三行到 **Additional Redirect URLs**：
  1. `https://你的帳號.github.io`
  2. `https://你的帳號.github.io/**`
  3. `http://localhost:3000`（開發用，可之後移除）

- 設定完成後，建議立即測試：
  - 使用 `supabase.auth.signInWithPassword()` 或 `signUp()` 等功能
  - 確認登入後能正確跳轉回 GitHub Pages

### 4.5 結論與建議

目前 Supabase 已簡化 CORS 設定，主要透過 **Authentication > URL Configuration** 來處理 GitHub Pages 的驗證與跳轉需求。只要正確設定 **Site URL** 與 **Additional Redirect URLs**，即可解決大部分的「Allowed Origins」相關問題。

### 第二階段：程式邏輯需求（提交給 Google Antigravity）

請 Google Antigravity 根據以下邏輯撰寫 JavaScript：

#### 1. 初始化連線
使用 `supabase-js` 透過 `Project URL` 與 `Publishable key` 建立客戶端連線。

#### 2. 引繼碼（Transfer Code）管理邏輯
* **生成功能：** 結合 `nickname` 加上 4 位隨機大寫英數字（範例：`杜甫#C9B1`）。
* **持久化：** 生成後必須存入瀏覽器的 `localStorage.setItem('flower_moon_id', ...)`。

#### 3. 儲存進度 (Save Data)
* **函式：** `saveGameToCloud()`。
* **邏輯：** 1. 從 `localStorage` 取得引繼碼。若無，則引導玩家輸入暱稱並生成新代碼。
    2. 將遊戲中當前的數據（得分、等級、JSON 進度等）封裝。
    3. 使用 `supabase.from('player_saves').upsert(data)`。其中 `id` 必須對應引繼碼，這會讓系統在 ID 相同時自動「覆蓋更新」舊存檔。

#### 4. 載入進度 (Load Data)
* **函式：** `loadGameFromCloud(inputCode)`。
* **邏輯：**
    1. 根據玩家輸入的 `inputCode` 到資料庫查詢。
    2. 若找到資料，將資料庫中的 `total_score`, `level_progress`, `settings` 等欄位重新注入到遊戲變數中。
    3. 同時更新本地的 `localStorage`，確保該裝置從此與該引繼碼綁定。

### 建立 game_logs 資料表（SQL for Supabase）
請至 Supabase 後台 → SQL Editor → New Query，貼上以下 SQL 執行：

```sql
-- 建立遊戲 LOG 資料表
CREATE TABLE game_logs (
    id          BIGSERIAL PRIMARY KEY,
    player_id   TEXT NOT NULL,              -- 玩家引繼碼 (e.g. "杜甫#C9B1")
    played_at   TIMESTAMPTZ DEFAULT NOW(),  -- 遊戲日期時間 (精確到秒)
    duration_s  INTEGER DEFAULT 0,          -- 遊戲完成時間(秒)，目前填0，日後實作
    game_no     INTEGER NOT NULL,           -- 遊戲編號 (1~12)
    difficulty  TEXT NOT NULL,              -- 難度 ("小學"~"研究所") 或關卡編號字串
    score       INTEGER NOT NULL DEFAULT 0, -- 本局分數
    is_win      BOOLEAN NOT NULL DEFAULT true -- 是否勝利
);
-- 開啟 RLS
ALTER TABLE game_logs ENABLE ROW LEVEL SECURITY;
-- 允許所有玩家插入自己的 LOG（使用 player_id 判斷，無需帳號驗證）
CREATE POLICY "Allow insert for all" ON game_logs
    FOR INSERT WITH CHECK (true);
-- 允許管理員 (service_role) 查詢全部
CREATE POLICY "Allow select for service role" ON game_logs
    FOR SELECT USING (true);
-- 建立常用查詢的索引，加快後台統計速度
CREATE INDEX idx_game_logs_player_id ON game_logs (player_id);
CREATE INDEX idx_game_logs_played_at ON game_logs (played_at DESC);
CREATE INDEX idx_game_logs_game_no   ON game_logs (game_no);
```

欄位說明
欄位	型別	說明
id	BIGSERIAL	自動遞增主鍵
player_id	TEXT	玩家引繼碼（如 杜甫#C9B1）
played_at	TIMESTAMPTZ	遊戲完成的時間戳記（精確到秒）
duration_s	INTEGER	遊戲耗時（秒），目前填 0
game_no	INTEGER	遊戲編號（1～12）
difficulty	TEXT	難度字串或關卡編號
score	INTEGER	本局最終分數
is_win	BOOLEAN	是否勝利（true/false）
