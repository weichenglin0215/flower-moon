# 《花月》Game18: 馬戲團 - 遊戲企劃書與遊戲心得

## 第一部分：遊戲企劃書 (Game Design Document)

### 0. 原型遊戲分析

#### A. 原型遊戲玩法解析

**馬戲團 (Circus Charlie, 1984, Konami)**：這是一款 NES 橫向捲軸動作遊戲，玩家扮演馬戲團小丑 Charlie，在六個不同場景中完成雜技表演。最具代表性的兩種關卡型態是：**火環關**，Charlie 騎著獅子向右奔跑，必須在節奏中精準起跳，穿越懸掛在半空中一個接一個的燃燒火環，落點若偏則立即失敗；**彈簧床關**，Charlie 從高空被彈射飛起，必須準確落在一排彈簧床上，每一張彈簧床的位置都不同，落點須精確到格。核心爽點在於「節奏感與精準落點」的完美融合——玩家需要在移動中預判飛行弧度，同時調控起跳時機，熟練後猶如在紙上彈奏一首節拍工整的樂章。操作方式：方向鍵控制左右，按鍵跳躍，判定精確且容錯極低，一旦起跳就無法收回。

原版最吸引人的核心爽點，正是那種「跳出去的瞬間什麼都來不及後悔」的極致緊張感——弧線已定，生死憑空而降，只有預判得到、才能降得安全。

#### B. 原型的介面設計與吸引力分析

原版《馬戲團》的介面以「舞台光感」為核心視覺語言：豔麗的紅黃配色、旋轉的火環、一望無際的橫向舞台，構成了一幅充滿節日喜慶感的視覺奇觀。火環燃燒時帶有輕微的閃爍動畫，彈簧床整排羅列形成強烈的節奏視感。最讓玩家「停不下來」的關鍵，在於**即時的弧線物理反饋**——每一跳的飛行弧度都是確定性的，玩家越熟悉越能預判，形成強烈的「我只差那一步」的重試衝動，讓失敗後不是沮喪而是馬上再來。

花月版本值得致敬保留的視覺元素：橫向舞台的移動感、火環的燃燒特效、以及彈簧床整排羅列的節奏視覺。這些元素將被移植到詩人騎書卷的舞台場景中，保留那份輕快的節日氣氛，同時注入漢字的書卷氣息。

#### C. 詩詞融合分析

| 問題 | 答案 |
|------|------|
| **什麼被替換成詩詞** | 火環上的裝飾紋樣與彈簧床的標記，被替換成詩句中的每一個漢字；原本「跳過所有環/落在床上」的動作，被轉化為「只跳過含有正確詩句字的環/落在標有正確詩句字的床上」 |
| **如何加入語文知識門檻** | 玩家必須在角色飛奔的過程中，瞬間判斷眼前這個環上的字是否正是詩句中「此刻應當出現的那一個字」——不僅要認識這個字，還要記住它在整句詩中的位置與順序；跳錯了字，扣血；略過正確的字，進度停滯且分數清零 |
| **詩詞學習效果** | 強化「字序記憶」與「語感記憶」的雙重積累——在移動壓力下，玩家被迫在腦中「逐字朗誦」下一個目標字，並在視覺掃描中快速比對，這種「移動中的識字判斷」模擬了古代說書人在台上背誦的臨場壓力，記憶深度遠超靜態閱讀 |

#### D. 手機觸控適配：矛盾點與解決方案

**矛盾一：跳躍精度問題（最關鍵）**

```
原始操作：實體按鍵（NES 手把），玩家可以感受按鍵的物理回饋，
          確認「確實按下去了」
矛盾：手機觸控螢幕沒有物理回饋，玩家難以確認跳躍指令是否被接收，
     且手指按下的瞬間往往伴隨輕微抖動，導致「以為已跳」卻錯過時機
解決方案：設置寬敞的「跳躍觸控區」覆蓋整個遊戲畫面的下半部（而非只有一個小按鈕），
         並加入 50ms 的視覺按壓反饋（跳躍按鈕短暫發光），
         讓玩家清楚感知輸入被接收
```

**矛盾二：視覺遮擋問題**

```
原始操作：電視螢幕放在前方，手把握在手中，眼睛可以完整看到整個畫面
矛盾：手機直立時，拇指觸控區域恰好覆蓋畫面下方，可能遮擋彈簧床的位置資訊
解決方案：彈簧床關卡採用「上方預覽欄」設計，在畫面頂端顯示接下來 3 個彈簧床
         的字元順序，讓玩家即使拇指遮擋下方，仍能提前判斷落點
```

**矛盾三：手勢衝突**

```
原始操作：手把上的跳躍按鍵與移動方向鍵分離，不會互相干擾
矛盾：手機觸控時，「點擊跳躍」容易被誤判為「滑動捲頁」，
     瀏覽器預設的 touchstart 行為可能中斷遊戲輸入
解決方案：在整個遊戲 canvas 區域呼叫 preventDefault()，
         並限制觸控事件只在 game-viewport 容器內有效，
         避免滑動手勢觸發頁面滾動而中斷遊戲
```

### 1. 遊戲概述與核心目的

**遊戲名稱**：馬戲團 (Poetry Circus)

**遊戲目標**：玩家扮演騎著書卷在詩詞舞台上飛馳的詩人，穿越兩種截然不同的關卡場景——在「跳圈關」中，以正確的詩句字序為標準，只跳過懸掛在空中那些載著正確詩字的燃燒火環；在「彈簧床關」中，從高空俯衝而下，準確落在標有下一個詩句字的彈簧床上，以連續完美的落點拼出完整詩句。每關結束後，完整詩句如幕布般緩緩展開，讓舞台的喧囂瞬間化為文字的寧靜。

**學習目的**：相較於傳統的「看詩背詩」，《馬戲團》逼迫玩家在「移動壓力」下執行字序判斷。當角色以固定速度奔跑、火環以不可違抗的節奏靠近，玩家必須在零點幾秒內完成「當前詩句到第幾個字了？下一個字應該是什麼？眼前這個環上的字對嗎？」的三步心理運算。這種在**動態場景下的字序識別與決策**，正是認知科學所謂的「壓力下提取練習 (Retrieval Practice Under Pressure)」，所形成的字序記憶遠比靜態閱讀更加牢固，且對語感的培養尤為顯著。

### 2. 遊戲玩法與核心操作 (Gameplay & Controls)

#### A. 遊戲流程

1. **選擇難度** → 系統依難度選取合適長度的詩詞，決定火環移動速度、干擾字比例、以及提示欄的顯示層級
2. **進入第一關（跳圈關）** → 詩人騎著書卷從左向右自動奔跑，舞台上一個接一個的燃燒火環由右向左迎面而來，每個火環外圈標有一個漢字
3. **玩家判斷並決策** → 瞬間識別眼前火環上的字，若符合詩句當前進度的下一個字，則**起跳穿越**獲得分數；若是干擾字，則**不跳（讓角色從環下方奔跑穿過）**；跳錯扣血，跳過正確字則進度清零
4. **關卡切換** → 完成當前詩句字數達標後，詩人優雅落下，舞台燈光漸暗，完整詩句如捲軸般展開於畫面中央，停留 3 秒供玩家閱讀記憶
5. **進入第二關（彈簧床關）** → 詩人被彈射至高空，舞台下方出現一排彈簧床，每張床上標有漢字；玩家必須操控詩人**向左或向右水平移動**，精準落在標有下一個詩句字的彈簧床上，彈起後繼續瞄準下一張床
6. **循環推進** → 兩種關卡交替出現，詩句長度隨關數遞增
7. **勝利條件**：完成所有詩句段落，通關畫面出現，所有詩句組成一首完整的金色卷軸
8. **失敗條件**：生命值歸零（跳錯字扣血），或計時器耗盡（彈簧床關未能在限時內完成正確字序）

#### B. 操作方式

| 平台 | 跳圈關（起跳） | 彈簧床關（左右移動） | 選擇確認 |
|------|-----------|------------|--------|
| **桌機（鍵盤）** | 按空白鍵 / 上方向鍵 | 左右方向鍵 | Enter / 點擊按鈕 |
| **桌機（滑鼠）** | 點擊畫面任意處 | 點擊左側或右側半區 | 點擊按鈕 |
| **手機（觸控）** | 點擊畫面下半部觸控區 | 點擊左側或右側半屏 | 點擊按鈕 |

**操作限制**：
- 跳圈關中，角色水平位移由系統控制，玩家**只能控制起跳時機**
- 彈簧床關中，玩家**只能控制左右移動**，下落弧度由物理引擎決定
- 每一跳一旦起跳，**不能在空中取消或調整**，須提前預判
- 彈簧床落點判定為「整體床面範圍」，有一定容錯，但越居中落下彈起高度越高（奬勵精準）

**手機專屬輔助**：
- 彈簧床關卡的「上方預覽欄」顯示接下來 3 個目標字，讓手指遮擋下方時仍能判斷方向
- 跳躍觸控區發光動畫確認輸入已接收
- 畫面中央顯示當前詩句進度條與下一個目標字的大字提示（可依難度關閉）

#### C. 即時反饋機制

**正確操作**：
- 跳圈關穿越正確字環：環體瞬間從燃燒橙色**轉為金色**，並爆出金色光粒；播放 `playSuccess()` 短促上行音；進度條向前推進一格
- 彈簧床精準落點：彈簧床閃爍金色光芒並播放彈跳音效，角色飛起的弧線留下一道短暫金色尾跡

**錯誤操作**：
- 跳圈關跳入錯誤字環：環體閃爍**朱紅色**並發出震動效果；播放 `playFailure()` 低沉音；生命值減少，角色跌倒再爬起動畫（約 1 秒）
- 彈簧床落在錯誤字床：床面瞬間縮回（誇張動畫），角色墜地播放 `playSadTriple()`；已累積的連續正確字數清零

#### D. 勝利/失敗的標準

- **勝利**：完成當局所有詩句段落的字序挑戰，所有火環/彈簧床上的詩字依序全部正確通過
- **失敗**：生命值（預設 3 顆心）耗盡，或計時器（僅限部分難度）歸零；失敗後畫面定格，顯示「詩人跌落舞台」動畫，並呈現「目前已完成的詩句」與重試按鈕
- **失敗後處理**：顯示目前詩句進度、正確字/錯誤字統計，以及「再試一次」與「換難度」兩個選項

#### E. 獲得高分的技巧

- **眼睛**：不要只盯著角色，要提前看兩到三個環的距離，讓眼睛走在角色前面，心算「準備起跳的字是不是正確的」
- **腦袋**：在腦中默誦詩句，保持節奏感——像在唱歌一樣跟上節拍，讓詩的韻律成為判斷的指南針
- **彈簧床關鍵技**：先確認目標字的位置（左邊還是右邊），在彈起後立即往目標方向移動，不要等到快落下才決策

```
├─────────────────────────────────────────┤  ← 詩句進度欄（40px）

│  ┌───────────────────────────────────┐  │  ← 遊戲主舞台（佔視口 60-65%）
│  │  🔥 [字] 🔥 [字] 🔥 [字] 🔥 [字]   │  │    跳圈關：火環由右向左飄移
│  │                                   │  │
│  │    🧑‍🎨 ←（詩人騎書卷向右奔馳）      │  │    角色水平奔跑，垂直跳躍
│  │  ════════════════════════════════ │  │    舞台地板
│  └───────────────────────────────────┘  │
├─────────────────────────────────────────┤
│  [====跳躍觸控區（手機用，含按鈕感光）====]  │  ← 觸控區（桌機隱藏）
└──────────────────────────────────────────┘  ← 紅色四方形計時條邊框(在遊戲區四周)
```

**彈簧床關切換時的佈局差異**：
- 主舞台區的角色出現在畫面頂端，從高空向下墜落
- 彈簧床整排陳列在舞台底部，每張床上有漢字標牌
- 觸控左半屏=向左移動，右半屏=向右移動，並非點擊個別床面

#### B. 視覺回饋系統

| 狀態 | 視覺表現 | 色彩邏輯 |
|------|---------|---------|
| 火環（普通） | 橙色燃燒動畫，字以白色顯示 | `hsl(30, 90%, 55%)`（火焰橙） |
| 火環（正確字，有提示） | 金色光暈脈動，字體加粗發亮 | `hsl(45, 90%, 65%)`（花月金） |
| 火環（穿越瞬間－正確） | 瞬間爆出金色光粒，環轉金 | 正確態：金橙 |
| 火環（碰到－錯誤） | 閃爍朱紅，角色跌落動畫 | `hsl(4, 90%, 60%)`（花月朱紅） |
| 彈簧床（普通） | 半透明白底，字以深色顯示 | `hsla(0,0%,100%,0.7)` |
| 彈簧床（目標字，有提示） | 翠綠色輝光邊框脈動 | `hsl(160, 60%, 45%)`（深翠綠） |
| 彈簧床（精準落點）| 閃爍金色，彈起弧線留金尾 | 金橙 |
| 彈簧床（落錯） | 床面縮回，朱紅字體放大淡出 | 朱紅 |

#### C. 計時器呈現方式

- **跳圈關**：計時器以「舞台天幕上方的橫向進度條」形式呈現，從左到右逐漸消耗，色彩從暗金色漸變為橙紅色。時間不足 10 秒時，條目開始快速閃爍
- **彈簧床關**：採用矩形邊框 SVG 環繞整個舞台，`stroke-dashoffset` 動畫讓邊框如馬戲帳篷的彩燈一樣逐段「熄滅」，配合音效製造緊張感
- **無計時模式（小學、中學）**：兩種關卡均不顯示計時器，以生命值作為唯一壓力來源

#### D. 使用體驗的流暢度設計

- **火環間距自適應**：根據難度設定的速度值，自動調整火環生成間距，確保玩家每次都有「剛好夠思考但不能懈怠」的決策空間
- **跌落動畫不強迫等待**：角色跌倒動畫播放 0.8 秒即自動復活繼續，玩家不需按任何按鍵，降低挫折感的中斷感
- **彈簧床彈力視覺預測輔助**：落點正確時，從床面延伸出一條虛線弧線（顯示 0.5 秒），預告下一個落點的大致位置，幫助玩家提前規劃，而不是每次都是盲跳
- **難度質感差異**：小學的舞台背景以暖白燈光為主，色調明亮；研究所的舞台背景呈深藍暗場，火環火焰更高更密，彈簧床移速加快，視覺上就能立刻感受到「這是不同層次的戰場」

### 4. 遊戲機制的繼承與創新 (Game Mechanics)

《馬戲團》繼承了橫向捲軸跳躍（Platform Runner）與節奏動作遊戲（Rhythm Action）的設計 DNA——前者帶來移動中的動態判斷張力，後者帶來字序節拍的韻律美感。在此基礎上，花月版本實現了以下三項獨家創新：

**創新一：雙關卡交替敘事結構**

原版《馬戲團》的六個場景相互獨立，各自展示不同雜技，但主題彼此割裂。花月版本將「跳圈關」與「彈簧床關」設計為同一首詩的「前半段」與「後半段」——跳圈關完成詩的前兩句，彈簧床關完成後兩句，兩關合力拼出一首完整的詩。這樣設計的原因，是讓玩家在「快節奏動作」與「精準空間判斷」兩種完全不同的心理狀態之間切換，恰好對應了詩的起承轉合——急促的動感（起、承）之後，彈簧床的空中懸浮感帶來輕盈的轉、合。玩家在不知不覺中，用身體感受了詩的節奏結構。

**創新二：干擾字的語義陷阱設計（知識門檻的核心）**

原版馬戲團的環上沒有任何文字，只是圓環本身，因此不需要任何知識就能玩。花月版本的火環上，除了當前進度的正確字之外，還會同時出現以下三種干擾字，缺一不可地構成語文知識門檻：

- **字形陷阱**：與目標字形近的字（如「清」vs「情」、「已」vs「己」）——考驗漢字識別能力
- **詞義陷阱**：同一語境中語義相近但位置不對的字（如在「窗含西嶺千秋雪」中，出現「東、南、北」而非「西」）——考驗詩詞背景知識
- **詩句陷阱**：來自同一詩人其他詩作的真實詩句字（如同一首詩相鄰句的字）——考驗精確的字序記憶

這種分層設計確保了即使玩家閉眼亂跳，正確率也不超過 20%。

**創新三：關卡間的「詩句幕布時刻」**

每個關卡結束時（不論勝負），舞台的追光燈會熄滅，畫面中央緩緩展開一幅竹簡卷軸，完整呈現本關所挑戰的那段詩句，字體以毛筆書法風格逐字浮現，配合悠揚旋律停留 3-4 秒。這個設計的目的不只是美學享受，而是刻意利用**「情緒高點後的記憶鞏固視窗」**——認知科學研究表明，強烈情緒（剛剛的成功或失敗）之後的短暫靜默期，大腦對新資訊的記憶編碼效率最高。幕布時刻讓詩句在玩家最有記憶容量的瞬間，完整地被觀看一次。

### 5. 核心物理與視覺設計 (Core Physics & Visual Design)

#### 跳圈關的物理設計

**火環生成節奏**：火環以固定時間間隔（`ringInterval`，依難度從 2200ms 到 900ms）從畫面右側生成，以固定速度（`ringSpeed`）向左移動。每個火環承載的字元在生成時由「字元分配器」決定：

1. **字元分配器演算法**：對於一個需要 N 個字的詩句段落，系統生成 M 個環（M 遠大於 N），其中恰好含有 N 個「正確字環」，分散安插在 M 個位置中。其餘環皆填入干擾字。正確字環的位置採用「最小間距保證」——相鄰兩個正確字環之間至少有 1 個干擾環，確保玩家每次都能清楚辨識當前環的字，不會因正確字連續出現導致判斷混亂。
2. **跳躍弧線物理**：角色跳躍採用標準拋物線（`jumpVelocity` + 重力加速 `gravity`），空中時間固定，火環環體半徑（`ringRadius`）決定穿越容錯範圍。角色的跳躍高度設計為「剛好能穿越正常高度的火環」，確保每一跳都是有意義的決策，而非覆蓋所有高度的萬能跳。

#### 彈簧床關的物理設計

**下落弧線**：角色從上方以初速為零開始自由落下，每次落在彈簧床上後，以 `bounceVelocity`（依難度調整）向上彈起，同時玩家可施加水平力（`horizontalForce`）控制左右位移。弧線物理使用 Canvas requestAnimationFrame 逐幀更新，確保流暢度。

**落點判定**：彈簧床的命中判定區域為床面寬度的 80%（留 10% 左右邊緣容錯），系統依落點偏離中心的距離決定反彈高度（精準落中心 = 全力彈起；偏邊緣 = 低力彈起），給予玩家精準落點的正向激勵。

**彈簧床字元排列策略**：每一輪彈跳，底部呈現的彈簧床陣列中，正確字床位置由「隨機不重複分佈」演算法決定——正確字床不會連續出現在同一側（避免玩家只需一直偏向某側），也不會每次都在中間（避免玩家原地不動就能通關）。

### 6. 音樂與音效設計 (Music & Sound Design)

《馬戲團》採用「節日氣氛背景樂 + 情緒精準爆點」的聲音策略，讓玩家在熱鬧的表演氛圍中保持高度專注。

**背景音樂**：跳圈關播放輕快的進行曲風格背景樂（可考慮參照花月內建的《望春風》，以輕快活潑的節奏烘托馬戲舞台的節日感），彈簧床關切換為稍慢的搖擺樂節奏，模擬空中飄浮的輕盈感。背景樂的節拍速度（BPM）與火環的生成間隔同步——難度越高，BPM 越快，音樂本身成為速度壓力的視聽化指標。

**操作音效**：
- 角色奔跑：輕微腳步節拍聲（與跑步動畫幀同步）
- 跳躍起飛：短促的「嗖」音（上揚音調）
- 彈簧床彈起：彈性振動音效（依彈力大小調整音調高低）

**正確/錯誤回饋音**：
- **穿越正確火環**：`playSuccess()`——清亮上行古箏音，與金色光粒動畫同步爆發，製造「這一跳對了！」的即時確認感
- **彈跳正確床**：`playSuccessShort()`——簡短但明確的確認音，維持節奏不中斷
- **跳入錯誤火環**：`playFailure()`——低沉的墜落感音效，配合角色跌倒動畫，情緒上的「落空感」極度真實
- **落錯彈簧床**：`playSadTriple()`——三個下行短音，連環扣心，讓玩家痛感自己剛剛「連續白費」的努力

**關卡間幕布時刻**：幕布展開時，背景樂音量淡出至靜默，改由 `melodyPlayer.playFullMelody('茉莉花')` 或 `melodyPlayer.playFullMelody('踏雪尋梅')` 緩緩奏出，竹簡字體逐字浮現的速度與旋律音符一一對應，讓「看完這段詩」本身成為一種享受而非中斷。

**通關勝利結算**：`ScoreManager.playWinAnimation()` 觸發，配合全螢幕金色拉炮（類馬戲彩帶）動畫，旋律切換至歡快版《望春風》高潮段，頒獎感十足。

**聲音與視覺的協同**：每次 `playSuccess()` 播放的瞬間，火環的金色光粒爆發與音效的第一個音符精確同步；彈簧床的彈跳音調高低與反彈弧線高度成正比——精準落點彈得最高，音調也最高亢，聽覺直接告訴玩家「這次很完美」。

### 7. 難度參數的精密控制 (Difficulty Scaling)

| 難度 | 火環速度 (px/s) | 環生成間隔 | 干擾字類型 | 提示（正確字光暈） | 計時器 | 生命值 | 彈簧床移速 |
|------|--------------|----------|----------|----------------|------|------|---------|
| 小學 | 120 | 2200ms | 僅字形陷阱 | 全亮（每個正確字環） | 無 | 5 顆 | 慢（靜止） |
| 中學 | 160 | 1800ms | 字形 + 詞義 | 下一個目標字亮 | 無 | 4 顆 | 中慢（微移） |
| 高中 | 200 | 1400ms | 字形 + 詞義 | 僅顯示「下一個字」文字提示 | 有（寬鬆） | 3 顆 | 中（緩移） |
| 大學 | 260 | 1100ms | 三種全開 | 無提示 | 有（標準） | 3 顆 | 中快（明顯移動） |
| 研究所 | **340** | **900ms** | 三種全開 + 詩句陷阱加倍 | **完全無提示** | 有（嚴格） | **2 顆** | **快（快速漂移）** |

**質變點在大學難度**：三種干擾字類型全面開啟，且提示光暈完全消失。玩家從「在亮格中確認字」，變成「在滿屏似是而非的漢字中靠記憶篩選正確字」——這是知識門檻從「視覺辨識」升級為「純記憶提取」的關鍵躍遷，體驗截然不同。彈簧床從靜止變為持續漂移，原本「瞄準後再跳」的策略失效，必須在移動目標上即時微調。

**研究所的極限設計**：火環間隔縮短至 900ms，與人類平均視覺辨識反應時間（約 250-300ms）和決策時間（約 400-500ms）相加後，留給玩家的緩衝只有 100-200ms，幾乎是逼迫玩家用「預知」而非「反應」來通關——意即玩家必須已經將整首詩爛熟於心，才能在環到來之前就確認答案。這是一場「知識儲備的壓力測試」，而非純粹的反射神經競賽。彈簧床快速漂移配合計時器嚴格限制，讓「又慢又猶豫」的玩家毫無生存空間。生命值只剩 2 顆，一個不小心就 Game Over，專門為頂尖玩家保留的。

**小學難度的特殊輔助**：彈簧床完全靜止不移動，是唯一讓玩家「落點可以慢慢瞄準」的難度；火環生成間隔充裕（2200ms），玩家有足夠時間讀完環上的字再做決策。這是設計給初學詩詞者或兒童的溫柔邀請，確保任何識字的人都能體驗到完整的遊戲流程。

## 第二部分：遊戲心得與分析 (Game Experience & Analysis)

### 1. 遊戲性分析

「這是一場在燃燒的舞台上，用腳步丈量詩句的飛奔告白。」

《馬戲團》是整個花月專案中，將「身體節奏」與「文字記憶」融合得最激烈、最具娛樂爆發力的一款作品。

當詩人騎著書卷在橙色火光中奔跑，火環們以不可撤回的速度撲面而來，玩家的心理狀態迅速進入一種奇特的「雙軌並行」狀態：一條軌道是身體的反應迴路（這個環有沒有過？跳還是不跳？），另一條軌道是詩詞的記憶迴路（現在詩到第幾個字了？下一個字是「光」還是「空」？）。兩條軌道在大腦中高速平行運算，稍有失調便會兩敗俱傷——要麼身體跳了、腦子沒跟上，要麼腦子記得字、身體卻慢了半拍。這種「智識壓力 × 動作壓力」的雙重奏，正是《馬戲團》最核心的爽點來源，也是它教學效果遠超純問答遊戲的理由。

彈簧床關則帶來截然不同的心理體驗。從高空下落的那一刻，時間彷彿被拉長了——玩家在空中短暫的懸浮感中掃視下方一排字床，鎖定目標，操控方向，在落地的剎那獲得確認。成功落點的彈簧床發出爽脆的彈跳聲，角色又被彈回空中，整個過程如同在詩句的韻律上跳繩——每一個字都是一個落點，每一次彈跳都是一個詩節的節拍。

學習效果上，《馬戲團》主要強化的是**字序記憶**與**語感記憶**的協同——玩家不只是「知道這首詩有這些字」，而是「在移動壓力下，準確知道此刻是哪個字、下一個是哪個字」，這正是詩詞記憶中最難、也最有價值的部分：在語流中精確定位。

**頂點時刻**：當玩家在研究所難度下，火環以 340px/s 的速度如彗雨般密集撲來，玩家已無暇思考、只憑詩句在大腦中流淌的韻律本能起跳，在接連十個正確穿越之後，最後一個字的火環爆出漫天金色彩彈，背景音樂切換為《望春風》高潮，舞台上的彩帶和燈光如同馬戲謝幕一般齊放——在那一瞬間，玩家意識到：「我不是在玩遊戲，我已經把這首詩烙進了我的神經裡。」

### 2. 精華推廣亮點

**亮點一：「火環密到讓你沒時間思考，只剩下直覺——而直覺，就是你背了多少詩」**

這是最適合短影音 (Reels / Shorts) 的場景：實況主在研究所難度下，畫面中火環如雨點般高速滾來，實況主喃喃唸著詩句，手指飛速連擊，偶爾一個「清」字跳到「情」字的火環上，爆出朱紅閃光並發出低沉墜落音，實況主大叫「啊不是啦！差那個氵而已！」——這個反應會讓旁觀者既覺得好笑，又忍不住好奇「如果是我，我能分辨嗎？」
標題可以下：**「字形長得太像，我被古詩陷阱耍了！《馬戲團》研究所速通挑戰」**

**亮點二：「彈簧床飛行的 0.5 秒，是整款遊戲最優雅的時刻」**

在彈簧床關，每次從彈起到下落的那短暫懸空，角色在空中、畫面靜定，玩家在腦中急速回憶「下一個字是哪個」——這個「思考懸浮的一瞬間」極適合放慢播放的截幀影片，讓觀眾感受到「哦，遊戲在這裡給你思考的空間」。同時，對於高手而言，這 0.5 秒的決策不是猶豫，而是行雲流水的確認，形成「慢鏡頭下的大師風範」的強烈反差效果。
標題可以下：**「看我在空中 0.5 秒判斷詩句字序，這就是研究所難度的優雅」**

**亮點三：「難度落差的反差挑戰——小學生版跟研究所版根本是兩款遊戲」**

讓一個剛學詩詞的小學生玩小學難度，再讓高中生玩研究所，把兩段錄影剪在一起：一個是悠哉穿越緩緩而來的環，另一個是手忙腳亂地被連環密集火環轟炸。這種難度反差的對比影片是天然的病毒素材，讓觀眾在笑聲中體會到「這個遊戲真的有在考詩詞知識」。
標題可以下：**「小學生秒通關，高中生被難哭——同一首詩，五種地獄等級的《馬戲團》」**

### 4. 總結

《馬戲團》讓古典詩詞第一次在「時速三百」的場景下接受檢驗。透過「跳圈關」與「彈簧床關」的雙關卡交替設計，玩家的記憶在動態壓力與空間判斷之間被反覆鍛造；三層語義陷阱的干擾字設計，從根本上杜絕了盲目亂跳的可能性，讓每一個成功穿越都是真實知識的勝利；關卡間的「詩句幕布時刻」，更將情緒高點轉化為記憶鞏固的黃金窗口，讓遊戲的教學效果蔓延至動作場景之外。研究所難度下的 900ms 火環間隔，是一道對「爛熟於心」的終極試煉——到了那個境界，字序不再是記憶，而是本能。這不是在讓詩詞「好玩」，這是在用馬戲的節奏，替詩詞打開一扇不曾有過的門。
