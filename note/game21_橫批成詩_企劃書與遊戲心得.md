# 《花月》Game21: 橫批成詩 - 遊戲企劃書與遊戲心得

## 第一部分：遊戲企劃書 (Game Design Document)

### 1. 遊戲概述與核心目的

**遊戲名稱**：橫批成詩 (The Poetry Banner)
**命名典故**：中國傳統門聯由左右兩幅「對聯直條」與頂端一幅「橫批牌匾」組成。本作的七根直棒宛如懸掛於門框的竹簡聯條，而貫穿其間的半透明金色橫框，正是那一幅等待被玩家親手題就的橫批——以指尖推移竹簡，讓千年前那句詩，在橫批之上重現光芒。

**遊戲目標**：在 7×7 的字元格子中，透過左右拖曳讓直棒換位、上下拖曳讓直棒滑動，使每根直棒中的正確字元都對齊中央的「黃金橫批框」（第 4 橫行），讓橫批從左到右拼出隱藏的題目詩句，即告勝利。

**學習目的**：花月系列的其他遊戲考驗「記住自己的詩」，《橫批成詩》更進一步，要求玩家在整個詩庫中進行「交叉定位」——你不只要知道「床前明月光」的內容，還要知道「哪首詩裡有個『床』字」、「那個『月』出現在詩句的第幾個位置」。這種跨詩句的字元空間記憶，是最接近真正古典文學涵養的認知訓練，遠超單首詩的死記硬背。

### 2. 遊戲玩法與核心操作

#### A. 遊戲流程

1. **選擇難度** → 系統抽取符合條件的詩詞，選取其中一句作為謎題（五言或七言，依難度決定）
2. **產生直棒**：每個謎題字對應一根直棒（一整句詩）；另依難度加入若干根干擾棒；所有棒的欄位順序隨機打亂，垂直位移也隨機設定，確保正確字元不在黃框位置
3. **顯示提示**：頂端提示行依難度設定，以不同延遲秒數逐字或整句顯示謎題詩句（如「床─前─明─月─光」），讓玩家知道目標
4. **低難度預置輔助（`fixBarCount`）**：指定數量的直棒已預先放在正確欄位，但垂直位移仍然錯誤，且這些棒**只能上下滑動，不能左右換位**，引導初學者分步學習兩種操作
5. **玩家操作**：以手指或滑鼠拖曳直棒，組合左右換位與上下滑動，讓黃框橫行拼出謎題
6. **勝利條件**：系統讀取黃框第 4 行所有欄位的實際字元，左到右完全符合謎題詩句
7. **失敗條件**：計時器歸零

**三種出題格式的差異：**

| 謎題格式 | 棒的數量 | 干擾棒 | 難度定位 |
|---------|---------|-------|---------|
| 五言（5 字）| 5 根謎棒 + 0-2 根干擾棒 | 可以有 | 小學至高中 |
| 七言（7 字）| 7 根謎棒 | 無（7 棒填滿 7 欄）| 大學至研究所 |

#### B. 操作方式

**核心互動：自由二維拖曳（Free 2D Drag）**

玩家點按任意直棒後，可自由向任意方向拖曳：

| 拖曳方向 | 效果 | 單位 |
|---------|-----|-----|
| **左拖** | 此棒與左側相鄰棒換位 | 每越過一個欄位寬度 = 一次換位 |
| **右拖** | 此棒與右側相鄰棒換位 | 同上 |
| **上拖** | 棒內容上移，黃框顯示的字往「棒的後方」移動 | 每移過一個字元高度 = 一格 |
| **下拖** | 棒內容下移，黃框顯示的字往「棒的前方」移動 | 同上 |
| **斜向拖** | 同時完成換位與內容滑動 | 自動分解為水平與垂直分量 |

**斜向拖曳的設計意義**：玩家只需一個連續手勢，就可以把一根棒「推到正確欄位」並「調整到正確字元對齊黃框」——這讓熟練玩家能以流暢的掃掠動作完成整個拼圖，形成行雲流水般的「書法手感」。

| 平台 | 開始拖曳 | 持續 | 結束 |
|------|---------|------|-----|
| **桌機（滑鼠）** | mousedown | mousemove | mouseup |
| **手機（觸控）** | touchstart / pointerdown | touchmove | touchend |

**操作限制**：
- `fixBarCount` 預置棒：**只能上下拖曳，左右完全鎖定**（有視覺鎖定標識）
- 棒的長度可長於 7 格，但只顯示 7×7 格子內的部分，超出部分不可見（不破壞介面）
- 棒滑動到邊界（棒的字元用盡）時自動停止，不能繼續滑

#### C. 即時反饋機制

- **拖曳中**：直棒跟隨手指流暢移動，每個字元高度的位移瞬間更新黃框顯示字
- **黃框即時配對指示**：黃框中每一欄，若當前顯示的字符合謎題對應位置，則以**金色字體**顯示；不符合則以**暗灰色**顯示——玩家能實時看到「目前對了幾個」，但不知道哪些位置是對的（防止秒解）
  - *注意：為確保挑戰性，金色僅表示「此欄當前字是謎題中的一個字」，不標示是否在正確欄位*
- **全解瞬間**：黃框字元由左到右依序亮起金光，觸發連鎖發光動畫，伴隨逐音符的旋律上行
- **計時警告**：剩餘時間低於 25%，SVG 邊框由暗紅轉為鮮紅，配合加速的心跳脈動

**佈局說明：**
- 頂端提示行與黃框橫行都是「橫向顯示字元」，玩家可以視覺上直接對比「提示行目標」與「黃框當前狀態」，天然建立「需要讓它們一致」的認知
- 七根直棒每棒寬度 = 格子寬度（約 57px，7 棒 × 57px ≈ 400px，配合 500px 容器留邊距）
- 黃框橫行有明顯的半透明金色底色 + 加粗邊框，與其他行明確區分

#### B. 視覺回饋系統

| 狀態 | 視覺表現 | 色彩 |
|------|---------|------|
| 普通格子 | 半透明白底，白色文字 | `hsla(0,0%,100%,0.15)` |
| 黃框橫行底色 | 半透明金色背景 | `hsla(45, 80%, 60%, 0.25)` |
| 黃框字（符合謎題某字）| 金色粗體文字 + 輕微光暈 | `hsl(45, 90%, 70%)` |
| 黃框字（不符合）| 暗灰色文字 | `hsla(0,0%,60%,0.5)` |
| 預置固定棒（`fixBarCount`）| 棒頂端顯示鎖鏈圖示 🔒，棒邊框特殊顏色 | `hsl(210, 60%, 50%)` |
| 提示行字元（已顯示）| 金橙色文字 | `hsl(36, 80%, 60%)` |
| 提示行字元（未顯示）| 暗色佔位符「_」| `hsla(0,0%,50%,0.3)` |
| 全解瞬間（逐欄亮起）| 由左至右依序金光炸裂 | 動畫逐欄觸發 |

#### C. 計時器呈現方式

SVG 矩形邊框**環繞整個 7×7 格子**（而非只圍繞答案區），`stroke-dashoffset` 動畫讓四邊緩緩消失。格子本身就是計時器的視覺容器，讓時間壓力感更加立體——邊框消失，代表「承載著謎題的竹簡架子正在被侵蝕」。剩餘 25% 時轉為鮮紅。

#### D. 使用體驗的流暢度設計

- **斜向拖曳的容錯設計**：系統根據拖曳向量的水平/垂直分量比例，智能判斷「主導方向」與「次要方向」。水平分量 > 垂直分量時，優先執行換位操作；反之優先執行滑動。這確保玩家不會因手指輕微偏斜而誤觸操作
- **慣性滑動（僅上下）**：直棒的垂直滑動支援一定的慣性效果，讓玩家快速刷過多個字元時手感更流暢，不需要一格一格精確移動
- **低難度鎖定棒的視覺引導**：預置固定棒的藍色邊框 + 鎖圖示，讓初學者立刻明白「這幾根只能上下動」，降低認知負擔，讓他們先專注在學習垂直滑動操作
- **勝利後防點擊保護**：全解動畫觸發後立刻禁用所有拖曳操作，防止玩家在動畫播放中誤觸棒件，干擾結算流程

### 4. 遊戲機制的繼承與創新

本作靈感源自兩個截然不同的傳統：其一是中國古代的「竹簡排序」（古人閱讀竹書，需要將散亂的竹簡按正確順序排列方能讀懂），其二是西方的「滑動拼圖（Sliding Tile Puzzle）」概念。但本作在詩詞學習的融合上，做出了在整個花月系列中最具空間感的獨家創新：

- **二維自由拖曳的操作革命（Free 2D Drag）**：花月系列所有遊戲的互動都是「點擊選擇」或「拖曳連線」。本作是首款要求玩家在**二維空間中自由操控物件**的作品——左右換位與上下滑動可以在同一個連續手勢中完成，讓解題過程充滿了「收拾桌面」般的物理真實感，與竹簡整理的歷史意象高度呼應。
- **跨詩句字元定位考驗（Cross-Poem Character Mapping）**：其他填空遊戲的考驗範圍侷限於「一首詩自身的記憶」。本作要求玩家知道「哪根直棒的哪個位置藏著對應的字」——等於同時考驗了多首詩的記憶，是花月系列中知識廣度要求最高的一款。
- **預置輔助系統差異化引導（`fixBarCount` Scaffolding）**：低難度特有設計。部分棒預先擺放在正確欄位但垂直位移錯誤，且鎖定只能上下操作。這個設計讓初學者能拆分學習「垂直滑動」與「水平換位」兩種操作，不會因兩個維度同時學習而感到迷失——是教學設計的精心安排。
- **黃框字元讀取判定（Character Reading Win Condition）**：勝利判定不依賴「哪根棒在哪個欄位」，而是直接讀取黃框橫行的實際字元序列，對照謎題。這解決了「一根棒含有多個謎題字」與「謎題含疊字」的邊界情形——當玩家把含有『明月』的棒定位，使『月』出現在黃框，系統正確讀到『月』而非假設它是『明』。此設計參照《詩韻鎖扣（Game9）》的鎖位判定邏輯。

### 5. 題目生成與直棒配對演算法

#### 1. 謎題詩句的選取

系統呼叫 `getSharedRandomPoem()` 取得符合難度的詩詞，選取一行作為謎題（五言或七言）。

#### 2. 每個謎題字的直棒配對

對謎題中每個字（以「床前明月光」為例，依序找「床」「前」「明」「月」「光」的直棒）：
- 從詩庫中搜尋**包含該字**的詩句
- 優先選取**字所在位置接近中段**的詩句（避免目標字剛好在棒的頂端或底端，減少滑動量過大的挫折感）
- 優先選取**語意上與謎題詩句有關聯**的詩句（如謎題是「床前明月光」，「月」的棒優先選取「舉杯邀明月」而非「月月加餐飯」，增加文學關聯的趣味）
- 研究所難度：優先選取同時含有**多個謎題字的詩句**（製造最大干擾，玩家必須精確滑動到正確字的位置）

#### 3. 干擾棒的生成

- 干擾棒所使用的詩句**必須不包含謎題中的任何字**（嚴格排除，避免干擾棒被誤以為是謎棒）
- 棒的長度（字數）與謎棒盡量相近，視覺上難以區分
- 小學難度：干擾棒數量為 0；研究所七言詩時亦為 0（7 棒填滿 7 欄，無空間）

#### 4. 初始隨機化設定

```
初始欄位順序：所有直棒（謎棒+干擾棒）隨機排列，確保正確順序不在初始狀態出現
初始垂直位移：每根棒隨機設定偏移量，且保證目標字不在黃框行
預置棒（fixBarCount）：欄位設定為正確位置，但垂直偏移仍隨機（且保證目標字不在黃框）
```

#### 5. 勝利判定邏輯

```javascript
// 參考 Game9（詩韻鎖扣）的判定寫法
function checkWin() {
    const yellowRowChars = []; // 收集黃框行所有欄位的實際字元
    for (let col = 0; col < totalBars; col++) {
        const bar = bars[col];
        const charAtYellow = bar.getCharAtRow(YELLOW_ROW_INDEX);
        yellowRowChars.push(charAtYellow);
    }
    // 讀取前 puzzleLength 個欄位，比對謎題
    const candidate = yellowRowChars.slice(0, puzzleLength).join('');
    return candidate === puzzleText;
    // 注意：不判斷棒的 id 或位移數值，只判斷黃框實際顯示的字
}
```

### 6. 音樂與音效設計

《橫批成詩》是花月系列中**最適合使用背景音樂**的遊戲——其他遊戲的填字選答需要玩家在腦中默唸詩句，背景音樂會造成干擾。但本作的操作是空間視覺的，玩家不需要在腦中逐字念讀，而是以視覺掃描+手指拖曳解題，適合搭配低音量的傳統音樂烘托氛圍。

**背景音樂**：以低增益播放傳統器樂旋律（推薦 `melodyPlayer.playFullMelody('茉莉花')` 或 `'踏雪尋梅'`），模擬「在書房中整理古籍竹簡」的沉浸感。音樂不能蓋過操作音效。

**操作音效**：
- 抓取直棒（拖曳開始）：`playOpenItem()`——輕柔的竹擊聲，暗示「拿起了一根竹簡」
- 直棒換位（左右越過相鄰棒）：`playSuccessShort()`——短促清脆音，每換一位觸發一次，形成節奏感
- 直棒垂直滑動（每移過一個字元高度）：`playGuzheng(index)` 低增益播放——如手指撥過算盤珠，造就「竹珠滑動」的聽覺質感；每個字元對應一個微小音符

**答案對齊音效**：
- 黃框某欄字元變為正確（符合謎題字）：播放對應音階的古箏音（每個謎題字位對應不同音階，5 字謎題對應 Do-Re-Mi-Sol-La，7 字謎題對應完整音階）
- 這讓黃框逐漸被填對的過程，自然形成一段**不完整的旋律片段**，直到全部正確後旋律才完整

**勝利結算**：
- 黃框字元由左到右依序亮起，同時對應音符由低到高播放（Do→Re→Mi→Sol→La，五聲音階正是中國傳統五聲調式）
- 最後一個字元亮起後，`playJoyfulTriple()` 三連音爆發，接著 `ScoreManager.playWinAnimation()` 播放完整的「橫批旋律」
- 聲音由「零散的滑動聲」→「逐字浮現的音符」→「完整的旋律爆發」，完美呼應視覺從「散亂竹簡」→「逐字對齊」→「橫批完成」的三階段

**超時失敗**：`playSadTriple()`，同時正確答案的黃框字元以藍色顯示（教育提示），讓玩家在惋惜中學習。

### 7. 難度參數的精密控制 (Difficulty Scaling)

| 難度 | 時限 | 詩評下限 | 詩型 | 提示延遲 | 提示模式 | 預置棒數 (`fixBarCount`) | 干擾棒上限 (`maxDecoyCount`) |
|------|------|----------|------|---------|---------|----------------------|--------------------------|
| 小學 | 120秒 | ★★★★★★ | 五言 | 0秒 | 即時全顯 | 3 棒 | 0 棒 |
| 中學 | 90秒 | ★★★★★ | 五言 | 5秒後逐字 | 每 3 秒一字 | 2 棒 | 1 棒 |
| 高中 | 70秒 | ★★★★ | 五言 | 10秒後 | 一次全顯 | 0 棒 | 2 棒 |
| 大學 | 55秒 | ★★★ | 七言 | 20秒後逐字 | 每 5 秒一字 | 0 棒 | 0 棒† |
| 研究所 | **40秒** | ★★★ | 七言 | **永不顯示** | 從不 | 0 棒 | 0 棒† |

†七言詩 7 根謎棒已填滿 7 欄，無空間置入干擾棒

**質變點一（中學→高中）**：`fixBarCount` 歸零，玩家首次面對「7 根棒全部需要自己排序」的完整挑戰，同時加入 2 根干擾棒（5 謎棒 + 2 干擾棒 = 7 棒），必須在操作中識別干擾棒並將其排除於謎題位置之外。這是從「輔助模式」到「完全自主」的轉捩點。

**質變點二（高中→大學）**：詩型從五言（5 字）升級為七言（7 字）。謎棒數量從 5 根增加為 7 根，七言詩的每根棒字數更多（最長可達 9-11 字），垂直滑動範圍更大，且 7 棒的操作複雜度遠高於 5 棒。更關鍵的是，七言詩本身語意更複雜，玩家找到「某字在哪首詩的哪個位置」的難度指數級上升。

**研究所的極限設計**：七言謎題（7 根棒全部填滿格子）、提示永不顯示（玩家必須從記憶中知道要拼什麼）、40 秒時限（平均每棒僅 5-6 秒）、完全沒有預置或輔助。玩家必須同時在腦中持有謎題詩句的記憶、在 7 根棒中識別各字的位置、以高效的斜向拖曳完成所有操作——這是空間記憶與古典文學知識的終極交叉考驗。

## 第二部分：遊戲心得與分析 (Game Experience & Analysis)

### 1. 遊戲性分析：「這是一場用指尖在時間的壓力下重建古詩空間感的立體拼圖」

《橫批成詩》是花月系列中操作維度最獨特的一款作品——所有其他遊戲的核心動作是「點擊選擇」，而本作的核心動作是「二維拖曳排列」，是真正意義上的空間拼圖。

心理狀態的三階段演變：

**第一階段：有序的混沌**——遊戲剛開始，7 根竹簡排列雜亂，黃框顯示的字毫無規律。頂端提示行的字元正在緩緩浮現，玩家的大腦開始掃描：「哪根棒裡有個『前』字？」、「那個『月』埋在哪首詩的第幾行？」這種「在迷霧中搜尋」的認知過程，比填字選答更接近真正的詩詞記憶提取。

**第二階段：掌控的快感**——當玩家理解了斜向拖曳的操作邏輯，每一個手勢都同時完成「換位」與「滑動」，整個解題過程變得流暢且節奏感強烈。每次將一根棒推到正確位置、目標字滑入黃框，那個短促的確認音與金色閃光給予即時的正向回饋，如同將一片拼圖精確嵌入它的缺口。

**第三階段：完成的儀式感**——最後一根棒對齊，黃框字元由左到右依序亮起，旋律從零散的音符凝聚為完整的五聲調式——這個「竹簡排好、橫批完成」的視聽體驗，讓玩家真實感受到「我親手重建了這句詩」，而非只是「我選對了答案」。

**頂點時刻**：在研究所難度，7 根完全無標記的竹簡，0 個提示字，40 秒倒計時。玩家憑藉對整個詩庫的記憶，以流暢的斜向拖曳逐一定位每根棒，最後一根棒滑入到位的瞬間，旋律爆發、金光炸裂——那不只是「答對了」，那是「我把七首詩的片段在一塊竹簡板上拼出了千年前的那句話」的滿足感。

### 2. 精華推廣亮點：足以引爆社群討論的核心魅力

- **「零提示七言速通」的視覺震撼**：
  研究所玩家以 20 秒完成七言詩的畫面——7 根竹簡以流暢的掃弧手勢迅速就位，黃框字元一一亮起——這個過程的視覺奇觀完全不需要旁白解說，任何人看到都能感受到「這人背詩背得有多熟」。短影音天然素材，無需剪輯說明。
  標題可以下：**「20 秒，7 根竹簡，零提示——他把整首詩裝進了那條金色橫批裡。」**

- **「斜向拖曳神技」的炫技感**：
  一般玩家是「先左右換位，再上下滑動」分兩步操作；高手玩家是斜向一刷，換位和滑動同時完成。這個操作差異在錄影中一眼可見——高手的操作如書法家揮毫，普通玩家的操作像搬貨物。這種「操作水平即知識水平」的可視化效果，非常適合社群挑戰影片。
  標題可以下：**「你操作竹簡的方式，暴露了你的詩詞功底——初學者搬，高手舞。」**

- **「哪根棒藏著那個字」的文學偵探感**：
  玩家面對 7 根詩句棒，必須識別哪根含有目標字，才能知道要滑動哪根、滑到哪裡。這個「在詩句叢林中定位一個字」的過程，高度類似文學研究中的索引查詢，對中文系學生或文學愛好者有強烈的共鳴感和挑戰欲。
  標題可以下：**「『月』這個字，在七首詩裡都出現了——但謎題要的那個『月』，只在其中一首的第五個字位置。你找得到嗎？」**

### 3. 對外推廣短語提案

> *「七根竹簡橫陳眼前，每根藏著你要找的那個字——不是選，不是填，是親手推動，讓那句千年前的詩，在黃金橫批裡一字一字地浮現。」*
> *「小心！有些棒子裡藏著兩個謎題字——你把它滑過頭，滑到的那個字是對的，系統還是算你錯。研究所難度下，你只有 40 秒，零提示，容不得一次滑錯。」*
> *「這不是在考你背不背得出古詩，這是在考你知不知道『舉杯邀明月』的『明』字在第四個位置，而不是第三個——這才叫真正讀懂了一首詩。」*

### 4. 總結

《橫批成詩》以「二維自由拖曳」的空間互動革命，為花月系列開創了前所未有的玩法維度。它不再問「這個字對不對」，而是問「你能在詩庫的叢林中找到它、定位它、親手把它推入黃金橫批的正確位置嗎？」——這個過程所要求的詩詞知識廣度，是整個系列最高的。從小學的三棒預置輔助到研究所的七棒零提示極限，難度的質變點清晰而殘酷；斜向拖曳的操作設計讓熟練玩家能展示出行雲流水般的「書法手感」；而每次全解瞬間的五聲旋律爆發，則讓「排好竹簡」這個行為上升為一場古典美學的儀式完成。它不只是遊戲，它是每一位背過唐詩的人，心底那份「我確實讀懂了它」的驕傲，被竹簡與橫批具象化的那一刻。
