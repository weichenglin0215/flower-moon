# 《花月》Game21: 橫批成詩 - 遊戲企劃書與遊戲心得

## 第一部分：遊戲企劃書 (Game Design Document)

### 1. 遊戲概述與核心目的

**遊戲名稱**：橫批成詩 (The Poetry Banner)
**命名典故**：中國傳統門聯由左右兩幅「對聯直條」與頂端一幅「橫批牌匾」組成。本作的直棒宛如懸掛於門框的竹簡聯條，而貫穿其間的金色橫框，正是那一幅等待被玩家親手題就的橫批——以指尖推移竹簡，讓千年前那句詩，在橫批之上重現光芒。

**遊戲目標**：在 **8（寬）× 13（高）** 的字元棋盤上，透過左右拖曳讓直棒換位、上下拖曳讓直棒滑動，使每根直棒中的正確字元都對齊中央的「黃金橫批列」（**第 7 列、0-based index 6**），讓橫批從左到右拼出隱藏的題目詩句，即告勝利。

**學習目的**：花月系列的其他遊戲考驗「記住自己的詩」，《橫批成詩》更進一步，要求玩家在整個詩庫中進行「交叉定位」——你不只要知道「床前明月光」的內容，還要知道「哪首詩裡有個『床』字」、「那個『月』出現在詩句的第幾個位置」。這種跨詩句的字元空間記憶，是最接近真正古典文學涵養的認知訓練。

### 2. 遊戲玩法與核心操作

#### A. 遊戲流程

1. **選擇難度** → 系統抽取符合條件的詩詞
2. **產生謎題**：依難度決定詩型（小學固定五言；其他難度於每局隨機抽五言或七言）
3. **產生直棒**：每個謎題字對應一根直棒（一整句詩，**保留原始字元、不補字、不截斷**）；另依難度加入若干根干擾棒（不含任何謎題字）；所有棒的欄位順序隨機打亂，垂直位移也隨機設定
4. **顯示提示**：頂端提示行依難度設定，以不同模式顯示謎題（見「提示行規格」一節）
5. **預置棒**：依 `maxFixBarCount` 隨機選取若干根謎棒，**上下左右全部就位且完全鎖定**（無法拖曳）
6. **玩家操作**：以手指或滑鼠拖曳直棒，組合左右換位與上下滑動，讓中央橫批列拼出謎題
7. **勝利**：系統讀取中央橫批列前 `puzzleLength` 欄的實際字元，左到右完全符合謎題詩句
8. **失敗**：計時器歸零

#### B. 棋盤規格（重要：與初版設計不同）

| 項目 | 規格 | 備註 |
|------|------|------|
| 棋盤寬度 | 動態 = `min(8, puzzleLength + decoyCount)` | 七言+干擾 3 → 干擾自動降為 1 |
| 棋盤高度 | **固定 13 列** | 中央列 = 第 7 列（index 6）= 黃金橫批列 |
| 直棒長度 | 依詩句原長（4~12 字皆可） | **不補字、不截斷**，五言與七言皆可作為直棒 |
| 直棒可超出範圍 | **是** | 棒可向上下溢出棋盤；任一字皆能被滑到中央列 |

#### C. 操作方式

**核心互動：自由二維拖曳（Free 2D Drag）—— 以 pointer events 統一滑鼠與觸控**

| 拖曳方向 | 效果 |
|---------|-----|
| **左拖／右拖** | 此棒與相鄰非 fix 棒換位 |
| **上拖／下拖** | 棒內容上下滑動，每移過一個字元高度為一格；可超出棋盤上下緣 |
| **斜向拖** | 同時完成換位與內容滑動 |

**拖曳限制與行為**：
- **Fix bar 完全鎖定**：`onDown` 與 `onMove` 內 `if (bar.isFixed) return`；視覺顯示 `not-allowed` 游標
- **跳過 fix bar 的水平換位**：玩家拖曳非 fix 棒越過 fix bar 時，會「跳躍式」與 fix bar 另一側的非 fix 棒互換 currentCol（fix bar 完全不動）
- **垂直滑動範圍**：`offset ∈ [MIDDLE_ROW - L + 1, MIDDLE_ROW]`，即 bar[0] 最低可到中央列、bar[L-1] 最高可到中央列
- **stageScale 換算**：拖曳座標除以 `window.stageScale`，確保各種縮放下手感一致

#### D. 即時反饋機制

- **拖曳中**：直棒跟隨手指流暢移動（`transition: none` 拖曳時關閉動畫）
- **中央列字配對指示**：⚠️ 僅在 `showRightChar = true`（小學限定）時開啟：當中央列字 = `bar.targetChar`（垂直已對齊正解）時，以金色 `yellow-hit` 高亮；其他難度完全不上色
- **拖曳音效**：拿起 `playOpenItem()`；換位 `playSuccessShort()`；垂直滑動每移一格 `playGuzheng(index, 0.4)`
- **全解瞬間**：黃框字元由左到右依序亮起金光（`win-flash` 動畫），每欄觸發對應 `playGuzheng(i, 1.0)`，最後 `playJoyfulTriple()` 三連音

#### E. 勝利後行為

⚠️ 勝利瞬間**完全凍結所有棒**：
- `this.isActive = false`，`onMove` 入口加 `if (!this.isActive) return` 守衛
- `this.drag = null` 清除進行中的拖曳
- 所有棒設為 `isFixed = true` 並套上 `.fixed` class，後續任何點擊都被擋下

### 4. 遊戲機制的繼承與創新

- **二維自由拖曳的操作革命**：花月系列首款要求玩家在二維空間中自由操控物件的作品。
- **跨詩句字元定位考驗**：要求玩家知道「哪根直棒的哪個位置藏著對應的字」——同時考驗多首詩的記憶。
- **預置輔助系統（`maxFixBarCount`）**：
  - 數量為「**上限**」：每局實際數量於 `[1, maxFixBarCount]` 範圍隨機（> 0 時至少保證 1 根）
  - 上下左右全部就位且完全鎖定（不是「只能上下移動」）
  - 重來時**重新隨機指派** fix bar（早期版本錯誤地把 fix 全部解開，已修正）
- **干擾棒偽裝**：視覺與一般棒完全相同，玩家無法靠外觀分辨，必須真的觀察字元。
- **可超出邊界的滑動**：直棒可向上下溢出棋盤，每個字皆能被滑到中央列。
- **黃框字元讀取判定**：勝利判定不依賴「哪根棒在哪個欄位」，而是直接讀取中央列實際字元序列。

### 5. 題目生成與直棒配對演算法

#### 1. 謎題詩句的選取（三段式退讓）

題目詩句篩選嚴格遵守 `poemMinRating`，且**句評分**（`line_ratings[i]`）也必須達標：

1. **嚴格**：`poem.rating ≥ poemMinRating` **且** `line_ratings[i] ≥ poemMinRating` **且** 孤立字（`countIsolatedChars`）**≤ 1**
2. **退讓一**：去掉孤立字限制，保留詩+句評分
3. **退讓二**：僅詩評分達標（最後保底）

**孤立字定義**：透過 `window.countIsolatedChars(clean)` 計算 — 該句中無法在其它句找到的字數量。若 > 1，視為不適合作為謎題（會產生太多 fix bar）。

#### 2. 直棒配對（呼叫 `window.getSharedBarLine`）

⚠️ 直棒詩句**不受 `poemMinRating` 限制**，但內部以評分加權隨機：

```js
window.getSharedBarLine(targetChar, {
    minLen: 4, maxLen: 12,
    excludePoemId: poem.id,   // 優先排除同首詩
    excludeLines: usedLines,  // 避免重複使用同句
    preferMid: true           // 偏好 targetChar 在句中段
});
```

選句策略：
- 權重 = `rating + 1`（保證最低評分也有機率）
- 同分群內依「字距中段距離」排序，取前 60% 隨機，兼顧多樣性
- 三段式退讓：嚴格 → 放寬同詩 → 完全放寬長度

#### 3. 干擾棒生成

- 不含任何謎題字（嚴格排除）
- 長度 4~12 字，視覺與謎棒無法區分
- 評分加權隨機選取

#### 4. 共用工具（已上移至 `script.js`）

```js
window.getSharedBarLine(ch, opts)   // 為「依字反查詩句」玩法挑句
window.countIsolatedChars(clean)    // 計算句中孤立字數量
window.getCharLineCount(ch)         // 某字出現的「行數」
window.getCharGlobalFreq(ch)        // 某字的字頻（使用 CharacterFrequencyRank）
```

並建立懶建立的 `window.__poemLineIndex`（char → 含此字的乾淨行陣列），O(1) Map lookup。

#### 5. char rank 工具更新

`tools/generate_char_rank.js` 與 `.py` 的 `minRating` 從 **4 改為 1**，讓 `CharacterFrequencyRank` 涵蓋全詩庫。⚠️ 修改後需手動執行一次重新產生 `data/最常見詩詞字排名.js`。

#### 6. 初始隨機化

```
初始欄位順序：隨機打散，所有棒打亂
初始垂直位移：每根棒隨機；謎棒保證目標字「不在」中央列
預置棒（隨機 fix）：currentCol = puzzleColIdx（正確欄位）
                  offset = MIDDLE_ROW - targetIdx（中央列字 = targetChar）
                  isFixed = true
```

#### 7. 勝利判定（參照 game9）

```js
checkWin: function () {
    const candidate = [];
    for (let col = 0; col < this.puzzleLength; col++) {
        const bar = this.bars.find(b => b.currentCol === col);
        const idx = MIDDLE_ROW - bar.offset;
        candidate.push(bar.charString[idx]);
    }
    return candidate.join('') === this.puzzleText;
}
```

只看中央列實際顯示的字，不判斷棒 id 或位移數值。

### 7. 提示行（hintStep）規格

| 模式 | 行為 | 使用的參數 |
|------|------|----------|
| **`'never'`** | 完全不顯示，提示行恆顯示「（無提示）」 | 忽略 `hintDelaySec`、`stepInterval` |
| **`'instant'`** | 新局立即全部顯示 | 忽略 `hintDelaySec`、`stepInterval` |
| **`'fullDelayed'`** | 等待 `hintDelaySec` 秒後一次全顯 | 使用 `hintDelaySec`；忽略 `stepInterval` |
| **`'progressive'`** | 等待 `hintDelaySec` 秒後，每 `stepInterval` 秒揭一字 | 使用兩者 |

**實作關鍵**（避免上局排程殘留）：
- `hintDelayHandle` 把柄同步 `clearTimeout`
- `_hintSession` 隔離 ID：每次 `startHintReveal`/`stopGame` 遞增，舊回呼觸發時若 ID 不符即放棄
- `'never'` 進入分支立即 `return`，不會錯誤地啟動 progressive 排程

### 8. 難度參數的精密控制（現行 `difficultySettings`）

| 難度 | 時限 | 詩評下限 | 詩型 | 提示模式 | 提示延遲 | 提示間隔 | maxFixBarCount | maxDecoyCount | showRightChar |
|------|------|----------|------|---------|---------|---------|---------------|--------------|--------------|
| 小學   | 20 秒  | ★★★★★★ | 五言 | instant | 0  | — | 3 | 0 | **true** |
| 中學   | 40 秒  | ★★★★★ | 五言 | progressive | 20 | 3 秒 | 2 | 1 | false |
| 高中   | 60 秒  | ★★★★ | 七言 | progressive | 40 | 6 秒 | 2 | 2 | false |
| 大學   | 80 秒  | ★★★ | 七言 | **never** | — | — | 1 | 3 | false |
| 研究所 | 100 秒 | ★★★ | 七言 | **never** | — | — | 0 | 3 | false |

⚠️ **poemType 規則**：
- 小學 = `'五言'` 固定使用 5 字謎題
- 其他難度（包含難度設定值為 `'七言'` 者）= **每局隨機抽 5 或 7**
- 直棒長度不受此限，五言七言皆可作為直棒

⚠️ **maxFixBarCount 規則**：
- 是「**上限**」而非固定值
- `> 0` 時每局實際數量於 `[1, maxFixBarCount]` 範圍隨機（至少保證 1 根）
- `= 0` 時保持 0
- 重來時必須帶 `randomizeBars(true)` 重新指派

⚠️ **showRightChar**：僅小學開啟。其他難度完全不在中央列字上著色，避免讓難度變得太低。

### 9. 計分設定（scoreManager.gameSettings.game21）

```
{ base: 100, heart: 0, time: 1, getPointA: 0 }
```

- `base = 100`：基礎通關分
- `heart = 0`：本遊戲無紅心機制
- `time = 1`：每秒剩餘 +1 分（時限長故每秒分數較低）
- `getPointA = 0`：單題決勝，無連續得分

## 第二部分：遊戲心得與分析 (Game Experience & Analysis)

### 1. 遊戲性分析

《橫批成詩》是花月系列中操作維度最獨特的一款作品——所有其他遊戲的核心動作是「點擊選擇」，而本作的核心動作是「二維拖曳排列」，是真正意義上的空間拼圖。

心理狀態的三階段演變：

**第一階段：有序的混沌**——遊戲剛開始，竹簡排列雜亂。提示行的字元正在緩緩浮現，玩家的大腦開始掃描：「哪根棒裡有個『前』字？」、「那個『月』埋在哪首詩的第幾行？」

**第二階段：掌控的快感**——當玩家理解了斜向拖曳的操作邏輯，每一個手勢都同時完成「換位」與「滑動」。每次將一根棒推到正確位置，金色閃光給予即時的正向回饋。

**第三階段：完成的儀式感**——最後一根棒對齊，黃框字元由左到右依序亮起，旋律從零散的音符凝聚為完整的五聲調式——這個「竹簡排好、橫批完成」的視聽體驗，讓玩家真實感受到「我親手重建了這句詩」。

**頂點時刻**：在大學/研究所難度，七言謎題零提示、無 fix 棒輔助，玩家憑藉對整個詩庫的記憶，以流暢的斜向拖曳逐一定位每根棒，最後一根棒滑入到位的瞬間，旋律爆發、金光炸裂。

### 2. 精華推廣亮點

- **「零提示七言速通」的視覺震撼**：研究所玩家以流暢手勢迅速就位的畫面，是天然的短影音素材。
- **「斜向拖曳神技」的炫技感**：一般玩家分兩步操作；高手玩家斜向一刷，換位和滑動同時完成。操作差異在錄影中一眼可見。
- **「哪根棒藏著那個字」的文學偵探感**：高度類似文學研究中的索引查詢，對中文系學生有強烈共鳴。

### 3. 對外推廣短語提案

> *「8 根竹簡橫陳眼前，每根藏著你要找的那個字——不是選，不是填，是親手推動，讓那句千年前的詩，在黃金橫批裡一字一字地浮現。」*
> *「『月』這個字，在數十首詩裡都出現了——但謎題要的那個『月』，只在其中一首的第五個字位置。你找得到嗎？」*

### 4. 開發實作紀錄與已知優化

開發過程中累積的關鍵修正：

1. **棋盤從 7×7 改為 8×13**：使用者規範要求；同時直棒可超出邊界、不再 `overflow: hidden`。
2. **直棒只用原始詩句字元**：移除 `randomFillerChar`、移除截斷邏輯；長度自然不一。
3. **`fixBarCount` → `maxFixBarCount`**：改為上限隨機，且 > 0 時至少保證 1 根。
4. **左右拖曳跳過 fix bar**：早期版本被 fix bar 卡死，現改為「跳躍式互換」演算法。
5. **重來重新指派 fix bar**：早期版本 `retryGame` 漏帶 `assignFixed`，導致 fix bar 全被解開。
6. **勝利凍結所有棒**：早期版本只設 `isActive=false`，但進行中的拖曳仍會繼續；現改為三層凍結。
7. **句評分篩選**：早期只篩詩級評分，造成好詩中的弱句被選為題目；現同步篩 `line_ratings[i]`。
8. **提示行修正**：早期 `setTimeout` 排程會跨局觸發，造成「研究所卻顯示提示」；現以 `_hintSession` ID 隔離。
9. **showRightChar 限定小學**：早期所有難度的中央列字都會上色，難度被破壞；現僅小學開啟。
10. **干擾棒偽裝**：早期干擾棒有紅色背景可分辨；現外觀與一般棒完全相同。
11. **黃框改為金邊透明**：早期半透明金底會遮蔽字元；現只剩上下金邊。
12. **共用函式上移**：`getSharedBarLine`、`countIsolatedChars` 等已遷至 `script.js`，供其他遊戲共用。

### 5. 總結

《橫批成詩》以「二維自由拖曳」的空間互動革命，為花月系列開創了前所未有的玩法維度。它不再問「這個字對不對」，而是問「你能在詩庫的叢林中找到它、定位它、親手把它推入黃金橫批的正確位置嗎？」——這個過程所要求的詩詞知識廣度，是整個系列最高的。

從小學的 3 棒預置輔助（含 `showRightChar` 提示）到研究所 7 棒零提示的極限，難度的質變點清晰而殘酷；斜向拖曳的操作設計讓熟練玩家能展示出行雲流水般的「書法手感」；而每次全解瞬間的五聲旋律爆發，則讓「排好竹簡」這個行為上升為一場古典美學的儀式完成。
