# 《花月》詩詞珠簾（原名：謫仙人）- 企劃書與開發心得

> **定位說明**：本頁不是遊戲，是一頁**無計分、無勝負**的純舒壓互動頁面。程式檔名與內部物件仍沿用開發初期的代號 `zhexianren.js` / `window.ZheXianRen`（謫仙人，取李白「謫仙人」典故），選單顯示名稱後續調整為更直白的「詩詞珠簾」，兩者指同一功能，開發文件與程式碼中會交錯出現，特此說明。本頁是花月「舒壓頁」品類的開創之作，稍晚推出的 `suiyuean.js`（隨遇而安）在架構與慣例上直接承襲本頁的設計決策（詳見 `note/suiyuean_隨遇而安_企劃書與遊戲心得.md` 開頭說明）。
>
> 本企劃書省略遊戲企劃書規範中專屬於「勝負判定」與「難度分級計分」的章節，改以「物理系統設計」與「文位驅動的成長感」為核心章節。

## 第一部分：企劃書 (Design Document)

### 1. 頁面概述與核心目的

**頁面名稱**：詩詞珠簾（原名：謫仙人）(Beaded Poem Curtain / Verse Reel Curtain)
**頁面目標**：畫面上方懸掛「謫仙人」匾額，匾額下方垂掛數條由詩句串成的珠簾。玩家以手指或滑鼠撥弄珠簾，珠子如串珠般上下相連擺動，並可雙指縮放/平移整個畫面近看細讀或退遠欣賞整片珠簾。
**設計目的**：花月的積分與文位系統本質上是一套持續累積的長期成長曲線，但多數頁面只在「當下這一局」呈現分數，玩家很難直觀感受到「我這幾個月的努力堆疊起來是什麼樣子」。詩詞珠簾把玩家的**文位（書僮～大儒）直接轉譯成畫面上垂掛的珠簾條數**——文位越高，珠簾越茂密——讓玩家一打開頁面就能用眼睛「看見」自己的長期成長，同時提供一處完全不計分、只需要撥弄珠子聽風看詩的純粹放鬆空間。

### 2. 互動流程與核心操作

#### A. 使用流程

1. **進入頁面** → 系統讀取玩家目前文位（`_effectiveRankName()`），依文位決定珠簾條數（書僮＝1 條 … 大儒＝15 條，見 §5.2），並隨機組合詩句建構每條珠簾（見 §5.1）
2. **懸掛呈現** → 匾額顯示玩家文位文字，下方珠簾各自因重力自然垂下，呈現初始靜止（實際上受微風持續施力，見 §5.3）狀態
3. **撥弄珠簾** → 玩家以手指/滑鼠拖曳畫面上的珠子，被撥動的珠子依 Verlet 繩索物理帶動同一條珠簾內上下相連的其他珠子擺動；左右不同條珠簾彼此完全獨立，互不碰撞、互不干擾
4. **自然微風** → 未被撥動時，珠簾仍會因持續施加的微風力道自然彎成 S 形曲線並輕輕擺盪，而非死板地垂直靜止
5. **不定時橫掃陣風** → 每隔 4～10 秒隨機觸發一次橫掃全畫面的陣風，風鋒在 0.5～2 秒內平滑掃過整個畫面（由左至右或右至左），沿途珠簾依序被吹動而非同時瞬間抖動，模擬真實穿堂風吹過一整排珠簾的效果
6. **雙指縮放/平移** → 支援手勢縮放（0.5x～3x）與雙指平移，方便近看細讀珠上文字或退遠欣賞整片珠簾的懸掛景象
7. **持續開放** → 沒有結束條件，玩家可以隨時關閉頁面離開，下次進入會重新隨機抽詩、重新組簾

**沒有勝利也沒有失敗**：與隨遇而安相同，本頁全程不設任何目標或結算，玩家撥弄珠簾純粹是為了那份手感與視覺樂趣。

#### B. 操作方式

| 平台 | 撥弄珠簾 | 縮放/平移 |
|------|---------|----------|
| **桌機（滑鼠）** | `mousedown` + 拖曳 | 滾輪 `wheel` 縮放（輔助操作） |
| **手機（觸控）** | 單指 `touchmove` 拖曳 | 雙指 pinch 縮放、雙指平移 |

**手機適配挑戰（矛盾點與解法）**：

**矛盾一：垂直撥弄手勢與瀏覽器原生捲頁手勢衝突**
- 撥弄珠簾需要垂直方向的拖曳手勢，這與瀏覽器原生「上下滑動捲頁」手勢完全相同
- 解法：畫布容器套用 `touch-action: none`，並在 `touchmove` 事件中呼叫 `event.preventDefault()` 完全接管垂直手勢，由遊戲自行計算位移量並映射為珠子推力

**矛盾二：單指撥弄與雙指縮放的手勢辨識**
- 觸控裝置常見誤觸——玩家想縮放卻不小心先用單指觸發撥弄，或反之
- 解法：`touchstart` 判斷觸控點數量，`e.touches.length === 2` 才進入縮放/平移模式（記錄雙指初始距離與中點），`=== 1` 才進入撥弄模式；`touchend` 時偵測剩餘觸控點數量做狀態切換（雙指→單指時重新起算撥弄基準，避免手勢交接瞬間跳動）

#### C. 即時反饋機制

- **撥弄**：手指移動方向直接位移半徑（`FLICK_RADIUS=44`）內的珠子座標，力道隨距離遞減（`FLICK_STRENGTH=0.85`），下一幀 Verlet 積分自動由位置差推算出速度，讓珠子自然帶著擺動慣性甩開
- **手勢提示淡出**：首次進入頁面顯示「手指撥弄珠簾．雙指縮放畫面」提示文字，6 秒後自動淡出；玩家一旦開始撥弄（`_wake()`）也會立即觸發淡出，不需要等滿 6 秒

#### D. 沒有勝負標準

本頁與隨遇而安同屬「零失敗結局」設計，全程沒有任何目標達成或未達成的判定。

#### E. 觀看/互動的建議節奏

- **近看細讀**：雙指放大到珠簾懸掛的局部，逐字閱讀珠上詩句，體驗如同手持一串真實的木珠詩簾
- **退遠欣賞**：縮小到能一次看見整排珠簾隨風擺盪的全景，感受文位越高、珠簾越茂密的視覺成就感

### 3. 介面設計與使用者體驗

#### A. 主介面佈局（由上到下）

```
┌─────────────────────────────────────┐
│              謫仙人／〈文位〉         │  ← 木質金框匾額（畫面上方橫樑）
│         │      │      │      │      │
│         珠      珠     珠      珠      │
│         │      │      │      │      │
│         珠      珠     珠      珠      │  ← 依文位決定條數（1～15 條）
│         │      │      │      │      │     的懸掛珠簾，各自獨立擺動
│         珠      珠     珠      珠      │
│         │      │      │      │      │
│  手指撥弄珠簾．雙指縮放畫面（6秒淡出）  │  ← 底部手勢提示
└─────────────────────────────────────┘
```

- 整頁為單一 `<canvas>` 全螢幕繪製，匾額與所有珠簾皆由 Canvas 2D 逐幀繪製（非 DOM 節點），效能考量與 §3-A 隨遇而安相同
- 右上角固定關閉鍵、底部手勢提示為浮動 DOM 元素，疊加於畫布之上，且不隨相機縮放/平移而變形

#### B. 視覺回饋系統

| 元素 | 視覺表現 |
|------|---------|
| 背景 | 暖米色宣紙徑向漸層（`hsl(42,34%,90%)` 中心 → `hsl(38,30%,80%)` 邊緣） |
| 珠簾絲線 | 極細半透明墨線（`hsla(30,25%,30%,0.25)`），連接同條珠子 |
| 珠上文字 | 依條序交錯的深淺墨色（`hsl(28,22%,18~30%)`），奇偶條明度略有差異 |
| 匾額 | 深木紋漸層底 + 雙層金框，文字為金色（`hsl(45,82%,62%)`）並帶金色光暈，文字依字數自動縮放字級以免超出匾額寬度 |
| 鼓勵珠簾（低文位限定） | 金褐色（`hsl(32,60%,34%)`），內容為固定的鼓勵語，與詩句珠簾並排懸掛提醒玩家提升文位 |

#### C. 匾額文字：依文位自動縮放字級

```javascript
let fontPx = 48;
ctx.font = `bold ${fontPx}px 'Noto Serif TC', serif`;
while (ctx.measureText(rank).width > maxTextW && fontPx > 20) {
    fontPx -= 2;
    ctx.font = `bold ${fontPx}px 'Noto Serif TC', serif`;
}
```
文位名稱長度不一（如「書僮」2 字 vs.「府案首」3 字），自動縮字確保文字永遠完整顯示在匾額寬度內，不會溢出或被裁切。

#### D. 使用體驗的流暢度設計

- **各條珠簾獨立隨機物理參數**：每條珠簾各自在 `[GRAVITY_MIN, GRAVITY_MAX]` 與 `[DAMPING_MIN, DAMPING_MAX]` 區間隨機取值，讓多條珠簾的擺動節奏彼此略有差異，避免所有珠簾像同一個模板複製貼上般同步晃動，更接近真實懸掛物「各自有各自的重量與慣性」的觀感
- **限制單幀最大位移**：`MAX_STEP=26` 防止撥弄力道過猛或縮放跳動時珠子瞬間爆開甩飛，維持視覺上的物理合理性
- **雙指手勢交接不斷點**：雙指變單指時重新起算撥弄基準座標，避免手勢切換瞬間因基準座標未更新而讓珠子產生不合理的瞬間跳動

### 4. 機制的繼承與創新

**繼承**：整體「無計分舒壓頁」的頁面定位、Canvas 全螢幕繪製、touch-action 接管手勢等基礎慣例，後續被 `suiyuean.js`（隨遇而安）直接沿用並發展出不同的物理玩法。

**創新設計決策**：

**⓵ 「文位轉譯為珠簾條數」讓長期積分成長變得可視**：這是本頁最核心的設計巧思——`_rankColumnCount()` 直接把玩家的文位映射為 1～15 條珠簾，讓原本只存在於數字與排行榜的「積分成長」轉化成一眼可辨的視覺豐富度。玩家不需要打開任何統計頁面，光是看珠簾的疏密就能感受到自己的長期投入累積了多少。

**⓶ Verlet 繩索物理模擬取代簡單的擺動動畫**：珠簾若用預錄的擺動動畫（如 CSS keyframes）會顯得死板重複；改用真實的 Verlet 積分 + 繩段約束模擬，每次撥弄的力道、方向、位置都會產生獨一無二的擺動軌跡，且多幀迭代後自然收斂回穩定狀態，物理真實感是動畫慣例無法達到的。

**⓷ 「行進波微風」而非「整條同步搖擺」**：微風力道不僅隨深度（越靠珠簾末端）遞增，還額外帶有沿珠子往下的相位差（`i * BREEZE_WAVE_K`），讓珠簾呈現 S 形曲線的行進波動，而非整條同時同向搖擺——這個細節讓珠簾看起來像是「風正在穿過它」而非「它自己在抖動」，是決定整體真實感的關鍵設計。

**⓸ 「橫掃陣風」模擬風吹過一整排珠簾的先後感**：若所有珠簾同時受到相同陣風力道，會顯得虛假（真實的風吹過一排物體有先後之分）。`_gustForceForColumn()` 依珠簾在畫面上的 X 座標與當前風鋒位置的距離計算力道（餘弦鐘形曲線，進出平滑），讓陣風呈現「由左至右（或反向）依序吹過」的真實層次感。

**⓹ 低文位玩家的隱性引導**：文位在「童生」（含）以下時，畫面左右兩側各加掛一條寫著「提升文位可以增加串珠的數量，加油吧！」的鼓勵珠簾——不使用彈窗或強制訊息打斷體驗，而是以「和其他珠簾一樣可以撥弄」的方式融入畫面本身，讓引導本身也具備舒壓頁該有的柔軟語氣。

### 5. 核心物理與視覺設計

#### 5.1 珠簾內容：同首詩連續詩句串成

```javascript
_buildPoemString: function () {
    // 隨機抽一首詩，從隨機起始行開始，以「句間逗號、句末句號」串接，
    // 直到達到 MIN_BEADS(10)~MAX_BEADS(32) 字數區間（40% 機率提前收尾，製造長短變化）
}
```

每條珠簾皆由**同一首詩**的連續詩句串成（句間「，」、句末「。」），長度 10～32 字不等，且刻意保留 40% 機率提前收尾，讓不同珠簾的長度自然參差不齊，視覺上更接近真實懸掛的珠簾長短交錯。

#### 5.2 珠簾條數：依文位動態決定

```javascript
_rankColumnCount: function () {
    return Math.max(1, Math.min(15, this._rankIndex() + 1));
}
```

文位在 `ScoreManager.ranks` 中的索引（書僮=0 … 大儒=14）直接 +1 對應珠簾條數，形成 1～15 條的線性映射，是 §4 創新⓵「文位可視化」的具體實作。

#### 5.3 Verlet 物理與行進波微風

```javascript
// 深度比：越接近末端越大；行進波相位隨珠往下遞移
const depth = n > 2 ? (i / (n - 1)) : 1;
const wave = Math.sin(time * BREEZE_SPEED + col.breezePhase + i * BREEZE_WAVE_K);
const wind = BREEZE_AMP * col.breezeAmp * wave * Math.pow(depth, BREEZE_DEPTH_POW);

b.px = b.x; b.py = b.y;
b.x += vx + wind + gust; // 微風 + 橫掃陣風（皆末端最強）
b.y += vy + gravity;     // 重力
```

每顆珠子的座標更新皆為「Verlet 速度（位置差）+ 微風 + 陣風」三者疊加，再透過 `CONSTRAINT_ITERS=3` 次繩段長度約束迭代，把每兩顆相鄰珠子的距離拉回固定的 `SEG_LEN=23`，模擬繩索的張力。

#### 5.4 橫掃陣風的風鋒位置計算

```javascript
_gustForceForColumn: function (anchorX, time) {
    const p = (time - g.start) / g.duration;   // 進度 0~1
    const from = g.dir > 0 ? -GUST_BAND : STAGE_W + GUST_BAND;
    const to = g.dir > 0 ? STAGE_W + GUST_BAND : -GUST_BAND;
    const frontX = from + (to - from) * p;      // 風鋒 X 座標隨時間線性移動

    const dist = Math.abs(anchorX - frontX);
    if (dist > GUST_BAND) return 0;
    const intensity = 0.5 * (1 + Math.cos(Math.PI * dist / GUST_BAND)); // 餘弦鐘形，進出平滑
    return g.dir * g.strength * GUST_FORCE_SCALE * intensity;
}
```

每條珠簾此幀受到的陣風力道，取決於自己的 X 座標與當前風鋒位置的距離——距離越近力道越強，超出 `GUST_BAND=72` 範圍則完全不受影響，形成「風正掃過這裡」的局部效果而非全畫面瞬間齊發。

#### 5.5 雙指縮放：保持觸控錨點不飄移

```javascript
_setZoomAt: function (target, sx, sy) {
    // 保持螢幕點 (sx,sy) 對應的世界座標不動
    const wx = (sx - this.panX) / this.zoom;
    const wy = (sy - this.panY) / this.zoom;
    this.zoom = target;
    this.panX = sx - wx * this.zoom;
    this.panY = sy - wy * this.zoom;
}
```

縮放時以雙指中點（或滾輪游標位置）為錨點反推平移量，確保縮放過程中該點在畫面上的位置不飄移，是所有支援 pinch 縮放的介面都必須處理的標準技巧。

### 6. 音樂與音效設計

- **背景音樂**：無持續播放背景樂，維持與隨遇而安一致的「安靜舒壓」基調
- **操作音效**：關閉頁面時播放 `SoundManager.playCloseItem()`，作為離開這處空間的輕柔提示音；撥弄珠簾本身無額外音效，保留純視覺與觸感的沉浸體驗，不用聲音打斷「靜靜看詩」的節奏
- **聲音克制的設計理由**：與隨遇而安會在每次碰撞給予音效回饋不同，詩詞珠簾刻意讓撥弄動作保持安靜——因為珠簾的樂趣核心在於「手感」與「視覺流動」，額外的音效反而會把注意力從指尖的觸控回饋拉開

## 第二部分：開發心得與分析

### 1. 頁面性分析：「這是一串會隨風擺動、也會隨著你的成長越掛越滿的詩簾」

詩詞珠簾最獨特之處，在於它把「玩家的長期投入」這種通常只存在於後台資料庫的抽象數字，轉化成了一種可以親手觸摸、親眼看見疏密變化的實體隱喻——珠簾。書僮階段只有孤零零一條珠簾垂掛，大儒階段則是十五條珠簾層層疊疊、隨風搖曳，玩家不需要閱讀任何排行榜或統計圖表，光是「珠簾變多了」這件事本身，就是一種比數字更有溫度的成就回饋。

這種設計對玩家心理的價值，在於它把「持續遊玩花月」這件事的回饋週期從「每一局」延伸到「每一次登入」——即使今天沒有特別想玩任何一款遊戲，光是打開詩詞珠簾看看自己現在掛了幾條珠簾、聽風把它們吹得沙沙作響，也是一種對長期投入的溫柔確認。

**頂點時刻**：不在於任何特定操作，而在於某次不經意打開頁面時，發現自己的文位又升了一級，珠簾比上次多掛了一條——那種「原來我不知不覺已經走了這麼遠」的驚喜與踏實感，是花月所有頁面中最貼近「時間的重量」的一次體驗設計。

### 2. 精華推廣亮點

- **「文位可視化」的長期黏著素材**：把積分成長轉化成珠簾條數的設計，天然適合做成「我的文位進度」的分享截圖，玩家可以驕傲地展示自己掛了幾條珠簾，形成社群間隱性的成長競賽。
  標題可以下：**「從一條珠簾到十五條——這是我在花月裡走過的每一步」**

- **「行進波微風」的實況美術展示**：珠簾受風吹拂呈現 S 形曲線行進波、橫掃陣風依序吹過畫面的細節，是極佳的技術展示素材，適合吸引對前端動畫/物理模擬有興趣的觀眾。
  標題可以下：**「這陣風，是真的『吹過』每一條珠簾，不是同時抖動的假動畫」**

### 3. 對外推廣短語提案

> *「你的每一分積分，都變成了一條真實垂掛的珠簾——回來看看，它現在有多茂密。」*

> *「小心，這陣風不是特效，是物理——每一顆珠子的擺動，都由你剛才撥弄的力道決定。」*

> *「十五條珠簾，十五個文位，這是花月寫給每一位長期陪伴者最溫柔的一份禮物。」*

### 4. 總結

《詩詞珠簾》（謫仙人）用一個看似簡單的視覺隱喻——珠簾條數對應文位——把花月的長期積分系統從抽象數字轉化成了可觸摸、可撥弄、隨風搖曳的實體畫面，是花月生態中「成長感可視化」設計的代表作。技術上，Verlet 繩索物理、行進波微風、橫掃陣風、雙指縮放錨點計算，每一項都是為了讓這串珠簾「感覺起來是真的」而非動畫特效；體驗上，本頁與稍晚推出的隨遇而安共同開創了花月「無計分舒壓頁」這個全新品類，證明了花月不只能教人背詩，也能給人一處安靜看詩、輕輕撥弄光陰的角落。
