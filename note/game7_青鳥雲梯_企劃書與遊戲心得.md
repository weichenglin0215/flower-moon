# 《花月》Game7: 青鳥雲梯 - 遊戲企劃書與遊戲心得

## 第一部分：遊戲企劃書 (Game Design Document)

### 1. 遊戲概述與核心目的
**遊戲名稱**：青鳥雲梯 (Flappy Poetry)
**遊戲目標**：玩家化身為一隻振翅高飛的青鳥，必須精準跳躍，並依照順序降落在由唐詩宋詞字元化作的「漂浮平台」上，直到成功著陸於最後一塊黃金平台。
**學習目的**：透過極具節奏感與微操作的橫向卷軸動作遊戲 (Platformer)，將古典文學從「平面閱讀」化為一條充滿高低起伏的「天空之路」，讓玩家在不斷挑戰落點的過程中，自然記憶冗長詩句的次序。

### 2. 遊戲機制的繼承與創新 (Game Mechanics)
本作參考了《Flappy Bird》與《Doodle Jump》的物理慣性，並重新構築出適合教育遊戲的專屬玩法：
- **物理拋物線跳躍**：青鳥受到重力 (`gravity`) 影響不斷下墜，點擊螢幕會給予向上的跳躍力 (`jumpForce`)。玩家必須精準控制拋物線的落點。
- **頂部邊緣碰撞 (Top-Edge Collision)**：為了提升操作的精細度，青鳥的碰撞判定被嚴格限制在方塊的「頂部邊界 (白色著地條)」。如果玩家從下方或側邊撞上方塊，將無法產生著陸判定，而是直接擦身而過導致墜落。
- **錯過懲罰 (Skip Penalty)**：若玩家飛過頭，錯過了某個字元方塊，該方塊會瞬間失去法力（變成深灰色並傾斜 30 度成為 `Ghost` 狀態），同時扣除玩家一顆紅心生命。
- **佛心的「存檔點系統 (Checkpoint)」**：考慮到高等難度下詩詞可能長達 56 個字，為了避免因為最後一個字的失誤而引發巨大挫折感，我們加入了極度貼心的 Checkpoint 機制！當青鳥墜崖死亡時，系統會自動搜尋「最後一個成功降落的正解平台」，讓青鳥在該處滿血復活，並將前方錯過的所有方塊重置，讓玩家能接續挑戰。

### 3. 動態難度與空間參數設計 (Difficulty Scaling)
為了讓「手殘黨」與「動作遊戲達人」都能獲得樂趣，本遊戲對參數的控制幾乎達到了苛刻的程度：
- **小學 / 中學 (安全島模式)**：
  - **靜止著陸 (`stopOnLand: true`)**：這項設定是給新手的最大恩賜。當青鳥落到方塊上時，整個畫面的橫向卷軸會「完全靜止」！玩家可以在方塊上待到天荒地老，慢慢尋找下一個目標字在哪裡。方塊極為寬大 (`size: 90`) 且距離極近。
- **高中以上 (無情推進模式)**：
  - **滑行地獄 (`stopOnLand: false`)**：從高中開始，降落在方塊上依然會得分，**但橫向物理卷軸不會停止！** 這意味著青鳥會在極小的方塊頂端繼續往前滑行，玩家如果不立刻起跳，半秒後就會從方塊邊緣「滑出墜崖」！
- **大學 / 研究所 (動態浮島與極限微操)**：
  - 除了沒有安全島，方塊寬度被縮減至極限的 50px，高低落差拉大 (`heightVar: 600`)，最高難度下還要挑戰 56 個字元的長篇古文。更可怕的是，加入了 **「動態浮島 (`moving`)」**：平台會以未知的速率上下震盪，玩家必須在滑行的瞬間預判下一個震盪點的交會處起跳。

### 4. 畫面與 UI 設計
- **天色漸變系統**：隨著玩家在長篇詩句的進度推進 (`progress = charIndex / poemChars.length`)，遊戲背景的 CSS 會從清晨 (`morning`) 漸變至午後 (`afternoon`)，最終在抵達終點前轉為傍晚 (`evening`)。這提供了一種「長途跋涉」的絕美史詩感。
- **環繞 SVG 計時器**：計時器化作圍繞整個遊戲畫面的外框線 (`game7-timer-path`)，以逆時針方向將螢幕逐漸「吃掉」，提醒玩家不能在 Checkpoint 拖延太久。

## 第二部分：遊戲心得與分析 (Game Experience & Analysis)

### 1. 遊戲性分析：「背詩也能體會到心臟漏跳一拍的感覺」
《青鳥雲梯》打破了傳統教學遊戲「靜止、回合制」的通病。
這款遊戲最迷人的地方在於其「物理慣性」帶來的節奏感。青鳥那微小的扇翅聲與重力下墜的撕裂感，配合玩家點擊螢幕的頻率，會自然形成一種鼓點般的節奏。當你連續五次精準踏上面積窄小、高低落差極大的字元方塊時，每一次響起的「咚 (Don Sound)」音效與字體的點亮，都會帶來強烈的「跑酷 (Parkour)」爽快感。背詩不再是用眼睛看，而是用手指的肌肉記憶「跳」出來的。

### 2. 精華推廣亮點：足以吸引動作玩家的高水準設計
在對外宣傳時，這款遊戲的許多隱藏機制，具備著讓動作玩家 (Gamers) 眼睛一亮的深度：

- **硬核的「滑冰地獄」與「節奏大師」玩法**：
  在社群推廣時，可以強烈對比「小學」與「研究所」的遊玩畫面。
  新手以為這是一款溫韾的跳躍遊戲，可以停在方塊上慢慢看風景；但在研究所難度下，這遊戲會瞬間變成《超級瑪利歐》最難的「全自動毒水關」。因為**「降落不等於安全」**！方塊成了塗滿冰塊的溜冰鞋，你點擊跳躍的容錯時間被壓榨到剩不到 0.3 秒，配合上下浮動的機關平台，這種類似《幾何衝刺 (Geometry Dash)》的極限連續反應測試，絕對能激起重度玩家的挑戰慾望。

- **佛心卻狡猾的「存檔點 (Checkpoint) 復活」**：
  在實況中，這個機制經常能引發戲劇性的笑料。當實況主好不容易跳過了 30 個字，卻手滑墜崖時，正準備崩潰大叫，赫然發現自己居然在第 30 個方塊上滿血復活！忍不住大讚遊戲設計師的溫柔。
  **但等等！** 復活後計時器並不會重置！如果在前面的難關摔死太多次、浪費太多時間原地復活，玩家最終會在即將跳上最後一塊黃金平台的那一剎那，因為「總時間歸零」而在空中無情暴斃。這種「給你希望又讓你絕望」的時間管理壓力，是本作極具話題性的亮點。

- **絕美的視覺進度反饋（從清晨飛到黃昏）**：
  本作隱藏了極具巧思的環境敘事。玩家操控的不過是一隻幾何拼湊的青鳥，但由於《花月》的詩詞往往蘊含意境，當玩家在大學難度下，歷經千辛萬苦跳躍了 40 個方塊，原本明亮的清晨背景悄然染上暮色 (`evening`)——這不僅是視覺上的防視疲勞設計，更營造出一種「為求一字，飛越了千山萬水，自晨至昏」的浪漫文學意象。

### 3. 對外推廣短語提案
> *「你以為這是快樂的 Flappy Bird，直到你發現降落的平台會滑動，而且下一個字的平台在半個螢幕外上下震盪。」*
> *「唯一一款讓你因為踩碎了『床前明月光』而墜入深淵的硬核動作跑酷遊戲。」*
> *「從清晨飛躍到黃昏，每一次精準的踩踏，都是在天空寫下一句千古絕唱。」* 

### 4. 總結
《青鳥雲梯》將背誦詩詞的靜態記憶，轉換成了橫向卷軸的動態反應。它的成功不僅在於套用了經典的跳躍物理機制，更在於它極其細膩的「難度分層」：用「靜止安全島」擁抱新手，用「無情滑行與動態平台」折磨硬核玩家；並且透過溫柔的 Checkpoint 機制包容失誤。這是一款在動作爽快感與視覺抒情上，平衡得近乎完美的典範之作。
