# 《花月》Game19: 詩碟狂襲 - 遊戲企劃書與遊戲心得

## 第一部分：遊戲企劃書 (Game Design Document)

---

### 0. 原型遊戲分析

#### A. 原型遊戲玩法解析

**太空侵略者（Space Invaders，1978，Taito）**：玩家操控畫面底部的砲台左右移動，射擊由上方緩慢步進逼近的外星敵陣。敵陣以規則的矩陣排列，左右集體移動，碰到邊界後整體下移並加速，直到壓入玩家陣線或玩家血量歸零為止。核心爽點在於「清場前的壓迫感」——敵陣越稀疏，移動越快，反而在快要通關的瞬間製造出最高張力。操作方式：桌機鍵盤左右鍵移動、空白鍵射擊；手機版多以左右滑動 + 點擊射擊實作。

《詩陣侵略》（game6）已在此原型的基礎上做了第一次詩詞化改造：以整排的詩句字元替代外星人陣列，「打完一整排即解鎖 Roguelite 獎勵」成為核心玩法循環。整排皆為正確字，玩家只要夠快夠準即可清排，知識門檻相對有限。

#### B. 原型的介面設計與吸引力分析

太空侵略者的介面以「充滿壓迫感的對稱敵陣 + 孤零零的底部砲台」構成最原始的視覺張力。密集的像素外星人在畫面上方緩慢蠕動，那種即將被壓頂的危機感讓玩家幾乎無法移開視線。四座碑林掩體則作為有限生命的最後緩衝——它們的存在既是保護，也是計時炸彈，最終都難逃粉碎的命運。

這個設計的吸引力核心在於**「資訊的不對稱壓縮」**：敵陣規模可見但不可控，玩家的子彈速度有限，死亡的逼近速度清晰可測——這種「知道自己快輸了卻又無法停手」的張力，正是它跨越半個世紀依然能俘獲玩家的根本理由。

花月《詩碟狂襲》保留了以下精髓：壓迫感（字陣緩緩下降）、孤立感（孤身砲台面對千軍萬馬）、以及驟然加速的臨界崩潰感。但在視覺上，敵陣從規整矩陣升級為**各行獨立游移的混沌字海**，進化出全新的心理壓迫維度。

#### C. 詩詞融合分析

| 問題 | 答案 |
|------|------|
| **什麼被替換成詩詞** | 外星人方陣被替換成詩句的每一個漢字；第一排只有 1 字，第二排 2 字，逐排遞增至最多 12 字，其中每排**只有一個正確字**，其餘皆為精心設計的混淆字 |
| **如何加入語文知識門檻** | 玩家必須知道這首詩當前輪到的是哪個字，才能從一排多達 12 個漢字中找出唯一正解；混淆字在語意上與正解高度相似，盲目亂射命中率極低 |
| **詩詞學習效果** | 強化**字序記憶**——玩家在彈幕壓力下必須逐字逐句精確提取「這一句的第幾個字到底是什麼」，是最接近臨陣默書的神經模擬 |

#### D. 手機觸控適配：矛盾點與解決方案

**矛盾一：精準點擊問題（最關鍵）**
- **原始操作**：桌機滑鼠精確點擊特定字元，不會誤觸左右相鄰字
- **觸控矛盾**：手機觸控面積約 10–20px，當一排有 10 個以上的字時，字與字間距極窄，誤觸概率極高
- **解決方案**：底部砲台改為**左右滑動追蹤**（touchmove 控制砲台 x 座標），射擊改為**自動對準最近字元並連發**，玩家只需控制方向而非精確點擊個別字元，解決手指觸碰不精準的根本矛盾

**矛盾二：炸彈躲避問題**
- **原始操作**：桌機滑鼠可以快速橫向位移躲開炸彈
- **觸控矛盾**：手機玩家觸控拖動時，另一隻手無法同時操作，難以做出即時的左右閃避反應
- **解決方案**：炸彈下落速度設計為「緩中帶急」節奏——下落初段較慢以提供反應時間，接近玩家時加速，並在小學至中學難度下增加炸彈視覺尺寸，確保手機玩家有足夠的視覺預警空間

**矛盾三：弧線特攻混淆字的視覺追蹤問題**
- **觸控矛盾**：手機畫面較小，弧線入場的特攻混淆字若移動速度過快，玩家眼球難以追蹤，手指也來不及反應
- **解決方案**：特攻混淆字入場時播放獨特的音效警示，並配合紅色光暈預告（入場前 1 秒），讓玩家在追蹤前就有心理準備，手機版特攻字的弧線半徑亦略大，確保視覺軌跡可跟隨

### 1. 遊戲概述與核心目的

**遊戲名稱**：詩碟狂襲 (Poetry Barrage)

**遊戲目標**：操控底部砲台左右移動，對著緩緩下降的詩詞字陣射擊。每一排字中只有一個正確字——必須識別當前詩句對應的字，在混淆字的干擾與炸彈的轟炸之間，逐字擊碎正確字、消滅整排，直到清空全陣。

**學習目的**：在《詩陣侵略》的基礎上進行**第二次進化**。舊版整排皆為正確字，玩家無需識別便能射擊；新版每排只藏一個正確字，其餘皆是精心布置的語文陷阱，玩家必須在砲火之間完成**「逐字精確提取」**的認知任務。這是對傳統背誦的最猛烈升級：以往背詩是靜態的眼球掃描，在《詩碟狂襲》中，它變成了在彈幕雨中眼疾手快的**臨陣默書**——知識就是武器，不懂詩的人在這裡只會空炮浪費時間。

### 2. 遊戲玩法與核心操作 (Gameplay & Controls)

#### A. 遊戲流程

1. **選擇難度** → 系統選取詩詞，生成字陣：第一排 1 個字（即首字，唯一正確字），第二排 2 個字，依此類推直到第 12 排有 12 個字（含 1 個正確字、11 個混淆字），字陣由上而下排列並緩緩下降
2. **辨認最底排** → 玩家只能射擊最底部那一排；字陣最底排以**明亮邊框**標示為可攻擊區域，其餘排視覺略暗以區別
3. **識別正確字** → 畫面頂部的「詩句提示欄」顯示當前詩句與已過的字數，玩家需判斷當前輪到哪個字，在最底排的混淆字中找出正解
4. **移動砲台射擊** → 玩家左右移動砲台瞄準目標字；滑鼠/觸控拖動砲台，點擊/放開射擊；子彈速度極快，幾乎瞬達
5. **擊中正確字** → 正確字開始膨脹成長（100%→最多 150%）、顏色由白漸紅；持續射擊直到正確字爆破，整排同時消失；解鎖三選一獎勵選擇
6. **擊中混淆字** → 白色小粒子散開，無傷害，不影響混淆字血量
7. **躲避炸彈** → 最底排混淆字不定時往下投擲紅色小炸彈；不定時出現特攻混淆字以弧線入場，靠近後噴射密集炸彈再飛離
8. **字陣清空** → 消滅最後一排後遊戲勝利
9. **失敗條件** → 計時器歸零 **或** 被炸彈命中超過生命值上限 **或** 字陣下降至底部壓線

**勝利條件**：清空所有排（或消滅至設定目標排數）

**失敗條件**：計時器歸零、生命值耗盡、或字陣下降至底部撞擊玩家砲塔

#### B. 操作方式

| 平台 | 移動砲台 | 射擊 | 暫停/繼續 |
|------|---------|------|---------|
| **桌機（滑鼠）** | 滑鼠左右移動 | 點擊左鍵（自動連發） | Esc 鍵 |
| **手機（觸控）** | 單指左右拖動 | 自動發射（追蹤最近正確排） | 點擊暫停鈕 |

**操作限制**：
- 砲台只能**左右移動**，不能前後移動
- 子彈為**垂直向上**飛行，不能橫向射擊
- 只有**最底排的字元**會被子彈命中並計算傷害；子彈穿越上方排時無效
- 每次只能發射一顆子彈（依升級可增加至散彈），自動追蹤砲台正上方的目標

**手機專屬說明**：觸控模式下砲台隨手指即時跟隨，放開手指後保持位置；射擊為自動連發，玩家只需專注移動即可。

#### C. 即時反饋機制

- **擊中正確字**：字元尺寸增大 10%（配合「充氣」動畫，上緣對齊保持穩定）、發出黃色粒子環繞特效、播放 `playHit()` 音效
- **正確字達到 150% 被擊毀**：爆炸粒子特效爆開，同排所有混淆字自動以白色消散特效一齊消失，播放 `playSuccess()` 音效；屏幕邊緣短暫金色光暈
- **擊中混淆字**：少許白色小粒子散開，播放靜音或極輕的碰撞音效，無傷害反饋
- **被炸彈命中**：紅心減少、畫面短暫紅色震盪、播放 `playFailure()` 音效
- **三選一獎勵觸發**：時間短暫凍結，獎勵選擇卡從畫面中央滑入

#### D. 勝利/失敗的標準

- **勝利**：在計時器歸零前清除所有設定的詩句排數，顯示通關動畫與分數結算
- **失敗**：被炸彈命中達到生命上限、計時器歸零、或字陣整體下降至底部壓線均觸發失敗畫面，顯示所完成的進度、正確字命中率，並提供「再試一次」選項
- **失敗後的處理**：呈現本局「已擊毀排數」與「正確字命中率」統計，讓玩家看見自己的進展，而非只看到失敗二字

#### E. 獲得高分的技巧

- **眼睛**：始終盯住詩句提示欄，提前一個字知道「接下來要打哪個字」，在新排出現前就做好識別準備
- **大腦**：快速在最底排中掃描字形——正確字往往藏在字形相似的群字之中，憑借對詩句的熟悉程度可瞬間鎖定
- **手指**：保持砲台在場地中央位置，可最快速度向兩側抵達任意字元的正上方
- **策略**：選擇三選一獎勵時，優先「穿透子彈」可讓正確字血量加速清除；優先「炸彈護盾」可延長在密集彈幕下的存活時間

### 3. 介面設計與使用者體驗 (UI/UX Design)

#### A. 主介面佈局

```
┌────────────────────────────────────────────────┐  ← 頂部資訊列（50px）

├────────────────────────────────────────────────┤  ← 詩句提示欄（60px）
│  「床前明月光，疑是地上霜」── 當前：第4字「月」   │    poem-hint-bar（捲動進度）
├────────────────────────────────────────────────┤
│  ┌──────────────────────────────────────────┐  │
│  │  ▒ ▒ 春 花 ▒ ▒ ▒ ▒ ▒ ▒ ▒ ▒（排12，最高）  │  │  ← 字陣區（各排獨立左右飄移）
│  │  ▒ ▒ ▒ ▒ ▒ ▒ ▒ ▒ ▒ ▒ ▒（排11）          │  │    正確字白色，混淆字灰色
│  │  …                                       │  │    各排移動速度、方向不同
│  │  光 床（排2，含1正確字「光」）             │  │
│  │═══════════════════════════════════════   │  │  ← 可攻擊排（金色底線/亮框）
│  │  霜（排1，首字）                          │  │    最底排特別亮顯
│  └──────────────────────────────────────────┘  │
│              ▲                                 │  ← 玩家砲台（可左右移動）
│        ███████████                            │
└──────────────────────────────────────────────┘  ← 紅色四方形計時條邊框(在遊戲區四周)
```

**靜態元素**：頂部資訊列、詩句提示欄框架、底部計時條框架

**動態元素**：字陣各排位置（獨立移動）、正確字尺寸（受擊膨脹）、炸彈軌跡、特攻混淆字弧線路徑、砲台位置、計時條剩餘長度

#### B. 視覺回饋系統

| 狀態 | 視覺表現 | 色彩 |
|------|---------|------|
| 普通混淆字 | 半透明灰底白字 | `hsla(220, 20%, 60%, 0.7)` |
| 正確字（未受擊） | 純白字、微金光暈 | `hsl(0, 0%, 100%)` 配金色邊框 |
| 正確字受擊中（10%–40%膨脹） | 字體放大、淡黃色 | `hsl(50, 90%, 75%)` |
| 正確字受擊重（50%–100%膨脹） | 字體更大、橘紅色 | `hsl(20, 90%, 60%)` |
| 正確字受擊致命（150%）| 極大、深紅色 → 爆炸 | `hsl(0, 90%, 50%)` |
| 混淆字受子彈碰觸 | 白色小粒子瞬閃 | `rgba(255,255,255,0.8)` |
| 可攻擊排（最底排）| 金色底線 + 輕微光暈 | `hsl(45, 80%, 55%)` |
| 被炸彈命中 | 全屏紅色震盪幀 | `rgba(220, 30, 30, 0.4)` |
| 通關正確字爆破 | Canvas 金色粒子爆炸 | 金色 + 白色碎屑 |

**花月配色哲學**：金橙系代表「正確且強勢的字」，深翠綠系代表「安全確認」，朱紅系代表「危險與混淆字投彈威脅」，三色形成清晰的語義邏輯，讓玩家不需額外學習即可直覺判讀場上狀態。

#### C. 計時器呈現方式

計時器採用**底部橫向耗盡長條**形式，從左到右以實心色塊表示剩餘時間。時間充裕時為深藍色，進入後半段（40% 以下）轉為橙色，進入危急狀態（20% 以下）轉為閃爍朱紅色，並觸發「節拍震盪」動畫（條高微幅上下跳動），讓玩家無需移開視線盯數字便能感知壓力。

#### D. 使用體驗的流暢度設計

- **最底排高亮標示**：遊戲最容易的誤解是「打錯排」——金色底線明確指示目前可攻擊的唯一排，上方排以視覺降暗（opacity 0.6）避免視覺干擾，防止新手誤以為上方字元也可射擊
- **正確字膨脹的上緣對齊**：字元受擊膨脹時以「上緣錨點」成長，避免字元膨脹後下移破壞視覺基準線，玩家追蹤目標時的視線不會因字元移動而偏移
- **挫折感管理**：失去生命後有 1.5 秒的短暫無敵幀（砲台閃爍），讓玩家不會在慌亂中被連續炸彈秒殺；失敗結算畫面優先顯示「你已打到第 X 排」而非直接呈現失敗標語

### 4. 遊戲機制的繼承與創新 (Game Mechanics)

本作繼承了**太空侵略者（Space Invaders）**的垂直壓迫射擊框架，以及《詩陣侵略》（game6）已驗證的「清排獲得 Roguelite 三選一獎勵」機制，但在幾乎每個核心機制上都進行了**顛覆性的重設計**：

---

**單字氣球成長系統（Per-Character HP Balloon Growth）**

game6 的每個字元只有固定血量，擊中即碎，缺乏積累感。《詩碟狂襲》引入了「氣球血量」機制：正確字的血量依照排數順序逐漸加多，最下面第一排只有一滴血，往上第二排兩滴血，第三排有三低血，依此類推，正確字在越高位置的血量越多越難以擊破。
正確字每次被子彈命中，尺寸放大 10%（視覺範圍：100% → 500%），顏色由純白 → 淡黃 → 橙色 → 深紅，像一顆被反覆充氣的氣球緩緩膨脹。這個設計有三重作用：第一，它讓玩家**清楚感受到傷害積累**，每一發子彈都看得見成果，射擊節奏從「盲目連發」升級為「看著氣球膨脹的期待感」；第二，字元膨脹後**視覺目標更大**，為玩家製造「快要成功了」的即時激勵；第三，在高難度下正確字血量（5次以上）較多，氣球必須充到極大才爆，那個臨爆前的大紅色漢字橫亙在排頭，視覺張力極強。

---

**混淆字炸彈攻擊（Decoy Bomb Barrage）**

game6 的敵方字元雖然有射擊，但發射邏輯與字元的「混淆字」屬性無關。《詩碟狂襲》將炸彈的攻擊權**專屬賦予混淆字**，且只有最底排的混淆字會發射。這個設計有深刻的教學意義：玩家若能快速識別正確字並連續命中，混淆字就無暇投彈；反之，玩家若遲疑、亂打，混淆字存活越久，投下的炸彈就越密集。**識字速度直接決定了炸彈密度**——這是一個把「語文知識反應速度」轉化為「防禦生存率」的精妙機制閉環。

---

**弧線特攻混淆字（Arc-Raider Decoys）**

場上偶爾出現 1–3 個脫離字陣的特殊混淆字，以優美的弧線從左方或右方入場，在畫面中段繞行後靠近砲台，投下一片密集的紅色小炸彈後再以弧線飛離。這個機制的創新在於**打破了「字陣是靜態威脅」的認知慣性**：玩家正全神貫注識別最底排時，一個從邊緣射入的弧線字會突然轉換戰場焦點，迫使玩家在「繼續攻擊正確字」與「緊急躲避特攻炸彈雨」之間做出即時判斷。這種「多重威脅同時並存」的設計，讓遊戲在視覺上充滿動感，在決策上充滿緊張，是整款遊戲最有「電光火石」感覺的時刻。

---

**梯形字陣規模（Triangular Formation Scaling）**

game6 的字陣從一開始就是矩形滿排。《詩碟狂襲》採用了「梯形增長」設計：第一排 1 字、第二排 2 字……第 12 排 12 字，字陣上寬下窄（或視覺為下窄上寬，因為新排從上方加入）。這個設計讓遊戲的開局感受如同「詩句逐字展開」，節奏從容而壯觀；同時，越往上的排字數越多，正確字藏得越深，搜索難度自然遞增，無需額外的難度設定便能形成**有機的內建難度曲線**。

---

**各排獨立飄移（Per-Row Autonomous Drift）**

game6 的字陣是整體一起左右移動。《詩碟狂襲》讓每一排字以**各自獨立的速度與方向**左右飄移，碰壁後各自折返。這個設計從根本上摧毀了「等字走過來再打」的懶人策略，也讓畫面視覺從整齊軍陣變成生機勃勃的混沌流體——詩的文字彷彿真的有了生命，在空中自由翻飛，挑釁著玩家的砲口。同時，各排移動速度不同，在字陣密集時形成豐富的視覺節奏感，是天然的美觀設計獎勵。

---

### 5. 核心物理與視覺設計 (Core Physics & Visual Design)

#### 子彈與命中判定系統

子彈為**垂直向上的快速直線飛行**，速度設定為畫面高度/秒（視難度約 600–1000 px/s），幾乎是瞬達最底排。這個設計決策背後有強烈的體驗理由：在《詩陣侵略》中，子彈速度適中，玩家有時間「在子彈飛行途中改變瞄準」——但這種設計模糊了「識字」與「反應時間」的比重。《詩碟狂襲》以極速子彈**強制要求玩家先識字再瞄準**，射擊行為必然晚於識字行為，確保語文知識是操作的前置條件而非可以繞過的東西。

**命中優先序**：子彈只對最底排字元產生碰撞；若最底排的正確字已被擊毀，子彈穿越至上方排時系統仍不計算碰傷（直到舊排消失、新排成為最底排為止）。這個邏輯透過 `isBottomRow()` 判斷函數在 Canvas requestAnimationFrame 迴圈中逐幀執行。

#### 正確字氣球膨脹演算法

正確字每次受擊後調用 `growCorrectChar(charId, increment = 0.1)`：
1. 當前尺寸係數 `scale` 從 1.0 遞增 0.1（上限 1.5）
2. 顏色以 `hsl(0, 90%, lerp(100%, 50%, (scale-1)/0.5))` 插值：白色（scale=1.0）→ 紅色（scale=1.5）
3. 元素以 `transform: scaleX(scale) scaleY(scale); transform-origin: top center` 實現**上緣對齊膨脹**，防止字元下移
4. 同時呼叫 `spawnParticles(x, y, 'yellow', 6)` 產生黃色粒子
5. scale 達到 1.5 時觸發 `destroyCorrectChar(charId)`：播放爆炸粒子 `spawnParticles(x, y, 'gold', 24)`、標記整排為消除狀態、觸發同排混淆字的消散動畫

#### 各排獨立飄移計算

每排字在初始化時被分配 `rowDrift` 參數：
```js
{
  speed: randomRange(difficulty.driftMin, difficulty.driftMax),  // px/s
  direction: Math.random() < 0.5 ? 1 : -1,                      // 初始方向
  phase: Math.random() * Math.PI * 2                             // 初始相位（避免同步抖動）
}
```
每幀 `update(dt)` 中以 `x += speed * direction * dt` 更新位置，碰壁（左右邊界）後 `direction *= -1`。排與排之間的相位差異確保了視覺上永遠有字排在向左移、有字排在向右移，形成流體感。

#### 特攻混淆字弧線路徑系統

特攻字的運動軌跡以**參數化貝茲曲線**定義：
```js
// 入場弧線：從左/右外側入場，弧頂在場地中央偏玩家方向
P0 = { x: -80, y: rowHeight * 3 }   // 起點（畫面外）
P1 = { x: width * 0.3, y: height * 0.6 } // 控制點（弧頂，靠近砲台）
P2 = { x: width * 0.5, y: height * 0.7 } // 接近砲台的最近點
// 達到 P2 後進入「投彈模式」→ 噴射密集炸彈
// 出場弧線：對稱弧線飛離至畫面另一側
```
整個路徑以 `t` 參數（0→1→2）分段計算，`t=0→1` 為入場，`t=1` 為投彈點，`t=1→2` 為出場。這確保了特攻字的軌跡永遠流暢，不會出現急轉彎的不自然感。

#### 炸彈物理

炸彈從混淆字位置垂直或帶有小角度偏移（±15度）向下落，以線性加速度 `vy += gravity * dt` 模擬重力（gravity 依難度設定），使炸彈下落曲線略帶拋物線弧度，增加躲避時的視覺趣味性與可預判性。

### 6. 音樂與音效設計 (Music & Sound Design)

《詩碟狂襲》採用「戰場緊張感 + 精準爆破爽感」的聲音策略，讓每個重要時刻都有清晰的音訊座標。

**背景音樂**：遊戲進行中播放**中速節奏感強的古風背景樂**（建議使用含有鼓擊節拍的合成古箏曲），節奏約 110–130 BPM，與字陣下降的壓迫感互相唱和。背景樂在遊戲開始時淡入，避免突兀；時間進入危急段（20% 以下）時，背景樂音調微幅升高半音，製造本能的心理緊張感，無需任何文字提示。

**操作音效**：
- 砲台左右移動：無音效（避免持續雜音干擾判讀）
- 子彈發射：輕盈的 `playShot()` 音效（短促清脆，不蓋掉其他音效）

**正確/錯誤回饋音**：
- 子彈命中正確字（膨脹）：`playHit()`——清脆的玻璃敲擊音，有明確的「有效命中」感
- 子彈命中混淆字：極輕的 `playMiss()` 悶音，語氣上傳達「打錯了」但不令人沮喪
- 正確字爆破（排消除）：`playSuccess()`——雙音上行的明亮音效，配合爆破粒子同步爆發
- 被炸彈命中：`playFailure()`——低沉震盪音，配合全屏紅色震盪幀

**特殊音效**：
- 三選一獎勵觸發：`playJoyfulTriple()`——三連音向上跳躍，宣告「獲得強化」的愉悅感
- 特攻混淆字入場：獨特的尖銳預警音（類似戰機警報短促嗶嗶聲），給玩家 1 秒預告
- 特攻混淆字投彈：連續快速的 `playHit()` 變體音效，模擬密集炸彈噴射的打擊感
- 計時器進入危急（20%）：低頻脈動音效持續循環，心跳感

**勝利/失敗結算音**：
- 通關：觸發 `melodyPlayer.playFullMelody('茉莉花')` 或 `'踏雪尋梅'`，旋律鮮明，與緊張的戰場聲音形成強烈的情緒落差，從「戰場」瞬間切換到「春風裊裊」
- 失敗：`playSadTriple()`——三連音向下跌落，短促而有尊嚴，不過分煽情

**聲音與視覺的協同**：每次正確字膨脹的 `playHit()` 與字體放大動畫完全同步（同一幀觸發），讓「聽到聲音的瞬間就是字變大的瞬間」，形成觸覺—視覺—聽覺的三重即時回饋。特攻字的投彈音效與炸彈顆粒逐顆噴出的 Canvas 動畫同步，讓密集投彈的聲音有顆粒感而非一片噪音。

### 7. 難度參數的精密控制 (Difficulty Scaling)

| 難度 | 時限 | 生命值 | 字陣下降速度 | 混淆字炸彈頻率 | 排漂移速度 | 特攻混淆字發生頻率(秒) | 三選一強化 |
|------|------|--------|-------------|--------------|---------|-----------|-----------|---------|
| 小學 | 120秒 | 6顆 | 極慢（10px/s） | 低（8秒/次/字） | 慢（30–60px/s） | 30 | 每排觸發 |
| 中學 | 100秒 | 5顆 | 慢（18px/s） | 中（6秒/次/字） | 中（50–90px/s） | 20 | 每排觸發 |
| 高中 | 80秒 | 4顆 | 中（28px/s） | 中高（4秒/次/字） | 中快（70–120px/s） | 10 | 每排觸發 |
| 大學 | 65秒 | 3顆 | 快（40px/s） | 高（3秒/次/字） | 快（100–160px/s） | 5 | 每排觸發 |
| 研究所 | **50秒** | **2顆** | **極快（55px/s）** | **極高（2秒/次/字）** | **極快（140–200px/s）** | 2 | 每排觸發 |

**質變點在高中難度**：特攻混淆字首次出現，使戰場從「靜態壓迫陣」進化為「多方向動態威脅」，玩家不再只需專注字陣，視線必須擴展到整個畫面邊緣，體驗質地截然不同。

**大學難度的關鍵轉變**：字陣下降速度明顯加快，時限縮短，特攻混淆字頻繁出現，三項壓力疊加後，「先識字再射擊」的策略視窗急速壓縮，玩家開始感受到「打字的腦子已經跟不上砲台的手」的撕裂感。

**研究所的極限設計**：時限 50 秒、生命值僅 2 顆、字陣下降速度已足以在 90 秒內壓到底線（玩家必須積極清排才能存活）、特攻混淆字幾乎持續不斷入場、每個正確字需要 8 發子彈方能擊破。這意味著玩家在整局遊戲中幾乎沒有任何「喘息」的時刻——字陣在上方壓下來、炸彈從左右飛進來、正確字像打不死的氣球在原地撐著——這是為那些能在腦海中背出整首詩、指尖反應媲美電競選手的頂尖玩家所設計的**全方位殘忍加壓機**。

**各難度的視覺/音效差異**：小學至中學難度，最底排的正確字以**金色邊框**輔助提示，方便辨識；高中以上移除金色邊框，玩家只能依賴自身詩詞知識搜索正確字。研究所難度下，詩句提示欄僅顯示詩名（不顯示當前輪到哪個字），玩家必須自行追蹤已打完的字數。

## 第二部分：遊戲心得與分析 (Game Experience & Analysis)

### 1. 遊戲性分析

「這是一場在漫天飛字的詩海中，用子彈寫完一首詩的暴力默書。」

《詩碟狂襲》是《花月》宇宙中**「戰場感最強烈、知識壓力最直接」**的一款作品。

它的進化意義在於徹底回答了 game6《詩陣侵略》留下的一個設計懸案：如果整排都是正確字，玩家是在「學詩」還是在「打地鼠」？答案顯而易見。《詩碟狂襲》用一個決定性的規則修正了這個問題——每排只有一個正確字，其餘都是敵人。

玩家的心理狀態會在一局遊戲中經歷三個清晰的階段：

**第一階段：輕鬆建立**（前三排）。字數少、正確字搜索容易，炸彈尚未密集，玩家開始建立「識字→射擊→消排」的節奏感，並從三選一獎勵中積累信心。這是遊戲給予的「溫柔的邀請」。

**第二階段：壓力積累**（中段，排數 5–9）。字排密度增加，各排開始以不同速度漂移，視覺上字海翻湧，正確字在一群長相相似的混淆字中越來越難辨認。特攻混淆字首次出現，玩家的視野被迫從「只看底排」擴展為「全場掃視」。大腦開始在「識字」和「躲彈」之間快速切換，心理負荷以幾何級數增長。

**第三階段：崩潰邊緣的極限高潮**（後段，排數 10–12）。字陣下降速度已令人感到真實的時間壓迫，每一秒拖延都意味著字排的物理逼近。正確字有八發子彈的血量，氣球膨脹到深紅色仍未爆破，畫面上同時有特攻字的弧線、密集炸彈的軌跡、翻飛的字海、以及砲台的子彈穿梭——這是一幅混沌而壯麗的戰場畫卷。

從學習科學的角度，《詩碟狂襲》實現了「提取練習效應」的極限應用：玩家不是在閱讀詩詞，他們是在**彈幕壓力下從記憶深處即時提取特定漢字**。這種「帶壓力的精確提取」比任何靜態背誦都更能強化長期記憶的神經路徑，因為每一次成功識字的瞬間，多巴胺的獎勵訊號會立即通過子彈命中正確字的視覺爆破與音效爆發得到強化。

**頂點時刻**：當玩家在研究所難度下，以 2 顆生命撐過了前十排的驚濤駭浪，用最後一發強化子彈將第 12 排那個被充到深紅色、幾乎佔滿半行版面的最後一個正確字徹底轟爆，金色粒子在畫面上如煙火炸裂、茉莉花旋律緩緩響起、整首詩的進度條在頂端一格一格全部點亮——那一瞬間所感受到的，不只是遊戲的勝利，而是對一首沉睡在記憶某個角落的古詩，一種以火力和汗水換來的重新擁有。

### 2. 精華推廣亮點

**氣球正確字：「我知道這個字是對的，但我還要再打它五槍」**

這是《詩碟狂襲》最獨特的視覺奇觀，也是最天然的短影音素材。當實況主識別出正確字，對著它狂掃連發，眼看著那個字從普通大小膨脹成橙色、再到深紅色、再到佔滿視野的巨型漢字——卻還沒爆！「還差一點！再打！」這種「臨門一腳」的蓄力感配合即將爆破的深紅色大字，是截圖 / Gif / 短影音最完美的定格畫面。

標題可以下：**「打了六槍還沒爆！這顆詩字氣球到底有多硬？研究所最後一排全記錄！」**

---

**特攻混淆字的「刺客入場」：字陣之外突然殺入一個游離字**

正常玩家聚焦在字陣底排時，一個從畫面右側以優美弧線飛入的特攻混淆字往往能造成「我視野裡多了一個東西！」的瞬間慌亂。如果玩家沒有及時閃避就被炸彈命中，那種「我明明打得好好的！突然被偷偷炸死！」的崩潰反應，與《詩陣侵略》裡被 Friendly Fire 炸掉掩體的效果異曲同工，都是實況精華的天然生產機。

標題可以下：**「詩詞版太空侵略者居然有「刺客」！這個字居然繞圈圈攻擊我？！」**

---

**難度反差的視覺震撼：小學版 vs. 研究所版同首詩**

將同一首詩在小學難度（第一排 1 字、字陣靜止、正確字金色提示）和研究所難度（12 排字海翻湧、特攻字橫行、深紅氣球搖搖欲墜）的遊戲畫面並排截圖，那個對比本身就是最強力的宣傳素材。「這是同一首詩、同一個遊戲。」

標題可以下：**「左圖 vs. 右圖：同一首《靜夜思》，這是小學生在玩的版本，這是研究所在玩的版本。」**

### 4. 總結

《詩碟狂襲》是《詩陣侵略》的血脈後裔，也是對它最徹底、最無情的超越。它用一個「每排只有一個正確字」的核心規則，將原版遊戲從「射擊遊戲附帶詩詞」的層次，提升到了「詩詞知識決定射擊成敗」的本質轉換。單字氣球成長系統讓每一發子彈都是可見的積累，而非瞬間消耗的資源；混淆字炸彈反制機制將語文反應速度和生存率直接掛鉤，讓「快速識字」成為真正意義上的戰鬥技能；弧線特攻混淆字的貝茲曲線路徑系統，讓這款遊戲在視覺上達到了「詩歌與彈幕美學同等壯麗」的水準；梯形字陣的有機難度曲線與各排獨立飄移所製造的流體字海，則是任何靜態教材都永遠無法模擬的詩詞生命感。在研究所難度下，它是一場以子彈與汗水鑄就的臨陣默書；在小學難度下，它是每一個孩子都能進入的第一扇詩詞大門。如此極端的包容性，正是花月精神最好的具現——古詩不只屬於書桌，它也屬於戰場。
