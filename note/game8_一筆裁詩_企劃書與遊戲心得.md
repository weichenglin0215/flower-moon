# 《花月》Game8: 一筆裁詩 - 遊戲企劃書與遊戲心得

## 第一部分：遊戲企劃書 (Game Design Document)

### 0. 原型遊戲分析

#### A. 原型遊戲玩法解析

**一筆畫（Flow Free / The Witness 循線解謎）**：玩家在網格上畫出一條不重複的連續線段，從起點到終點，沿途必須覆蓋所有指定節點或格子。核心爽點在於「路徑規劃」——大腦必須同時在空間中預判多步，一旦找到優雅的解法，那種「全部扣上」的滿足感近乎上癮。操作方式：桌機以滑鼠點擊起點後拖曳，觸控以單指滑動。

**文字尋寶（Word Search）**：在充滿字母的矩形網格中，找出隱藏的目標單詞，可以橫向、縱向或斜向排列。核心爽點是「掃描與辨識」——在視覺雜訊中捕捉到目標的瞬間快感。操作方式：點擊起始字母、點擊結尾字母，或直接框選。其最大缺點是順序任意——玩家不需要理解單詞意義，只要形狀吻合即可，學習深度極淺。

#### B. 原型的介面設計與吸引力分析

一筆畫遊戲的介面以「整潔的網格 + 彩色路徑」為核心視覺語言。路徑的即時渲染創造了強烈的「即時回饋感」，每滑過一格就延伸一節，讓玩家的手指動作與畫面變化高度同步，形成沉浸式的心流。文字尋寶的吸引力則來自「資訊密度」——滿格的文字製造了視覺探索的愉悅感，找到隱藏單詞的瞬間有強烈的「發現感」。

花月版本保留了以下元素：即時路徑渲染（SVG 線條）、充滿字元的方格網、以及找到目標後的視覺高亮反饋。

#### C. 詩詞融合分析

| 問題 | 答案 |
|------|------|
| **什麼被替換成詩詞** | 原始的字母或數字節點，被替換成詩詞的每一個漢字；路徑的「連線邏輯」被綁定到詩句的閱讀順序 |
| **如何加入語文知識門檻** | 玩家必須知道這首詩的正確字序，才能判斷下一格應該往哪個方向走；反應再快，不背詩就無法連線 |
| **詩詞學習效果** | 強化「空間記憶」與「字序記憶」的雙重結合——玩家不只記住詩句，更記住了每個字在網格中的相對位置感 |

#### D. 手機觸控適配：矛盾點與解決方案

**矛盾一：手指遮擋問題（最嚴重）**
- **原始操作**：桌機滑鼠游標極細，拖曳時玩家可以清楚看到游標下方的格子內容
- **觸控矛盾**：手指觸控面積覆蓋 1-2 個格子，玩家完全無法看到手指正在經過的字元
- **解決方案**：在畫面上方設置 `game8-char-preview` 元素，即時顯示觸控點當前所在格子的字元，讓「手指看不到的字」出現在「手指看得到的地方」

**矛盾二：觸控精度問題**
- **原始操作**：滑鼠可以精確指向任意像素，格子可以設計得較小
- **觸控矛盾**：手指觸控半徑約 10–20px，格子太小容易誤觸相鄰格
- **解決方案**：9×7 網格配置在 500px 寬的容器中，每格約 50px 寬，符合觸控最小互動尺寸要求

**矛盾三：拖曳手勢與瀏覽器捲頁衝突**
- **觸控矛盾**：單指上下拖曳可能觸發瀏覽器的「頁面捲動」，中斷遊戲內的連線手勢
- **解決方案**：在 touchmove 事件處理器中呼叫 `event.preventDefault()`，阻止瀏覽器預設的捲頁行為，確保遊戲內手勢的優先權

### 1. 遊戲概述與核心目的

**遊戲名稱**：一筆裁詩 (One-Stroke Poetry)
**遊戲目標**：在 9×7 的文字網格中，透過滑鼠拖曳或觸控不放的方式，依照詩詞的正確字序，一筆畫出連貫所有隱藏字元的曲折路徑。
**學習目的**：將以往單調的「文字尋寶 (Word Search)」升級為「動態空間的文意連結」。玩家不能跳躍或中斷，必須大腦與指尖同步運作，在拖曳的過程中不斷預判下一個字形的方位，藉此達成對全詩「字面、句構、空間感」的三重深度記憶——這是純粹背誦或閱讀無法複製的立體記憶方式。

### 2. 遊戲玩法與核心操作

#### A. 遊戲流程

1. **選擇難度** → 系統依難度選取詩詞，生成 9×7（63格）的字元網格，詩句字元隱藏於誘餌字元之中
2. **辨認提示** → 根據難度，特定格子會以金色光暈發光：
   - `hints: all`（小學）→ 當前句所有格子全亮
   - `hints: startEnd`（中學、高中）→ 只有句首與句末格子亮
   - `hints: start`（大學、研究所）→ 只有全詩第一個字亮
3. **開始拖曳** → 玩家按壓起點格，沿上下左右方向滑過相鄰格子，SVG 路徑線即時渲染
4. **釋放判定** →
   - **分句模式**（小學至高中，`splitPath: true`）：完成當前句後放開，系統確認是否正確，再繼續下一句
   - **一氣呵成模式**（大學、研究所，`splitPath: false`）：全詩必須一筆畫完，中途放開即視為失敗
5. **勝利條件**：全詩所有字元依序連線完成
6. **失敗條件**：錯誤次數超過上限 **或** 計時器歸零

#### B. 操作方式

| 平台 | 開始連線 | 延伸路徑 | 確認提交 | 中斷重試 |
|------|---------|---------|---------|---------|
| **桌機（滑鼠）** | mousedown 於起點格 | mousemove 滑過格子 | mouseup 放開 | 路徑自動重置 |
| **手機（觸控）** | touchstart 於起點格 | touchmove 滑過格子 | touchend 放開 | 路徑自動重置 |

**操作限制**：
- 只能走上、下、左、右四個方向，**不能斜向移動**
- 已走過的格子**不能回頭重複**
- 路徑必須完全連續，不能跳格

**手機專屬輔助**：畫面頂端的 `char-preview` 浮動框，即時顯示手指當前所在格子的字元，解決手指遮擋視線的問題。

#### C. 即時反饋機制

- **拖曳中**：藍色 SVG 路徑線隨手指即時延伸；`char-preview` 即時更新顯示當前字元
- **正確句完成**：路徑顏色轉為金色並固定；播放 `playSuccess()` 音效；下一句提示亮起
- **路徑錯誤**：播放 `playFailure()` 音效；錯誤計數 +1；紅心減少；路徑自動重置，可重新嘗試

### 3. 介面設計與使用者體驗

#### A. 主介面佈局

```
┌─────────────────────────────────────┐  ← 頂部資訊列（40px）
│  分數: 0        [小學][重來][開新局]  │    score-board + controls
├─────────────────────────────────────┤  ← 生命值列（40px）
│            ♥ ♥ ♥ ♥ ♥ ♥              │    hearts display
├─────────────────────────────────────┤
│  詩名 / 朝代 / 作者   [進度：第1/4句] │  ← 詩詞資訊列
│ ┌─────────────────────────────────┐ │
│ │  [計時器矩形邊框 SVG Ring]        │ │  ← 計時器包覆整個網格
│ │  ┌───────────────────────────┐  │ │
│ │  │  字 字 字 字 字 字 字 字 字 │  │ │
│ │  │  字 字 字 字 字 字 字 字 字 │  │ │  ← 9×7 文字網格
│ │  │  （63格，含詩句字與誘餌字） │  │ │
│ │  └───────────────────────────┘  │ │
│ │  [已完成路徑 SVG - 金色]          │ │  ← SVG 疊層（已完成）
│ │  [進行中路徑 SVG - 藍色]          │ │  ← SVG 疊層（進行中）
│ └─────────────────────────────────┘ │
└─────────────────────────────────────┘
        ↑ char-preview（手機輔助，浮現於格子附近）
```

#### B. 視覺回饋系統

| 狀態 | 視覺表現 | 色彩 |
|------|---------|------|
| 普通格子 | 白色文字，半透明背景 | `hsla(0,0%,100%,0.15)` |
| 提示發光格（all / startEnd） | 金色光暈脈動 | `hsl(45, 90%, 65%)` |
| 拖曳路徑（進行中） | 藍色粗線 SVG | `hsl(210, 80%, 60%)` |
| 已確認路徑（正確） | 金色粗線 SVG | `hsl(45, 80%, 55%)` |
| 正確完成的格子 | 字元加深顯示 | — |
| 錯誤觸發 | 紅心動畫縮小 | `hsl(4, 90%, 60%)` |

#### C. 計時器呈現方式

計時器採用**矩形邊框 SVG**形式，環繞整個 9×7 網格的四邊。剩餘時間以 `stroke-dashoffset` 動畫表現：從完整四邊逐漸「消失」，視覺上呈現邊框被時間「侵蝕」的緊張感。時間充裕時為暗紅色，進入警戒狀態後顏色可加深，與遊戲緊張感同步升級。

#### D. 使用體驗的流暢度設計

- **格子尺寸容錯**：每格約 50px，符合手指觸控最小互動尺寸（建議 44px 以上），大幅降低誤觸率
- **拖曳防抖**：格子判定以「觸控點進入格子中心範圍」觸發，而非邊緣，避免快速滑動時跳格
- **放手即判定**：不需按確認鍵，拖曳放手後自動判斷，減少操作步驟
- **失敗不懲罰路徑**：路徑錯誤時只計入一次錯誤，但玩家可以立即重新嘗試，不強制等待，降低挫折感的「冷卻期」

### 4. 遊戲機制的繼承與創新

本作參考了一筆畫益智遊戲（如《The Witness》的循線解謎）與文字尋寶（Word Search），但在「網格生成」與「文字誘餌」上做出了極其複雜的獨家創新：

- **連續拖曳連線 (Continuous Path Drawing)**：玩家點選起點後，必須在網格上滑動，路徑由 SVG 線條即時渲染（`.game8-path`）。不能走斜線，只能上下左右移動，且已走過的格子不能回頭。
- **分句挑戰 vs. 一氣呵成（`splitPath` 參數）**：
  - **分句模式（小學～高中）**：允許玩家「一句一句連」。例如先連完「床前明月光」放開手，再接續連「疑是地上霜」，降低短期記憶壓力。
  - **宗師模式（大學、研究所）**：強制關閉 `splitPath`。玩家必須將長達 56 字的古文從開頭到結尾**「一筆畫完，中間絕對不能斷開或放手」**，對指尖穩定度與文學記憶力都是究極的考驗。
- **上下文智能誘餌 (Contextual Decoys)**：系統透過 `SharedDecoy.getThematicDecoy()` 動態生成「在語意上極具干擾性的假字」。如果正解是「月」，旁邊可能就埋伏著「星、日、雲」，讓盲目瞎猜的機率降到最低。

---

### 5. 史詩級的路徑生成演算法 (Aesthetic DFS Generation)

為了讓玩家每次連線都覺得「像是在紙上寫書法一樣行雲流水」，本作在背後的程式花費了龐大算力來評分路徑的美學（`evaluatePath`），這是本企劃最值得驕傲的巧思：

1. **預埋路障強迫繞道（`preOccupied`）**：在生成路線前，系統預先在網格抽樣埋下 1/3 的「死樁」，逼迫演算法必須像河流一樣蜿蜒繞行，拒絕無聊的直線。
2. **三格一轉的黃金節奏（Perfect Turns vs Bad Straights）**：程式記錄路徑的轉彎頻率。直線超過四格被「重罰扣分（`badStraights`）」；連續直行三格後進行一次 90 度完美轉彎則被「重賞」，確保連線圖形充滿優雅的折角與對稱感。
3. **拒絕死角與擁擠（Distribution Rating）**：程式將 9×7 劃分為 9 個大區塊，若路線只擠在一角、留下大片空白（`emptyRegions`），該路線即使合法也直接捨棄。系統在 60 次以上的反覆演算中，為玩家挑選出圖形最平均展開的唯一解。

### 6. 音樂與音效設計

《一筆裁詩》採用「靜默氛圍 + 精準音效爆點」的聲音策略，強化專注感與情緒落差。

**背景音樂**：遊戲進行中**刻意不播放背景音樂**，保持靜默。這個設計讓玩家在拖曳時能完全專注於大腦中詩句的聲音，模擬「在書房中獨自提筆寫字」的沉浸狀態。任何背景音樂都會干擾玩家在腦海中默誦詩句的韻律。

**操作音效**：
- 開啟遊戲 / 選擇難度：`playConfirmItem()`（清脆確認音，宣告「行動開始」）
- 點擊重來按鍵：`playOpenItem()`（輕柔提示音）

**正確/錯誤回饋音**：
- **分句成功**：`playSuccess()`——雙音上行的古箏音，音調向上傳達「登上一層」的成就感，與完成一句詩的節奏感契合
- **路徑錯誤**：`playFailure()`——低沉下行音，瞬間打斷沉浸狀態，製造明確的「走錯了」的警示感

**勝利結算**：通關後觸發 `ScoreManager.playWinAnimation()`，系統播放完整的短旋律（如《茉莉花》片段），配合全螢幕金光炸裂動畫。此刻的聲音從「沉默的緊張」跳躍為「旋律的爆發」，形成極大的情緒落差，是整局遊戲的聲音頂點。

**聲音與視覺的協同**：每次 `playSuccess()` 播放的同時，已完成路徑從藍色切換為金色，聲音的上揚音調與路徑顏色的「升溫」完全同步，讓玩家在聽覺與視覺雙重管道上感受到「答對了」。

### 7. 難度參數的精密控制 (Difficulty Scaling)

| 難度 | 時限 | 詩評下限 | 錯誤容忍 | 提示模式 | 分句模式 | 誘餌池 | 最少句數 |
|------|------|----------|----------|----------|----------|--------|---------|
| 小學 | 80秒 | ★★★★★★ | 6次 | `all`（全亮） | ✅ 可分句 | normal | 4句 |
| 中學 | 75秒 | ★★★★★ | 5次 | `startEnd`（首尾） | ✅ 可分句 | normal | 4句 |
| 高中 | 70秒 | ★★★★ | 4次 | `startEnd`（首尾） | ✅ 可分句 | normal | 4句 |
| 大學 | 65秒 | ★★★ | 3次 | `start`（僅起點） | ❌ 一氣呵成 | normal | 6句 |
| 研究所 | **60秒** | ★★★ | 3次 | `start`（僅起點） | ❌ 一氣呵成 | **hard** | **8句** |

**質變點在大學難度**：`splitPath` 從 `true` 切換為 `false`，玩家從「分句逐步完成」變為「全詩一氣呵成不能放手」，體驗截然不同，如同從爬坡變成跳懸崖。

**研究所的極限設計**：時限最短（60秒）、句數最多（8句起跳）、誘餌池升級為 `hard` 模式——假字幾乎可以與真字構成通順的偽句，讓「感覺連對了」成為最危險的錯覺。玩家一旦連錯一格就得全部重畫，這是專門為頂尖益智玩家保留的刑求場。

## 第二部分：遊戲心得與分析 (Game Experience & Analysis)

### 1. 遊戲性分析：「這是一場宛如在大腦中刻字的極限指尖運動」

《一筆裁詩》是整個專案中，將「空間幾何美學」與「文學測驗」結合得最深邃的一款作品。

當玩家在螢幕上拖曳出一條條黃金比例的折線時，他們實際上是在「重塑」這首詩的閱讀習慣。以前讀詩是從左到右，但在這裡，一首「白日依山盡」可能是「先往下走三格，再向右拐兩個彎，最後一個倒鉤折返」。這種將文字視覺化為一幅巨大「抽象畫」的過程，會引發強烈的心流 (Flow) 體驗。

從使用體驗的角度，小學難度的操作行雲流水——全格發光、分句提交，玩家幾乎只需「找到亮格、連起來」，像在玩一款有詩詞版本的連連看，入門門檻極低。隨著難度上升，提示逐漸熄滅，玩家只剩下自己的記憶與網格中密密麻麻的誘餌字相對。到了研究所，手指必須在 60 秒內完成一次不中斷的蛇形長征，每一格都是記憶與空間判斷的雙重考驗。

**頂點時刻**：當玩家在研究所難度下，不中斷地在螢幕上盤旋繞行長達三十秒，最終一筆連對第 56 個字、全螢幕金光炸裂、成功旋律齊鳴的那一瞬間，所獲得的多巴胺分泌量將遠超任何傳統問答遊戲。

### 2. 精華推廣亮點：足以引爆社群討論的核心魅力

- **「一筆入魂」的極限盲解實況挑戰**：
  這是非常適合短影音 (Shorts / Reels) 或遊戲實況的環節。
  標題可以下：**「挑戰全網最強記憶！56 字古文一筆畫不中斷盲解！」**
  實況主必須在滿畫面似是而非的漢字中，拖著一條藍色軌跡前進。由於獨家的 `SharedDecoy` 會塞入「干擾造句」，實況主往往會自信滿滿地連出一條極度流暢的句子，卻在最後一步發現「啊！前面連到假字了，這邊無路可走！」，隨之而來的崩潰重來，絕對是直播效果滿分的精華。

- **令人驚豔的「美學演算法」幕後揭秘**：
  在推廣時，可以特別向喜愛解謎遊戲的硬核 Gamer 展示路徑生成邏輯。「我們遊戲中的連線不是隨機亂塗，而是經過演算法幾千次計算、確保符合『三格一轉』黃金比例的優雅分佈。」開發團隊對細節的病態執著，會讓懂遊戲的資深玩家產生極高的敬意，進而將其視為高品質的付費級 Indie Game 而非免洗教具。

- **從「小學的溫柔」到「研究所的惡意」**：
  遊戲充分掌握了玩家的挫折曲線。小學難度像是在玩幼教版的連連看，溫馨且充滿安全感；但當玩家自滿地跳到研究所級別，發現「斷句被沒收了」、「終點不亮了」、「周圍還長滿了語意陷阱」，這種「請君入甕」的難度落差，會激發成年玩家不服輸的好勝心。

### 3. 對外推廣短語提案

> *「你以為這只是古詩連連看？試試看在 60 秒內，不中斷、不回頭，一口氣在迷宮裡把《長恨歌》給一筆畫出來！」*

### 4. 總結

《一筆裁詩》徹底顛覆了我們對「填字遊戲」的想像。透過極其複雜的啟發式 DFS 路徑生成演算法，我們確保了關卡永遠具備高度的空間美感與挑戰性；透過嚴密的 Contextual Decoys，我們確保了玩家必須真正理解詩詞而非盲目瞎試；一套精心設計的靜默音景搭配精準音效爆點，則讓每一次成功都在聽覺與視覺雙重管道上製造難忘的情緒高峰；而最重要的 `splitPath: false` 一筆畫極限設定，讓這款遊戲具備了進軍電競挑戰或實況競技的雄厚潛力。手機觸控適配上對「手指遮擋」問題的創意解法，更體現了團隊對實際使用體驗的深刻關注。這不是一款教具，這是一場用程式碼寫就的書法行為藝術。
