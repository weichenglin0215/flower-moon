# GAME25 ~ GAME30 視覺與動畫優化清單

> 依據 [GAME24 開發日誌](game24_從第一次生成程式碼至完美修改.md) 整理的通用優化點，套用到 GAME25~30。

> 依序處理，每個遊戲完成後勾選 ✅。

## 通用優化點（每個遊戲都檢查並套用）

### A. Overlay 與字型基礎
- [ ] 背景採深藍暗黑徑向漸層 `radial-gradient(ellipse at center, hsl(210, 30%, 25%) 0%, hsl(220, 40%, 10%) 70%, hsl(0, 0%, 3%) 100%)`
- [ ] 文字色 `hsl(40, 50%, 80%~85%)`
- [ ] 字型 `'Noto Serif TC', serif`
- [ ] overlay 套 `registerOverlayResize`、`stopGame()` 隱藏 container

### B. Header（50px 高）
- [ ] padding `2px 8px`、bg `rgba(0,0,0,0.4)`
- [ ] Score: `hsl(50, 70~75%, 60~62%)` / 24px bold / `padding-left: 50px` / 金光 text-shadow
- [ ] nav-btn: 紅色立體漸層 `linear-gradient(180deg, hsl(0,55%,32%) 0%, hsl(0,60%,22%) 100%)`、邊框 `hsla(40,100%,80%,0.35)`、字色 `hsl(45,80%,70~75%)`、hover/active 立體回饋
- [ ] difficulty-tag: padding `2px 6~8px` / 20px / `hsl(45, 30%, 85~90%)` / 動態底色（依難度色）

### C. Sub-header（36~40px 高）
- [ ] bg `rgba(0,0,0,0.4~0.45)` + 白色細邊底線 `border-bottom: 1px solid rgba(255,255,255,0.1)` 或金色細邊
- [ ] 主要資訊左、詩名右
- [ ] 詩名 `.poem-info` 18~20px / `hsl(40,55%,82%)` / 底線 / `text-underline-offset: 4px` / 截 16 字 + ellipsis + 全名放 `title` 屬性

### D. 中央內容區
- [ ] 主要區域以「卡片」/「字塊」呈現，狀態變化要有動畫過渡 `transition`
- [ ] 字塊：圓角 8~14px、漸層底、雙陰影（上方高光 + 下方陰影）營造立體感
- [ ] 字塊文字 `line-height: 1; padding-bottom: 0.05em` 修中文字基線偏移
- [ ] 字體尺寸盡量依容器動態算（避免不同難度比例失衡）

### E. 動畫與節奏
- [ ] 進入／離開動畫：spawn、fade、bounce 等使用 `cubic-bezier` 緩動函式
- [ ] 階梯延遲（stagger）：多元素同時出現時，每個 40~80ms 階梯延遲，營造層次感
- [ ] 動畫總時長控制在 200~600ms，不阻塞玩家
- [ ] 玩家操作期間：成功/失敗即時視覺反饋（金光、紅色震動）

### F. 特效（若該遊戲有消除/合成/匹配等動作）
- [ ] 粒子：用 cell 或目標元素的色相同色系
- [ ] 衝擊波：白色擴散圓環
- [ ] 字魂飛升：消除元素飛入頂端對應進度卡（同色發光終點）
- [ ] **scale 雙重縮放陷阱**：若用 `getBoundingClientRect()` 算 FX 座標，必須除以 `window.stageScale` 還原本地像素

### G. 進度顯示
- [ ] 若有「目標收集進度」：用多卡橫排（彩色字塊 + X/Y），達標卡片轉金黃 + 金光脈動
- [ ] 若有「計時器」：採紅白雙框 SVG（仿 game9），或時限模式用單框（金 → 暗紅漸層）

### H. 階段過場（若有多階段）
- [ ] 非最後階段：① 進度發光 → ② 棋盤掉落 → ③ 「恭喜！」金黃橫幅 2 秒 → ④ 進度消失 → ⑤ 新進度出現 → ⑥ 新棋盤滑落 → ⑦ 解鎖。全程 `isAnimating = true` 鎖玩家輸入。
- [ ] 最後階段：不淨空盤面，直接進度卡發光 → `gameOver(true)` → ScoreManager 勝利動畫 → MessageBox。

### I. ScoreManager 整合
- [ ] `playWinAnimation` 的 `heartsSelector` 不能為空字串（用 `.gameNN-no-hearts` 等永不命中但合法 selector）
- [ ] 步數模式必須 `this.timer = this.movesLeft; this.maxTimer = this.maxMoves; this.startTime = 0`
- [ ] `updateTimerRing(ratio, 'win')` 要正確支援；步數模式呼叫 `updateMovesRing()` 重畫紅白雙框
- [ ] `onComplete` 確實開 MessageBox

### J. 詩名顯示與點擊
- [ ] poem-info 可點擊開啟詩詞詳情（`openPoemDialogById`）

## 工作原則

1. 一次只處理一個遊戲，完成後再進下一個。
2. 修改前先讀 game.js + game.css + note，理解該遊戲的玩法與現有狀態。
3. **不更動玩法邏輯**，只改 DOM 結構（必要時）、CSS、動畫排程、視覺反饋。
4. 修改完成後將狀態改為 ✅ 並條列實際改動。
5. 若遇到 5-hour limit，每 10 分鐘檢查一次是否解除，自動繼續。
