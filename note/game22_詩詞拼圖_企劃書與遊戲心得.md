# 《花月》Game22: 詩詞拼圖 - 遊戲企劃書與遊戲心得

## 第一部分：遊戲企劃書 (Game Design Document)

### 1. 遊戲概述與核心目的

**遊戲名稱**：詩詞拼圖 (Poetry Mosaic)
**命名典故**：「碎璧重圓」是本作更詩意的精神內涵——將一首完整的詩詞如玉璧般切碎，化為一片片拼圖，再由玩家親手拼回完整圓滿；但「詩詞拼圖」直白而親切，讓任何玩家一眼明白遊戲的本質。

**遊戲目標**：系統將一首詩的多行文字切割成各種形狀的拼圖片（1×1、1×2、1×3、L型、1×4、2×2），並打亂排列在 7×8 的格子中。玩家透過拖曳移動拼圖片，使整個格子的文字恢復為完整的詩詞原文。

**學習目的**：背詩最困難的不是記住「某個字」，而是記住「整首詩在空間上的完整結構」——哪一句緊接在哪一句之後，每個字在整首詩的脈絡中處於什麼位置。《詩詞拼圖》迫使玩家從「整體詩篇的空間記憶」出發進行重建，而非從局部線索推理，是花月系列中對「全詩背誦」要求最嚴格的一款。

### 2. 遊戲玩法與核心操作

#### A. 遊戲流程

1. **選擇難度** → 系統抽取符合評分的詩詞，依難度決定使用幾行（4 至 8 行），形成一塊完整的「詩詞文字板」，水平排列在 7×8 格子中
2. **拼圖切割**：系統以「隨機多格形切割演算法」將文字板分割成多片拼圖，形狀依難度偏向大片或小片
3. **初始打亂**：所有拼圖片從格子中取出，以隨機順序逐片嘗試放回格子（隨機位置）；能放入的留在格子中，放不進去的移至下方暫存區
4. **提示顯示**：頂端提示行隨機選取詩中一句，依難度設定延遲顯示若干字元；若 `showHintInGrid` 為 true，對應格子同時以黃色半透明文字標示正確字元
5. **玩家組裝**：拖曳格子中的拼圖片或從暫存區取片，將所有片恢復為正確的詩詞排列
6. **勝利條件**：系統逐格讀取 7×8 格子的實際字元，完全符合原詩即告勝利
7. **失敗條件**：計時器歸零

**拼圖片的六種形狀：**

| 形狀代號 | 外觀示意 | 格數 | 說明 |
|---------|---------|-----|------|
| **1×1** | `■` | 1格 | 單字片，最小單位 |
| **1×2** | `■■` 或 `■`<br>`■` | 2格 | 橫排或直排兩字 |
| **1×3** | `■■■` 或直排三字 | 3格 | 橫排或直排三字 |
| **L型** | `■■`<br>`■` 或各旋轉態 | 3格 | L形三字，跨兩行或兩欄 |
| **1×4** | `■■■■` 或直排四字 | 4格 | 橫排或直排四字 |
| **2×2** | `■■`<br>`■■` | 4格 | 2×2 方塊四字 |

> 所有拼圖片**不需旋轉**，維持切割時的原始朝向——文字本身就說明方向，強迫旋轉只是浪費玩家時間。

#### B. 操作方式

**拖曳移動（主要操作）：**

| 操作 | 行為 | 平台 |
|------|------|-----|
| 長按並拖曳格子中的拼圖片 | 片隨手指移動 | 桌機 / 手機皆同 |
| 放開在目標位置（空格）| 片落入該格 | — |
| 放開在目標位置（有片）| 觸發自動置換（見下） | — |
| 從暫存區**向上**拖曳縮圖 | 片以原尺寸進入格子 | — |
| 拖曳片**向下**至暫存區 | 片縮小進入暫存區 | — |

**移動單位**：所有片的上下左右移動以**一個字元格為單位**，每移一格精確對齊網格。

**自動置換機制（Anti-Blocking Displacement）：**

當玩家將片 A 拖放到片 B 的位置時：
1. 系統計算片 B 的**原始形狀所佔格子**是否已被片 A 完全覆蓋
2. 找到格子中**最近的可容納片 B 的空白區域**（以片 B 的幾何中心為原點，螺旋形外擴搜尋）
3. 若找到：片 B 自動滑移到該空位
4. 若格子中無合適空位：片 B 自動縮入暫存區

**復原按鈕（Undo）：**
每次拖曳（包含觸發的自動置換）都完整記錄到操作歷史堆疊（最多 20 步）。按「復原」一鍵撤銷上一次操作，所有受影響的片（含被自動置換的片）全部回到操作前的位置。

#### C. 即時反饋機制

- **拖曳中**：被拖的片放大至 1.05 倍，顯示金色邊框 + 陰影，飄浮在其他片上方
- **放下時**：若落在正確位置（字元符合原詩），該片輕微閃光一次（淡金色光暈）；若不正確，無特殊效果（讓玩家自行判斷）
- **片碰撞置換**：被置換的片顯示 0.3 秒的橘色邊框閃爍，表示「我被移走了」，然後滑移到新位置
- **提示格子高亮**：`showHintInGrid` 啟用時，提示字元所在格子顯示黃色半透明底色 + 文字，以視覺錨點引導玩家定位提示行
- **完成時**：所有片依序（由左至右、由上至下）亮起金光，觸發連鎖光波動畫，配合旋律結算

#### B. 視覺回饋系統

| 狀態 | 視覺表現 | 色彩 |
|------|---------|------|
| 拼圖片（普通）| 半透明深色底 + 2px 白色圓角邊框 + 字元清晰可讀 | 底 `hsla(0,0%,0%,0.4)` |
| 拼圖片（懸停）| 邊框轉金色 | `hsl(45, 80%, 60%)` |
| 拼圖片（拖曳中）| 放大 1.05× + 金色邊框 + 浮動陰影 | — |
| 拼圖片（剛放置正確）| 短暫金色光暈閃爍（0.5秒）| — |
| 被置換的片 | 橘色邊框閃爍 0.3 秒後滑移 | `hsl(30, 80%, 60%)` |
| 提示格子底色 | 黃色半透明（`showHintInGrid` 啟用時）| `hsla(50,100%,70%,0.3)` |
| 提示格子字元 | 黃橙色半透明文字（不蓋住拼圖字）| `hsla(36,80%,65%,0.5)` |
| 暫存區縮圖 | 原始片縮小至 0.55× | 同普通片樣式 |
| 片間間隙 | 2px 空白間隙 | 與遊戲區背景同色 |

**片間間隙設計哲學**：不以顏色區分不同片，而是用**明確的間隙**讓玩家一眼辨別片的邊界。這確保文字的可讀性，同時讓拼圖的「切割感」直觀呈現。

#### C. 計時器呈現方式

SVG 矩形邊框環繞整個 7×8 遊戲區（不含暫存區）。`stroke-dashoffset` 動畫讓邊框逐漸被時間侵蝕。剩餘 25% 時轉為鮮紅，配合邊框閃爍動畫強化緊張感。

#### D. 使用體驗的流暢度設計

- **片的對齊輔助**：拖曳過程中，當拼圖片接近某個合法格子位置（±半格以內），自動磁力吸附對齊，讓精準放置更自然
- **暫存區的進出動畫**：片進入暫存區時以縮小動畫（scale 1.0 → 0.55）進入；從暫存區拉出時以放大動畫（scale 0.55 → 1.0）恢復，讓玩家直觀感知片的縮放變換
- **防誤拖保護**：拖曳啟動需要超過 3px 的位移才觸發，防止玩家輕觸查看時意外移動拼圖片
- **暫存區顯示當前片數**：暫存區右上角顯示「當前存放片數 / 總片數」（如「3/18」），幫助玩家掌握遊戲進度
- **詩詞資訊欄**：提示行右端顯示詩名/朝代/作者（可點擊開啟詩詞資料頁），讓玩家理解所拼之詩的背景

### 4. 遊戲機制的繼承與創新

本作以傳統「拼圖遊戲（Jigsaw Puzzle）」為根基，但以漢字詩詞的獨特屬性做出了三重獨家突破：

- **語義驅動的拼圖（Semantic Jigsaw）**：傳統拼圖的組裝依靠「形狀吻合」或「圖案連接」；本作的每一片都是有意義的文字，玩家組裝的依據是「詩詞語意的流動」——「床前」接「明月光」，而非隨機圖案接隨機圖案。這讓拼圖解題和文學記憶深度融合，而非兩件分開的事。
- **防卡死的自動置換系統（Anti-Blocking Displacement）**：標準拼圖遊戲中，放片無效時玩家只能放回原位。本作的自動置換機制讓「放片」永遠有效——即使格子滿了，放下的片也會讓被擠的片自動找到最近的空位。這消除了滿格焦慮，讓玩家專注在「哪個字應該在哪裡」而非「我放不進去」。
- **三維度提示參數（Triple Hint System）**：`hintDelay`、`hintCharCount`、`showHintInGrid` 三個參數獨立控制「何時看到提示」、「看到幾個字」、「提示是否顯示在格子上」，讓五個難度之間的提示體驗有完全不同的質感，而非只是「提示開/關」二元切換。
- **復原鍵的安全網設計（Undo Safety Net）**：拼圖遊戲中的誤操作往往造成連鎖混亂（A 擠走 B、B 擠走 C……）。20 步復原歷史確保玩家可以大膽嘗試，不必擔心一個誤拖毀掉幾分鐘的進度。

### 5. 拼圖切割與打亂演算法

#### 1. 隨機多格形切割（Random Polyomino Tiling）

系統以貪婪遞迴算法將 7×N 的詩詞文字板分割為拼圖片：

```
切割流程：
1. 從格子左上角出發，掃描第一個尚未分配的格子
2. 依難度的形狀偏好權重，隨機選擇一種拼圖形狀
3. 嘗試從當前格子為錨點放置該形狀（檢查邊界與已分配狀態）
4. 若成功：標記所有佔用格子，記錄為一片，繼續掃描下一個未分配格
5. 若失敗：降級至更小的形狀嘗試，直到 1×1 必然成功
6. 重複直到所有格子皆分配完畢
```

切割完成後，所有片合在一起恰好**完全覆蓋**整個文字板（無重疊、無空缺）。

#### 2. 初始打亂（Scramble）

```
打亂流程：
1. 取出所有切好的拼圖片（維持形狀，不旋轉）
2. 將片的順序隨機打亂
3. 依序逐片嘗試放入遊戲格子：
   a. 隨機選取格子內一個起始錨點
   b. 以該錨點為片的左上角，檢查形狀所需格子是否全部空白
   c. 若全部空白：放入，標記佔用
   d. 若有重疊：嘗試最多 N 個隨機位置後放棄
4. 無法放入格子的片 → 進入暫存區（縮圖顯示）
```

結果：遊戲區有部分拼圖（排列雜亂）、部分空白；暫存區有幾片溢出的片。

#### 3. 勝利判定邏輯

```javascript
// 參考 Game9（詩韻鎖扣）的字元讀取判定寫法
function checkWin(grid, expectedPoem) {
    for (let row = 0; row < grid.rows; row++) {
        for (let col = 0; col < grid.cols; col++) {
            const actualChar = grid.getCharAt(row, col);
            const expectedChar = expectedPoem[row][col];
            if (actualChar !== expectedChar) return false; // 逐格比對字元
        }
    }
    return true;
    // 注意：不比對「哪一片拼圖在哪個位置」
    // 只比對「每個格子顯示的字是否與原詩相符」
    // 處理疊字情況：若兩片含相同字且互換位置，字元讀取結果相同，判定仍然正確
}
```

### 6. 音樂與音效設計

《詩詞拼圖》屬於「深度空間思考型」遊戲，玩家需要在全局視角下規劃拼圖順序，適合輕柔的背景音樂伴奏，但音效必須不干擾集中思考。

**背景音樂**：低增益播放傳統器樂（推薦 `melodyPlayer.playFullMelody('送別')` 或 `'春神來了'`，舒緩節奏符合拼圖冥想狀態）。音樂在遊戲啟動時自動播放，通關後切換為勝利旋律。

**操作音效**：
- 抓取拼圖片（長按觸發）：`playOpenItem()`——輕柔拾起聲，如同拿起一張紙牌
- 拼圖片放置成功（落入正確位置）：`playSuccessShort()`——清脆的確認音，每次放對都有即時鼓勵
- 拼圖片放置不正確（落入錯誤位置）：無音效（靜默，避免過度懲罰感，讓玩家自行判斷繼續調整）
- 被置換的片自動滑移：`playGuzheng(index)` 低增益短音——「滑走的聲音」，如算盤珠撥動
- 從暫存區取出片：`playOpenItem()`

**特殊音效**：
- 復原操作：`playCloseItem()`——「倒退一步」的收縮音感
- 暫存區溢出（片自動進入暫存）：`playFailure()` 極低增益短音——輕微警示，不懲罰

**勝利結算**：
- 最後一片就位後，遊戲區的所有片由左上到右下依序閃光（每格間隔 80ms，總時長約 1-2 秒）
- 閃光序列完成後，`playJoyfulTripleSlow()` 三連上行音，接著 `ScoreManager.playWinAnimation()` 全旋律爆發
- 效果：從「靜默的拼接工作」→「由左上向右下的金色波浪掃過整首詩」→「旋律盛放」——視聽完美同步

**聲音設計的學習效果**：每次放對一片的 `playSuccessShort()` 形成「正確行為即時回饋」的條件反射，即使玩家不知道整首詩是否正確，每個局部的「就位感」也能強化對那幾個字在詩中位置的記憶。

### 7. 難度參數的精密控制 (Difficulty Scaling)

| 難度 | 時限 | 詩評下限 | 詩句行數 | 提示延遲 | 提示字數 | 格內提示 | 最小片格數 |
|------|------|----------|---------|---------|---------|---------|---------|
| 小學 | 180秒 | ★★★★★★ | 4行 | 0秒（立即）| 全部（7字）| ✅ 顯示 | 4格（只用 2×2、1×4） |
| 中學 | 150秒 | ★★★★★ | 6行 | 5秒 | 5字 | ✅ 顯示 | 2格（不用 1×1） |
| 高中 | 120秒 | ★★★★ | 6行 | 10秒 | 3字 | ❌ 不顯示 | 1格（所有形狀皆可出現）|
| 大學 | 90秒 | ★★★ | 8行 | 20秒 | 2字 | ❌ 不顯示 | 1格 |
| 研究所 | **60秒** | ★★★ | **8行** | **永不顯示** | **0字** | ❌ 不顯示 | 1格 |

**質變點一（中學→高中）**：`showHintInGrid` 關閉，格子上的黃色錨點消失。玩家從「有視覺指引的接龍」升級為「純靠記憶的自主組裝」，難度感知落差極大。

**質變點二（高中→大學）**：詩句行數從 6 行增加至 8 行，總字元從最多 42 字跳至 56 字（七言律詩全篇），拼圖片數量可能高達 35-50 片以上，超過玩家工作記憶容量，必須依賴系統性的策略（如先固定某幾行）而非逐片嘗試。

**研究所的極限設計**：8 行詩、60 秒時限、零提示、所有形狀皆可出現（包含大量 1×1 單字片，最多可達 56 片）。玩家面對密密麻麻的 56 個可能散落四處的字元碎片，必須憑藉對整首詩的完整記憶，在不到 1 分鐘內將它們全部放回正確位置。計時器的邊框快速消失，暫存區可能塞滿十幾片等待安置的碎片——這是為詩詞達人設計的終極空間記憶馬拉松。

## 第二部分：遊戲心得與分析 (Game Experience & Analysis)

### 1. 遊戲性分析：「這是一場拿著詩的碎片，在時間沙漏中讓整首詩重新呼吸的修復工程」

《詩詞拼圖》是花月系列中唯一以「整首詩的空間重建」為核心考驗的作品。

從玩家心理的角度看，遊戲有非常清晰的三個認知層次：

**第一層：形狀判斷**——「這片三個字排成直的，應該是詩的某一列」。初期玩家先靠拼圖形狀做分類，把大片和小片分組。這層不需要背詩，純粹是空間邏輯。

**第二層：文字識別**——「這片是『明月光』，它一定在某一行的第三到第五個位置」。當玩家辨識出片上的文字，記憶開始接管幾何判斷，指引片的方向。

**第三層：詩篇整合**——「這行有『床前』，那下一行應該是『疑是地上霜』，但那幾片在哪裡？」玩家開始從詩意脈絡推理每片的相對位置，形成跨片的「上下文組裝策略」。最高難度的玩家幾乎不需要逐片試錯，他們已經在腦海中看到整首詩的版圖，只是在執行一個已知答案的空間操作。

**頂點時刻**：研究所難度下，56 片散落在格子與暫存區，計時器的紅色邊框急速消退。玩家以連貫的手勢連續放片——每一片放下都是一個字在記憶中被確認的瞬間——最後一片就位，金光由左上掃過右下，旋律爆發，整首詩在眼前完整地呼吸了一次。那個瞬間，不是「我拼對了拼圖」，而是「我的記憶足夠完整，足以讓一首詩從碎片中重生」。

### 2. 精華推廣亮點：足以引爆社群討論的核心魅力

- **「研究所 56 片速通挑戰」的視覺震撼**：
  56 片、60 秒、零提示的研究所關卡，玩家在時限內以飛速連續放片的畫面——片片落定、金光連波——是非常適合短影音的「人類極限」挑戰素材。觀眾看到的不只是遊戲技巧，而是「這個人真的把《長恨歌》完整地裝在腦子裡了」。
  標題可以下：**「56 個字拆成碎片，60 秒重組，零提示。這就是詩詞記憶力的頂峰。」**

- **「格內黃色提示」小學版的親切感**：
  小學難度下，格子上有黃色半透明字告訴玩家正確答案的位置，玩家只需把正確的片放到亮著的格子——這讓完全不懂古詩的小朋友也能體驗「拼詩」的樂趣，並在拼的過程中自然記住字的位置。適合製作「帶小孩挑戰」系列的溫馨影片。
  標題可以下：**「6 歲小孩第一次拼唐詩——她不會背，但她拼成功了！」**

- **「誤拖連鎖置換」的混亂爆笑時刻**：
  高難度時，一個誤拖可能觸發連鎖：A 擠走 B、B 擠走 C、C 進暫存區……玩家看到辛苦整理好的半首詩瞬間大亂，然後拼命按復原。這個「拼圖版多米諾骨牌」的混亂瞬間，在實況直播中必然引發觀眾大笑，是絕佳的「蛋糕崩塌」娛樂素材。
  標題可以下：**「一個誤拖毀掉整首詩——我只是想移一片，結果……」**

### 3. 對外推廣短語提案

> *「這首詩被切成了 56 塊碎片。你能不能在 60 秒內，憑著記憶，讓它重新完整？這不是拼圖遊戲，這是用手指考試。」*
> *「小心！把 A 片移走的那一秒，B 片就會被自動推到別的地方——多移幾次，你會在一片混亂中開始懷疑自己還記不記得這首詩。」*
> *「每一次把正確的那片精準放入它原本的位置，都是你的大腦在說：『這個字，我記得它住在這裡。』——《詩詞拼圖》讓背詩變成了一件有手感的事。」*

### 4. 總結

《詩詞拼圖》以「多格形拼圖的空間組裝」為核心互動，將詩詞學習從「記住一個字」提升至「記住整首詩的空間結構」——這是花月系列對「全詩背誦」最立體、最具體的詮釋。三參數提示系統讓難度曲線細膩而層次分明；自動置換機制消除了格子滿載的操作挫折；20 步復原歷史提供了安全的實驗空間；而那片金光由左上向右下掃過整首詩的勝利瞬間，是整個系列中視覺面積最大、情感衝擊最廣的通關演出。能在 60 秒內徒手從碎片中重建一首完整的七言律詩，不是在玩遊戲——是在向古典文學的記憶能力，致上最有手感的敬禮。
