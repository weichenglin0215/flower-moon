# 《花月》詩仙過五關 - 遊戲企劃書與遊戲心得

> **遊戲性質說明**：詩仙過五關並非獨立的選單遊戲，而是任一遊戲通關勝利後自動彈出的積分翻倍小遊戲（Bonus Mini-Game）。它掛載在 `gameMessage` 結算視窗的流程中，以李白形象主持，邀請玩家用通關積分參與五輪翻牌押寶。

---

## 第一部分：遊戲企劃書 (Game Design Document)

### 0. 原型遊戲分析

#### A. 原型遊戲玩法解析

**賭場翻倍功能（Casino Gamble Feature / Double-Up Feature）**：在角子老虎機、電子撲克或多種賭場遊戲贏得彩金後，系統提供一次「翻倍機會」——玩家從兩個選項中選一（紅/黑、高/低、正/反面），猜對則彩金翻倍，猜錯則全部歸零。這個機制的核心爽點是「已贏到手的東西面臨損失的緊張」，而非空手博取，因為「怕失去的恐懼」遠比「渴望獲得的期待」更能激動人心。操作方式極度精簡：點擊兩個按鈕之一，結果立即揭曉。

翻倍押寶的最大吸引力，在於它把「賭徒謬誤」與「風險感知」結合成一種純粹的心理遊戲。連贏三次的玩家內心會同時存在兩個聲音：「我手氣正好，再押一次！」與「連對三次後下一次一定輸！」這種自相矛盾的內心交戰，是它歷久不衰的根本原因。

#### B. 原型的介面設計與吸引力分析

翻倍功能的視覺設計核心是「緊張感製造機器」：翻牌動畫刻意放慢、揭曉前的音效節節升高、成功與失敗之間的極大情緒落差，所有設計都在有目的地操控玩家的心理節奏。介面刻意以「已獲得的彩金數字」佔據視覺焦點，讓玩家每次選擇前都在心理計算「若猜錯就損失了多少」——你保護的是自己已有的東西，而非期望未知的收益，這讓風險感知強烈程度倍增。

花月版本保留了：翻牌揭曉動畫、二擇一按鈕、積分數字的視覺放大效果，以及失敗清零的殘忍感。

#### C. 詩詞融合分析：從原型到花月的轉化邏輯

| 問題 | 答案 |
|------|------|
| **「什麼被替換成詩詞」** | 二擇一的「紅黑牌」被替換成五言詩句的正確字與混淆字；「翻牌」的開獎動作被保留，但牌面從抽象符號變成了具體的漢字 |
| **「如何加入語文知識門檻」** | 兩個按鈕中一個是詩中真正出現在該位置的字，另一個是語意相近、字形相似或同音的干擾字；純憑運氣每關只有 50% 成功率，五關合計僅 3.1% 通過率 |
| **「詩詞學習效果」** | 強化「字序記憶」——玩家不只需要認識詩中的字，更需要精確記住每一個位置上的正確字是哪一個；而翻牌前的腎上腺素分泌，會讓這次記憶提取的神經印跡比平靜狀態下背誦深刻數倍 |

#### D. 手機觸控適配：矛盾點與解決方案

**矛盾一：翻牌動畫進行中的誤觸防護**
- **原始操作**：桌機滑鼠點擊後立即確認，無法雙擊取消
- **觸控矛盾**：翻牌動畫前段緩慢（1 秒），手指若未完全離開按鈕，可能觸發第二次點擊，或誤觸另一個按鈕
- **解決方案**：點擊後立即對兩按鈕設 `pointer-events: none`，整個翻牌動畫完成並判斷結果後才解鎖互動；搭配狀態機管理確保一次只能有一個翻牌動作在執行

**矛盾二：翻牌旋轉時的視覺存在感**
- **觸控矛盾**：垂直翻牌動畫旋轉至 90° 時，按鈕元素幾乎側邊化（視覺寬度接近 0），若按鈕尺寸偏小，玩家可能以為按鈕消失或誤以為觸發失敗
- **解決方案**：按鈕設計為大型圓角矩形（每個至少 150×80px），確保旋轉至最薄時仍有足夠的視覺質感；搭配 CSS `perspective` 製造三維深度感，讓翻牌如同實體卡片而非二維淡出

**矛盾三：彈窗在不同螢幕尺寸的定位**
- **觸控矛盾**：彈窗為全覆蓋式疊層（`position: fixed`），在高縮放比的手機上如果定位使用絕對像素可能偏移
- **解決方案**：透過 `window.registerOverlayResize` 同步彈窗的 `left`/`top`/`scale`，與其他遊戲疊層採用完全相同的縮放架構，確保在所有螢幕尺寸上視覺置中

### 1. 遊戲概述與核心目的

**遊戲名稱**：詩仙過五關 (Poetry Immortal's Five Gates)
**遊戲性質**：通關後彈出的積分翻倍小遊戲，非選單獨立遊戲
**遊戲目標**：在詩仙李白的見證下，以通關積分為籌碼，連續猜中一句高等級五言詩的五個字——每猜對一字（過一關）積分翻倍，猜對五次積分乘以 32 倍取回；猜錯則積分歸零，飲恨而終。
**學習目的**：相較於花月其他遊戲主動考驗詩詞知識，詩仙過五關提供的是「心跳版的主動提取」——玩家在剛通關的多巴胺高峰中，被一首高評分五言詩二度考驗。翻牌前一秒的腎上腺素分泌，觸發了認知科學所稱的「情緒喚起強化記憶固化（Emotional Arousal Enhances Memory Consolidation）」機制：緊張狀態下的主動記憶提取，會讓詞序記憶的神經印跡比平靜狀態下背誦深刻數倍，且難以遺忘。猜對了，記憶加深；猜錯了，你會更想記住這首詩——以備「下次再戰詩仙」。

### 2. 遊戲玩法與核心操作

#### A. 遊戲流程

1. **觸發入口** → 玩家在任一遊戲贏得勝利，`gameMessage` 結算視窗底部出現「下一局」與「過五關」兩個橫排按鈕
2. **進入彈窗** → 點擊「過五關」，全螢幕彈出詩仙過五關對話框（紫色漸層底），顯示目前通關積分
3. **題目出現** → 一句高評分五言詩以灰白色呈現（所有五字可辨識，但色澤淡薄如幽靈文字），字串尾端顯示乘數標籤「X2」
4. **選牌押注** → 左右各一個大按鈕，左按鈕寫「左」，右按鈕寫「右」；系統將當前關卡待猜字的正確字與一個混淆字隨機分配至兩按鈕（玩家不知道哪邊是正確字）
5. **翻牌揭曉** → 點擊任一按鈕，觸發垂直 180° 翻牌動畫：前 90° 耗時 1 秒（緩慢旋轉，製造極限等待），後 90° 耗時 0.2 秒（瞬間彈出，製造開獎驚喜）；牌面由「左」/「右」文字翻開揭示真正的漢字
6. **判定結果**：
   - **猜對（翻出正確字）** → 該字在詩句顯示列從灰白「綻放」為金黃粗體；積分展示 1.5 秒翻倍計數動畫（數字滾增，字體膨脹至 2 倍大）；動畫完成後積分字體縮回預設大小；兩按鈕翻回正面顯示「左」「右」；若五關未滿則繼續下一關
   - **猜錯（翻出混淆字）** → 積分瞬間清零；彈窗背景從紫色漸層轉為深灰漸層（0.5 秒過渡）；底部按鈕文字換為「下次再來」；點擊後彈窗關閉
7. **過五關（五連勝）** → 全部五字依序轉金；金色星星煙火從畫面中央爆散；底部按鈕改為「過5關，領取32倍積分」
8. **領取積分** → 點擊「領取」，彈窗以縮小動畫同步移動至左上角的積分顯示區，視覺上「積分落入得分格」；動畫結束後彈窗從 DOM 移除
9. **中途放棄** → 任何成功猜對後，玩家可隨時點擊「不敢比了，獲取積分」以當前已翻倍積分安全收場並關閉彈窗

**勝利條件**：連續猜對 5 關，積分乘以 32 倍
**失敗條件**：猜錯任意一關，積分歸零
**中途收益**：猜對至少一關後，可任意時刻選擇放棄，保留當前翻倍後的積分

#### B. 操作方式

| 平台 | 選牌互動 | 翻牌觸發 | 放棄或領取 |
|------|---------|---------|-----------|
| **桌機（滑鼠）** | hover 時邊框發光提示 | `click` | `click` 底部按鈕 |
| **手機（觸控）** | — | `pointerdown` / `touchstart` | `tap` 底部按鈕 |

**操作限制**：
- 點擊後兩按鈕即時鎖定（`pointer-events: none`），不可重選或取消
- 翻牌動畫完整播放期間（共 1.2 秒）無法進行任何操作
- 第一關開始前積分已確定（通關積分），不與遊戲中的累積計算混淆

**彈窗適配說明**：彈窗以 `position: fixed` 掛載於 `document.body`，透過 `registerOverlayResize` 同步 `left`/`top`/`scale`，確保所有螢幕尺寸下完整置中顯示，不受 `#stage` 的 CSS transform 影響。

#### C. 即時反饋機制

| 時機 | 視覺反饋 | 音效反饋 |
|------|---------|---------|
| 點擊按鈕（翻牌前段，0-1 秒） | 按鈕緩慢旋轉，透視漸縮 | 低頻心跳音循環 |
| 翻牌後段（1-1.2 秒） | 按鈕快速彈出，牌面字元閃現亮起 | 開獎音效爆發 |
| 猜對 | 詩句該字從灰白綻放為金黃粗體；積分數字 1.5 秒滾增；字體同步膨脹至 2 倍 | 短促成功音 + 計數滾動音效 |
| 猜錯 | 積分閃紅後瞬間清零；背景 0.5 秒漸變為深灰漸層 | 三音下行悲鳴 |
| 過五關 | 金色星星煙火從中央爆散 | 完整喜悅旋律 |
| 領取積分 | 彈窗縮小並滑入左上角積分區 | 古箏滑音下行 |

#### D. 勝利/失敗標準

**勝利（過五關）**：五次全部猜對，觸發金色煙火，顯示「過5關，領取32倍積分」按鈕，點擊後彈窗以動畫收入積分顯示區，積分確定生效。

**失敗**：猜錯任一關，積分歸零，背景轉深灰，顯示「下次再來」按鈕，點擊直接關閉彈窗。失去的積分不找回，沉沒成本悉數放棄。

**特殊情境（文位升級保護）**：玩家通關時積分即已發放，文位可能因此升級。詩仙過五關的翻倍積分若再觸發升級，視為系統彩蛋贈禮，不扣回通關積分、不觸發降級邏輯，確保流程不因重複升降位引發複雜性問題。

#### E. 獲得高分的技巧

- **深耕高評分詩詞**：題目選自評分 ★★★★ 以上的五言詩，平時認識的高評分詩詞越多，成功率越高；背詩真的有差
- **識字辨形**：混淆字多為字形相似（如「己」vs「已」）、語意相近（如「月」vs「日」）或同音字，對漢字字形有精確認識的玩家有天然優勢
- **知進知退**：連勝一兩關後若對接下來的字不確定，果斷選「不敢比了」落袋為安，比貪心押注更聰明
- **善用翻牌前的一秒**：熟悉後，翻牌前段 1 秒不只是等待，而是一次高壓下的快速記憶確認儀式——善用這一秒在腦海中默念詩句

```
背景：紫色漸層 → 深灰漸層（0.5s 過渡）
積分：顯示 0（閃紅後定格）
按鈕：變為「下次再來」（點擊關閉彈窗）

**過五關後介面變化**：
```
詩句：五字全部金黃粗體 ＋ X32 標籤

**尺寸規劃（500×850px 邏輯舞台基準）**：
- 彈窗寬度：460px，高度：約 560px，圓角 16px
- 詩句各字字體：36px，字距 6px，灰白色 `rgba(255,255,255,0.38)`
- 乘數標籤（X2 等）：20px，金黃色 `#FFD700`
- 積分數字：預設 48px，翻倍動畫時膨脹至 96px（`transition: font-size 0.8s ease-out`）
- 翻牌按鈕：各 180×80px，兩個橫排居中，間距 16px，圓角 12px
- 底部放棄按鈕：寬 300px，高 48px，圓角 24px

#### B. 視覺回饋系統

| 狀態 | 視覺表現 | 色彩規格 |
|------|---------|---------|
| 詩句未猜中（灰白） | 半透明淡字，可辨認但色澤沉靜 | `rgba(255,255,255,0.38)` |
| 詩句已猜中（金黃） | 不透明金黃粗體，外發光暈 | `#FFD700`，`text-shadow: 0 0 10px #FFB300` |
| 乘數標籤（X2/X4…） | 金黃色小標，隨關卡更新 | `#FFD700`，font-size 20px |
| 翻牌按鈕（正常） | 金黃色邊框，深紫半透明底，淡黃字 | border `2px solid #FFD700`，bg `rgba(80,0,120,0.6)` |
| 翻牌按鈕（hover，桌機） | 外發光擴大，邊框升溫白金 | `box-shadow: 0 0 16px #FFF8DC` |
| 翻牌旋轉中 | CSS `rotateY()` + `perspective` 3D 旋轉 | 旋轉至 90° 時切換牌面內容 |
| 翻牌揭曉（牌面亮起） | 漢字以大字閃現，一幀強光爆發 | 字體 48px，`#FFFFFF` 閃後穩定為 `#FFD700` |
| 正確回饋 | 積分計數滾動，字體 `font-size` 膨脹 | 金黃發光，動畫結束後縮回 |
| 錯誤回饋 | 積分 `#FF4444` 閃後清零，背景漸變 | 彈窗 bg 由 `#6A0DA0→#1A1A1A` 過渡 |
| 過五關煙火 | 金色星星從中央爆散至四角（Canvas 特效） | `#FFD700`、`#FFA500`、`#FFFACD` 三色混合 |
| 彈窗歸位動畫 | 縮小並同步平移至左上角積分顯示區 | CSS `transform: scale + translate`，600ms |

**配色核心準則（花月規範）**：
- 金橙（`#FFD700` / `#FFA500`）＝一切強調、獎勵、正確、成功
- 深紫漸層（`#4B0082 → #6A0DA0`）＝正常遊玩背景，神秘而高貴
- 深灰漸層（`#333 → #1A1A1A`）＝失敗後的悲涼底色，心情轉折的視覺信號
- 淡黃（`#FFF8DC`）＝一般說明文字，柔和可讀

#### C. 計時器呈現方式

詩仙過五關**刻意不設計計時器**。這個決策本身是一種設計語言：賭注的緊張感來自「輸贏立判的二選一」，而不是時間壓力。去除計時器讓玩家在翻牌前的那一秒鐘，有充分的心理空間感受自己到底確不確定——這種「沒有藉口的空間」反而比時間催趕更讓人緊張。

若玩家長時間不動，系統不強迫結束；詩仙等待的耐心，本身也是一種角色塑造。

#### D. 使用體驗的流暢度設計

- **防誤觸鎖定機制**：點擊後即時設 `pointer-events: none` 至兩按鈕，翻牌動畫完整播放、結果判定完成後才解鎖，防止衝動下的多次點擊
- **動畫中斷保護**：翻牌動畫期間若切換頁面或應用，狀態機保留當前關卡狀態，返回後繼續；不允許中途重選
- **積分動畫可視性**：計數滾動使用 `requestAnimationFrame` 逐幀更新，確保低階設備上數字增長也流暢；字體膨脹用 CSS `transition` 確保硬體加速
- **挫折感管理**：失敗後深灰底色與「下次再來」的文案，刻意不提供「立即重玩」選項——失去的感覺需要被感受完整，這反而強化了「下次要記住這首詩」的學習動機；彈窗關閉後玩家回到 gameMessage 結算視窗，仍可點選「下一局」繼續遊戲，挫折感不蔓延到主流程

### 4. 遊戲機制的繼承與創新

**繼承**：本作參考了賭場角子老虎機的「翻倍押寶（Gamble Feature）」——二擇一、猜對翻倍、猜錯歸零，是賭博文化中最古老且最純粹的機制。詩仙過五關保留了這個機制的核心張力，但以「詩詞知識」取代了純粹的運氣，使得「勝算」從 50% 擴展至「與你的詩詞底蘊成正比」。

**創新設計決策**：

**⓵ 李白形象的主持設計：從功利賭注到浪漫邀約**
李白是中國文學史上「酒、詩、不羈人生」的終極符號——他的人生本身就是一場詩仙的豪賭。讓他擔任過五關的主持人，在文化語境上形成完美的自我指涉：「你贏了花月的考驗，但詩仙問你：你敢不敢用詩詞知識跟我再賭一把？」這不只是角色選擇，而是一種「文化邀約」，讓押注的行為從功利計算升華為浪漫的文學挑戰。

**⓶ 前慢後快的翻牌節奏：多巴胺調控的精密工程**
不同於傳統翻牌動畫的等速旋轉，詩仙過五關採用「前 90° 耗時 1 秒、後 90° 僅耗 0.2 秒」的極端非線性設計。前半段的緩慢旋轉模仿了老虎機滾輪「快要停下」的心理高峰體驗，製造了「知道答案但無法確定」的焦慮等待；後半段的快速彈出則像一根針刺破氣球，創造了「瞬間開獎驚喜」。

CSS 實作以分段 `@keyframes` 精確控制各段時間比例（總時長 1.2 秒，`0%→83.3%` 對應前 1 秒 rotateY 0→90°，`83.3%→100%` 對應後 0.2 秒 rotateY 90→180°），搭配 `animation-timing-function: linear` 確保各段勻速。旋轉至 90° 的 edge-on 最薄時刻，以 JS 切換按鈕牌面內容（由「左」/「右」切換為真正的漢字），確保「揭曉感」在視覺最窄點完美完成。

**⓷ 積分膨脹動畫：翻倍的儀式感**
積分翻倍時不只是數字改變，而是一場視覺儀式：數字在 1.5 秒內以 `requestAnimationFrame` 配合 `easeOutCubic` 緩動曲線從舊值「計數滾動」到新值（快速衝刺後緩慢收尾），同時 CSS `transition` 讓字體從預設大小膨脹至 2 倍，彷彿積分因翻倍而「脹滿喜悅」。動畫結束後字體以同樣的 transition 縮回，留下完整的視覺記憶。這讓每一關的成功都成為一個可截圖、可分享的視覺爆點。

**⓸ 五關累積的懸念架構：乘數不只是數字，是心理壓力**
乘數從 X2、X4、X8、X16 到 X32，每進一關懸念翻倍。第一關感覺像小試身手，風險微乎其微；但到了第四關，玩家手上已是原積分的 8 倍，此時第五關的「再賭一次」需要巨大的心理勇氣——因為輸掉的不只是數字，是你剛剛幾乎要成功的七分之六成就。五關的設計恰好在「可承受的誘惑」與「不可承受的損失恐懼」之間達成完美的心理平衡點。

**⓹ 彈窗歸位動畫：讓積分「真實地落入口袋」**
過五關後點擊「領取」，彈窗不是普通的淡出消失，而是使用 `getBoundingClientRect()` 取得左上角積分顯示元素的精確座標，計算 CSS `transform: scale() translate()` 向量，在 600ms 內讓彈窗縮小並滑向積分區，視覺上「獎金從天而降，精準落入得分格」。這個設計把「抽象的數字變化」轉化為「具象的財富移動」，強化了積分對玩家的真實感。

**⓺ 不計時的緊張感設計：沉默才是最大的壓力**
去除計時器是刻意的逆向設計：大多數遊戲用時間壓力製造緊張，詩仙過五關則用「沉默的無限等待」製造壓力——你可以想無限久，但你知道自己不確定，那個不確定感本身才是真正的緊張來源。計時器的存在反而提供了「時間到了只好猜」的逃避路徑；沒有計時器，你必須正視自己是否真的記得這首詩。

### 5. 狀態機設計與翻牌動畫技術

詩仙過五關的技術核心是「精確的有限狀態機 + 分段動畫控制系統」。

**五關狀態機（Five-Gate State Machine）**：
```
IDLE
 ↓ show()
INIT         ← 讀取通關積分、選取五言詩、渲染灰白詩句
 ↓
WAITING      ← 顯示左右按鈕（「左」「右」），等待玩家點擊
 ↓ 玩家點擊
FLIPPING     ← 鎖定按鈕，播放 CSS @keyframes pokerFlip（1.2s）
 ↓ 動畫結束
JUDGING      ← 判斷正誤
 ├─ 正確 → CORRECT_ANIM（積分膨脹 1.5s）
 │    ├─ 五關未滿 → 延遲 0.8s → WAITING（下一關）
 │    └─ 五關滿了 → WIN（煙火動畫）
 └─ 錯誤 → LOSE（背景變色、歸零、按鈕換文）

WIN    → 等待玩家點「領取」→ COLLECT_ANIM（彈窗歸位 600ms）→ 關閉
LOSE   → 等待玩家點「下次再來」→ 關閉
任何狀態的 QUIT_BTN → QUIT_ANIM → 關閉
```

**翻牌動畫 CSS 精確計時設計**：
```css
/* 翻牌關鍵幀：前 83.3% 時間旋轉前 90°；後 16.7% 時間旋轉後 90° */
@keyframes pokerFlip {
    0%    { transform: rotateY(0deg); }
    83.3% { transform: rotateY(90deg); }   /* 此處 JS 切換牌面內容 */
    100%  { transform: rotateY(180deg); }
}
.flipping {
    animation: pokerFlip 1.2s linear forwards;
    perspective: 600px;
}
```

JS 監聽 `animationiteration` 或使用 `setTimeout(800ms)` 在 83.3% 處切換按鈕顯示的文字（由「左」/「右」替換為正確字/混淆字），確保翻至最薄時內容已切換，翻過來即為揭曉的漢字。

**積分計數器動畫（requestAnimationFrame + easeOutCubic）**：
```javascript
// 1.5 秒內將 oldScore 線性滾動至 newScore，字體同步膨脹
function animateScore(fromVal, toVal, duration = 1500) {
    const start = performance.now();
    function step(now) {
        const t = Math.min((now - start) / duration, 1);
        const eased = 1 - Math.pow(1 - t, 3); // easeOutCubic
        scoreEl.textContent = Math.round(fromVal + (toVal - fromVal) * eased);
        if (t < 1) requestAnimationFrame(step);
        else { /* 動畫結束，縮回字體 */ }
    }
    requestAnimationFrame(step);
}
```
字體膨脹由 CSS 控制：`transition: font-size 0.8s ease-out`，JS 直接設定 `el.style.fontSize = '96px'` 觸發，動畫結束後設回 `'48px'`。

**彈窗歸位動畫（getBoundingClientRect + CSS transform）**：
```javascript
function collectAnimation() {
    const scoreRect = document.getElementById('menu-score').getBoundingClientRect();
    const dialogRect = dialog.getBoundingClientRect();
    const scale = window.stageScale || 1;
    const tx = (scoreRect.left - dialogRect.left) / scale;
    const ty = (scoreRect.top - dialogRect.top) / scale;
    dialog.style.transition = 'transform 0.6s ease-in, opacity 0.6s ease-in';
    dialog.style.transform = `translate(${tx}px, ${ty}px) scale(0.15)`;
    dialog.style.opacity = '0';
    // 600ms 後移除 DOM
}
```

### 6. 音樂與音效設計

詩仙過五關的音效策略採用「靜→心跳→爆發→餘韻」四段式情緒弧線，每一個聲音都精準服務於「賭注緊張感」的製造與釋放。

**背景音樂**：進入彈窗後，背景以極低音量（約 15%）播放細碎的古箏撥絃或低鳴的琵琶聲，不是輕鬆愉快的遊戲音樂，而是「幕後有人在看你出牌」的微妙壓迫感。翻牌期間音樂短暫靜音，讓心跳音效佔滿音景。

**操作音效**：
- 進入彈窗：`playConfirmItem()` ——清脆而莊嚴，宣告「賭局正式開始」
- 點擊按鈕觸發翻牌：低頻心跳音（單一低沉鼓擊）在翻牌前段 1 秒循環，節奏平穩如脈搏，製造等待中的生理臨場感

**正確/錯誤回饋音**：
- **猜對**：翻牌後半彈出瞬間觸發 `playSuccessShort()`（清脆上揚），隨後積分計數滾動期間以每 0.15 秒一聲的古鐘音漸強，彷彿數鈔聲音此起彼伏，財富感知逐秒增強
- **猜錯**：`playSadTriple()` ——三音向下的古箏哀鳴，模擬籌碼落入虛空的聲音；積分歸零後系統保持 1.5 秒的靜默，讓失落感在無聲中完整瀰漫，這段靜默本身就是最大的音效設計

**特殊音效**：
- 過五關煙火：`playJoyfulTriple()` 先行爆發，緊接 `melodyPlayer.playFullMelody('望春風')` 前幾小節，金色煙火配合旋律盛開，宣告玩家達到「詩仙認可」的知識境界

**勝利結算音**：玩家點擊「領取積分」後，彈窗縮小歸位的 600ms 動畫中播放一段快速下行的古箏滑音（glissando），視覺上彈窗縮入積分區，聽覺上金音落地，製造「獎金入袋」的完整感官體驗。

**聲音與視覺的協同時間表**：

| 時間點 | 視覺 | 音效 |
|--------|------|------|
| 點擊按鈕後，0-1s | 翻牌前段慢旋轉 | 低頻心跳循環 |
| 1-1.2s | 翻牌後段快速彈出、字元閃亮 | 開獎音效爆發 |
| 1.2-2.7s（猜對） | 積分滾增，字體膨脹 | 古鐘計數音漸強 |

| 過五關 | 金色煙火爆散 | 三連喜悅音 + 完整旋律 |

### 7. 難度設計與進關壓力系統

詩仙過五關無傳統的「小學至研究所」難度分層，所有玩家面對相同難度的題目——詩仙面前人人皆同。其難度控制來自以下三個層次：

**層次一：詩詞選題難度（技術參數）**

系統透過 `getSharedRandomPoem` 選取評分 ★★★★ 以上（`poemMinRating: 4`）、恰好五字（`minChars: 5`、`maxChars: 5`）的單句五言詩句，確保題目具備知識門檻：非路人玩家隨眼看一遍就能確認每一個字。混淆字選取採用 `SharedDecoy.getThematicDecoy()` 生成語意相近或字形相似的干擾選項，確保二擇一不是「廢字 vs 顯然正確字」的送分題。

**層次二：累積乘數的心理壓力（設計參數）**

| 關卡 | 當前乘數 | 押注後可得 | 心理狀態描述 |
|------|---------|-----------|------------|
| 第1關 | ×1（通關原值）| ×2 | 「反正沒損失，試試看」，輕鬆下注 |
| 第2關 | ×2 | ×4 | 「小有積累，要不繼續？」，開始在意 |
| 第3關 | ×4 | ×8 | 「回頭路已遠，放棄真可惜」，依戀感浮現 |
| 第4關 | ×8 | ×16 | 「手汗開始了，再一關而已」，理性邊緣崩解 |
| 第5關 | ×16 | ×32 | 「全都是這一翻了」，手抖真實發生 |

**層次三：知識 vs. 運氣的分流機制（核心設計）**

熟悉該詩的玩家每關成功率約 90%，五關複合通過率≈59%；
不熟悉的玩家每關成功率約 40-50%，五關複合通過率≈1-3%。
這 20 倍的成功率差距，從根本上確保了「詩詞知識決定命運，而非瞎猜運氣」——讓「靠知識打牌」成為可以被玩家親身感受到的事實。

**「質變點」說明**：本遊戲的質變不在任何技術參數的切換，而在玩家心理狀態的轉折——第四關的腎上腺素激增是設計的必然結果，也是讓「高壓下的記憶提取」發揮最大學習效果的關鍵時刻。

## 第二部分：遊戲心得與分析 (Game Experience & Analysis)

### 1. 遊戲性分析

「這是一場詩人與賭徒之間的靈魂握手——你用自己掙來的詩詞積分，跟詩仙打賭你真的記得他的詩。」

《詩仙過五關》是花月生態中唯一一款讓玩家「拿著自己的成就下注」的機制，它在通關喜悅的頂點插入了一個哲學問句：你對自己的詩詞知識，有多少把握？

當玩家剛剛以精準答題換來一千分，那份積分是有重量的——它代表了注意力、記憶力和努力。然後，詩仙出現，用一句你「應該認識」的古詩挑釁你。這一刻的心理結構極其微妙：你保護的不是抽象數字，而是你剛用腦力掙來的東西。「怕失去」的恐懼，永遠比「渴望獲得」的期待更強烈——這是行為經濟學的損失規避定律，詩仙過五關把這個定律精準地嵌入了詩詞學習之中。

翻牌前的那一秒，是整個遊戲最精彩的一秒。玩家的大腦同時在做三件事：高速提取這首詩的字序記憶、比對兩個按鈕的視覺線索（哪一個感覺「對」？）、以及快速計算「如果我錯了，我失去了多少」。這三種思維在 1 秒鐘內的高速交叉，製造了任何填答遊戲都無法複製的「知識版腎上腺素」。而這種高壓狀態下的主動提取，正是認知科學「提取練習效應（Retrieval Practice Effect）」加上「情緒喚起強化記憶固化」的雙重加成——此刻的記憶，比任何平靜刷題都更持久。

從遊戲體驗的弧線來看：第一關輕鬆入局，第三關開始有些暖手汗，第四關已是真實的心跳加速，第五關則進入「冰封」狀態——玩家往往會停頓很久，在沉默中與自己博弈。這種節節升高的壓力弧線，在不調整任何技術參數的情況下，純粹靠「已有積分的重量」自動生成，是這款遊戲最精妙的被動難度系統。

**頂點時刻**：第五關翻牌後段的 0.2 秒瞬間彈出，金黃字「一字化龍」亮遍五格，金色星星從四面八方爆散，積分計數器在 1.5 秒內從 16 倍滾上 32 倍、數字膨脹至佔滿積分欄——在那個瞬間，所有的心跳、猶豫和恐懼都化成了純粹的震顫。那感覺不是通過了一個小遊戲，而是「詩仙親口認證了你的詩詞底氣」。

### 2. 精華推廣亮點

- **「翻牌前的一秒鐘」——天然的短影音情緒素材**
  鏡頭聚焦玩家手指懸停在按鈕上方，翻牌緩慢旋轉，心跳音效逐漸緊繃，1 秒後——「砰」，快速彈出。玩家或歡呼衝天或一聲哀嚎，整段影片不超過 15 秒，卻包含了「期待→緊張→爆發」的完整情緒弧線。這是為 Shorts / Reels 量身定製的天然素材。
  標題可以下：**「用 5 萬分賭一句李白的詩，翻牌那一秒我差點忘了呼吸」**

- **「第五關手抖實況」——人性的現場直播**
  當實況主手上握著相當於 8 萬分的虛擬財富，面對第五關「再翻一次就 32 萬，翻錯就是零」的抉擇，他說的每一句話都是人性的鏡子：「我知道這首詩的啊！但萬一記錯一個字...」「不行不行，再讓我想想...」「我是詩仙的粉絲，我一定行！」這種讓觀眾比玩家更緊張的視角，是無可取代的直播互動素材。
  標題可以下：**「花月詩仙過五關第5關，手抖到無法點擊的真實紀錄」**

- **「知識 vs. 運氣」的數學揭密——讓背詩變得有說服力**
  在推廣時特別強調這個反直覺事實：這個遊戲**不是** 50% 運氣的賭博，而是「你對高等級詩詞的掌握度決定你的成功率」。熟悉該詩的玩家五關通過率約 59%，不熟悉者約 1-3%，差距 19 倍。「背詩真的有差」——這句話通過詩仙過五關的數據，第一次變成了可以用親身體驗驗證的命題。
  標題可以下：**「背詩的人和沒背詩的人，在同一個賭局裡命運相差 19 倍——花月詩仙過五關」**

### 3. 對外推廣短語提案

> *「你以為通關就結束了？詩仙在門口等你。他說：積分押上來，用你的詩詞知識，跟我過五關。」*

> *「小心！第四關開始，手上已有的積分會讓你的手指顫抖——那不是恐懼，那是你終於知道什麼叫做『把自己的知識攤開來賭一把』。」*

> *「我們用認知科學設計了一個秘密：翻牌前那一秒鐘的腎上腺素，是你大腦寫入詩詞記憶最深刻的時刻。這不是在做遊戲，這是在用最貴的心跳，換最不會忘記的古詩印記。」*

### 4. 總結

《詩仙過五關》以一個幾乎不需要獨立遊戲流程的彈窗機制，在花月生態中創造了所有主遊戲都無法複製的情感密度：它把「知識確認」這個原本平淡的動作，轉化為一場有血有肉的存在主義選擇——你對這首詩有多確定？確定到敢把剛贏到手的積分全押上嗎？

翻牌前 1 秒的慢旋節奏、積分膨脹的視覺儀式、以及彈窗縮回積分區的具象歸位動畫，每一個細節都在向玩家說：「你贏得的不只是數字，是你自己的詩詞底氣。」而李白這個角色的存在，讓這場賭注從功利計算升華為一種文化浪漫：詩仙邀你入局，不是要取走你的積分，而是要確認你是否真的值得那份積分。

猜錯了，你記住了這首詩；猜對了，你記住了在詩仙面前勝利的那一刻。無論哪種結果，《詩仙過五關》都讓那首詩，從此刻起不再只是一串字符。
