# 《花月》Game9: 詩韻鎖扣 - 遊戲企劃書與遊戲心得

## 第一部分：遊戲企劃書 (Game Design Document)

### 1. 遊戲概述與核心目的
**遊戲名稱**：詩韻鎖扣 (Nuts & Bolts: Verse)
**遊戲目標**：將打亂在各個鎖扣（螺栓）上的文字螺帽，透過不斷的搬移、暫存與排序，重新拼湊回完整的四句唐詩宋詞。不同於傳統的同色排序，每一次的同色堆疊，都是在正確還原一句名詩。
**學習目的**：將備受大眾喜愛的「水排序 (Water Sort) / 漢諾塔」邏輯益智遊戲，與詩詞的單字順序結合。這不僅考驗玩家的空間規劃與多步推演能力（工作記憶），更能反向測驗玩家對詩句上下文的熟悉度——因為當顏色失去辨識度時，只有讀懂詩意才能解開死結。

### 2. 遊戲機制的繼承與創新 (Game Mechanics)
本作繼承了「水排序」必須符合「同色相疊（或空槽）」與「空間上限」的經典規則，並做了極具深度的創新：

- **批量搬移 (Multi-Move)**：為了解決中後期搬移操作的繁瑣感，遊戲允許玩家「一次抓起多個已經排序正確的相鄰螺帽」並一起移動。這極大地增加了操作的爽快度與大局觀的佈局彈性。
- **無解防禦機制 (Guaranteed Solvability)**：為了避免隨機生成導致無解的死局，我們的關卡生成演算法是從「完成態」開始反推。透過控制 `exchangeQuantity`（預先拔出的螺絲數量）與 `totalNumberOfExchange`（打亂的步數），保證每一道題目絕對有解。
- **壓力引擎：怠工懲罰 (`inactivityThreshold`)**：水排序通常是節奏極慢的休閒遊戲，但我們加入了強烈的壓力要素！畫面的外框不再是普通的倒數計時，而是**「剩餘步數的紅白節點環」**。當玩家停止思考、不採取動作超過 **5 秒**，系統就會發布「怠工警告」並**強制扣除一步**。這逼迫玩家必須在大腦運算與動手操作間無縫切換，極大提升了遊戲的刺激感。

### 3. 多維度的難度調控 (Difficulty Scaling)
為了拉開休閒玩家與益智狂人的差距，本遊戲的難度控制不再只是簡單的「時間長短」，而是動用了五種維度的複合參數：
1. **提示階級 (`hasHint`)**：
   - 小學：對所有螺帽標示出該落在句子的「第幾個字 (1, 2, 3...)」，成為純粹的數字排序遊戲。
   - 中學/高中：逐步剝奪提示，只留下「句首、句末」提示，其餘需靠玩家本身對詩句的認知。
2. **顏色剝奪 (`color`)**：
   - 到達最高難度（研究所）時，遊戲會啟動深色且難以辨識的顏色組 (`hard' / 'expert`)。當視覺上的顏色分類被拔除，玩家只能被迫去閱讀每一個漢字的字義，在大腦中拼湊「這三個字到底能不能組成一句話？」
3. **撤銷權 (`undo`) 與容錯率**：
   - 休閒難度可隨時悔棋（退回上一步）；研究所難度無情拔除撤銷鍵，錯一步即滿盤皆輸。
4. **混亂深度 (`totalNumberOfExchange`)**：
   - 從初學者的 16 步輕微打亂，飆升到研究所的 56 步極度混沌。打亂得越深，玩家需要使用的空槽中轉次數就越多。

### 4. 聲光回饋與 UI 設計
- **仿聲學音階**：本遊戲捨棄了錄製的音效檔，改用 HTML5 AudioContext 的 `Oscillator`（振盪器）來合成音效。拿起螺帽是清脆的高頻 (1200Hz)，放下是扎實的中頻 (600Hz)，錯誤是沉悶的低頻 (200Hz)，而完成一整柱鎖扣時會發出頻率爬升的悅耳共鳴 (1500Hz -> 3000Hz)。這個設計讓整體遊玩充滿了實體的機械敲擊美感。
- **動態鎖扣高度**：程式會自動計算當前詩句的「最長字數 (`maxLineLength`)」，並據此動態調整鎖扣 UI 的視覺高度，確保在遇到長短句混搭的宋詞時，介面依然能完美對齊。

## 第二部分：遊戲心得與分析 (Game Experience & Analysis)

### 1. 遊戲性分析：「披著休閒遊戲外皮的高壓腦力激盪」
《詩韻鎖扣》成功將枯燥的「讀詩」轉化為「整理房間」般的秩序重建快感。
在市面上的水排序遊戲中，玩家通常可以拿著手機發呆十分鐘想下一步。但在本遊戲中，那條隨時會因為你「怠工 5 秒」而扣血的紅白色計時環，就像一顆定時炸彈。
當玩家手忙腳亂地把「明」跟「月」暫存在空鎖扣上，卻突然發現主鎖扣已經塞滿了「床、前、疑、是」而導致卡死時；那種眼看著剩餘步數一格一格被怠工懲罰吃掉的窒息感，會讓人在完成拼圖的瞬間爆發出極大的成就感。

### 2. 精華推廣亮點：吸引益智與解謎狂熱者的商業細節
如果將這款遊戲作為專案的推廣重點，有幾個設計可以強烈吸引熱愛益智遊戲 (Puzzle / Brain-teaser) 的硬核受眾：

- **拔除顏色的「純文字盲解」**：
  這是其他水排序遊戲做不到的！向高階玩家展示「研究所」難度：畫面上所有螺帽都是極其相近的暗沉色系，提示數字全部消失。你不能再盲目地「把紅色疊在紅色上」，你必須真真切切地去閱讀：「喔，第一柱最下面是『春』、第二柱上面是『曉』，所以我應該把它們拔出來湊成一對」。
  這種將「空間解謎」與「語意解謎」完美結合的機制，是本作最驚豔的創舉。

- **行雲流水的批量搬移 (Multi-Move) 快感**：
  為了推廣實況效果，可以重點展示玩家如何利用高階技巧。當玩家辛苦地湊齊了「海、上、生」三個字後，不需要一個一個點，而是可以「喀啦」一聲，一鍵將這三個字連根拔起，完美鑲嵌入目標鎖扣中。配合合成器的爬升音效，這種「大清倉」的視聽回饋（Juiciness）絕對能牢牢抓住觀眾的眼球。

- **「絕對有解」的演算法承諾（逆向生成）**：
  解謎玩家最討厭「運氣成分導致的死局」。在推廣時，可以自豪地揭露我們的幕後技術：「每一次打亂，都是系統用合法步數逆向推演的結果；只要你思路清晰，這局遊戲就絕對解得開。」這種對邏輯絕對嚴謹的保證，會大幅增加硬核益智玩家挑戰的意願。

### 3. 對外推廣短語提案
> *「當顏色消失，你還能憑藉對詩句的記憶，解開這堆錯綜複雜的文字密碼鎖嗎？」*
> *「首創『怠工懲罰』的暴力水排序遊戲！發呆五秒就扣步數，這裡不容許你大腦當機！」*
> *「別被它休閒的外表騙了，在最高難度下，這是一款『就算你知道答案，也可能走進死胡同』的極限推演挑戰。」* 

### 4. 總結
《詩韻鎖扣》巧妙地將古典詩詞與現代益智機制綁定。它不要求玩家擁有極致的反應速度，而是將試煉場放在了玩家的「短期工作記憶區」。結合了防止死局的嚴謹演算法、逼死人的怠工懲罰、以及漸進式剝奪視覺提示的難度設計，本作在《花月》系列中，無疑是最具深度的邏輯腦力測試劑，也是一款不折不扣、商業潛力極大的高品質益智遊戲。
