# 《花月》Game23: 縱橫集句 - 遊戲企劃書與遊戲心得

## 第一部分：遊戲企劃書 (Game Design Document)

### 1. 遊戲概述與核心目的

**遊戲名稱**：縱橫集句 (Verses Woven in Crossword)
**命名典故**：「集句」是中國古典文學的一種創作技法——從不同詩人的詩句中各取一行，組合成一首意境相通的新詩。「縱橫」既指本作九字縱橫交叉的拼圖格局，也呼應了古代文人「縱橫捭闔、引經據典」的博學氣質。本作正是讓多首詩詞在同一張格子上縱橫相交、互相嵌合——詩與詩在交叉點共享一個字，宛如古代文人以「集句」技法將不同詩人的精華熔於一爐。

**遊戲目標**：在 9×9 的格子中，將打散的拼圖片重新組合，還原出一幅「以一句詩為主軸，多首詩詞縱橫交叉」的詩詞填字畫——每首副句詩都與主句共享一個字，形成多維度的詩詞交叉記憶網絡。

**學習目的**：《縱橫集句》在花月系列中達成了前所未有的「跨詩連結」考驗。玩家不只要記住主句詩，更要在腦海中同時持有多首不同詩詞的片段，並理解它們在哪個字上互相交會。這種「一字連兩詩」的認知訓練，正是古代詩詞學習中「融會貫通、舉一反三」的最高境界——把本來孤立的詩詞記憶，編織成一張彼此呼應的意義網絡。

### 2. 遊戲玩法與核心操作

#### A. 遊戲流程

**題目的生成（三個步驟）：**

**步驟一：建立交叉填字格局**
1. 系統選取一首詩的一行作為**主句**（五言或七言），放置於 9×9 格子的**中央軸線**（橫向主句置於第 5 行；縱向主句置於第 5 欄）
2. 針對主句的每一個字，從詩庫中搜尋**包含該字的不同詩句**作為**副句**，並將副句沿垂直軸線（若主句橫，則副句直；若主句直，則副句橫）插入對應欄/行
3. 主句字與副句在交叉點**共用同一個格子**（如主句「床」= 副句「遶**床**弄青梅」中的「床」字，兩者在同一格相交）
4. 將剩餘空格依 `decoyCharCount` 填入**干擾字**（來自同評分詩庫但不屬於任何答案句的字元）

**步驟二：切割拼圖（同 GAME22）**
將所有有字的格子（詩句字 + 干擾字）以「隨機多格形切割演算法」分割為多片拼圖（1×1、1×2、1×3、L型、1×4、2×2），維持原始切割朝向，**不旋轉**。

**步驟三：打亂並還原（同 GAME22）**
所有拼圖片取出後隨機放回 9×9 格子；無法放入的進下方暫存區。

**遊戲進行：**
5. 提示行在頂端依難度延遲顯示**主句文字**（無論主句橫或直，提示均水平顯示，如「床前明月光」）
6. 若 `showHintInGrid = true`，主句所在的橫行或直欄顯示**黃色半透明底色**，作為空間錨點
7. 玩家拖曳拼圖片，將散亂的片重新組合成原始的交叉填字格局
8. **勝利條件**：逐格讀取 9×9 格子的實際字元，所有有字格子完全符合預期格局
9. **失敗條件**：計時器歸零

**範例格局（橫向主句「床前明月光」，置於第 5 行）：**

```
     欄1  欄2  欄3  欄4  欄5  欄6  欄7  欄8  欄9
行1:  _    _    舉   安   動   _    _    _    _
行2:  _    _    杯   一   如   _    _    _    _
行3:  遶   _    邀   片   有   _    _    _    _
行4:  _    _    _    _    _    _    _    _    _
行5: [床] [前] [明] [月] [光]  _    _    _    _   ← 主句橫行（第5行）
行6:  弄   不   月   _    _    _    _    _    _
行7:  青   見   _    _    _    _    _    _    _
行8:  梅   古   _    _    _    _    _    _    _
行9:  _    人   _    _    _    _    _    _    _
（空白格依 decoyCharCount 填入干擾字後，再切割成拼圖片）
```

#### B. 操作方式

完全繼承 GAME22 的操作設計：

| 操作 | 行為 |
|------|------|
| 拖曳遊戲區拼圖片 | 移動至新位置，若目標有片則觸發自動置換 |
| 放置時目標有其他片 | 被置換片移至最近合適空位，若無空位則進暫存區 |
| 從暫存區向上拖曳 | 片以原尺寸進入遊戲區 |
| 向下拖曳至暫存區 | 片縮小進入暫存區 |
| 點擊「復原 ↩」按鈕 | 撤銷上一步（含連鎖自動置換），最多 20 步 |

**移動單位**：所有方向以**一個字元格**為單位

**按鈕佈局**（頂部，與 GAME22 相同）：
`[分數] | [↩復原] [難度] [重來] [開新局]`

#### C. 即時反饋機制

- **拖曳中**：片放大 1.05×，金色邊框 + 陰影浮起
- **片放入正確位置**：詩句字格子閃爍金色光暈（0.5秒）；干擾字格子不做特殊標示
- **交叉點正確就位**：交叉格（主副句共用字）觸發**雙色脈動閃爍**（主句色 + 副句色交替），因為這個格子同時滿足了兩個詩句的約束
- **被置換的片**：橘色閃爍 0.3 秒後自動滑移
- **全解瞬間**：片由左上向右下逐格亮起金波（每格間隔 60ms），比 GAME22 更快（9×9=81格），形成更壯觀的掃場波浪

#### B. 視覺回饋系統

| 格子類型 | 視覺表現 | 色彩 |
|---------|---------|------|
| 主句格（正確就位）| 金色底色 + 粗邊框 | `hsl(45, 80%, 50%)` |
| 副句格（正確就位）| 深翠綠輕暈 | `hsl(145, 60%, 35%)` |
| 交叉格（主副共用，正確）| 雙色脈動（金+翠綠）| 動畫 |
| 干擾字格（正確）| 無特殊標示 | 同普通片 |
| 主句行底色（`showHintInGrid` 啟用）| 黃色半透明帶 | `hsla(50,100%,70%,0.2)` |
| 拼圖片（普通）| 深色半透明底 + 2px 白色圓角邊框 | 同 GAME22 |
| 片間間隙 | 2px 空白 | 與背景同色 |

#### C. 計時器呈現方式

SVG 矩形邊框環繞整個 9×9 遊戲區，`stroke-dashoffset` 逐漸消失。剩餘 25% 時轉為鮮紅並加速閃爍。

#### D. 使用體驗的流暢度設計

- **交叉點特殊視覺標示**：交叉格（主副句相交的格子）在 `showHintInGrid = true` 時顯示**特殊小鑽石標記**（◆），提示玩家「這格是兩首詩的交會點，必須放置對的字」，幫助初學者建立填字交叉的空間概念
- **干擾字的視覺融入**：干擾字格的拼圖片外觀與詩句字片相同，使玩家無法輕易分辨；但難度越低，干擾字越少，讓初學者主要面對詩句片的組裝
- **主句提示方向指示**：提示行左側顯示小圖示：橫向主句顯示「↔」、縱向主句顯示「↕」，讓玩家即使在高難度下（`showHintInGrid = false`）也知道主句的軸向
- **暫存區片數計**：右上角顯示「暫存 N 片」即時更新，讓玩家掌握尚未歸位的片數

### 4. 遊戲機制的繼承與創新

《縱橫集句》是 GAME21《橫批成詩》與 GAME22《詩詞拼圖》的完整整合，但並非兩者的簡單疊加——而是創造了一種**前兩作都無法單獨實現的詩詞學習體驗**：

- **繼承 GAME21：填字交叉格局作為知識骨架**
  GAME21 的「最終完成畫面」正是本作的起點——多首詩詞在交叉點共享字元的格局，是本作的核心問題結構。玩家必須理解「這個格子同時屬於主句和副句」，這種雙重歸屬的認知是 GAME21 特有的挑戰。
  
- **繼承 GAME22：拼圖機制作為探索介面**
  GAME22 的切割、打亂、暫存、自動置換、復原機制原封不動地搬入本作，提供了熟悉的拼圖操作手感。玩家不再需要從頭學習新的操作方式。

- **創新一：干擾字的雙重功能（Decoy-Noise System）**
  GAME22 的格子完全填滿詩文（無噪音），本作的 `decoyCharCount` 引入了漢字噪音層。干擾字讓玩家無法靠「找空格的形狀」來定位詩句，必須靠語文記憶判斷哪些字屬於詩句、哪些是雜訊。這是花月系列中第一次將「噪音」作為難度的核心設計參數。

- **創新二：主句方向的不確定性（Orientation Ambiguity）**
  中高難度下，主句方向（橫/直）隨機決定，且 `showHintInGrid = false`。玩家拿到一段主句文字，卻不確定它是橫著還是直著擺在格子裡——必須從拼圖片的內容中推理出軸向，再倒推副句的方向。這種「先找方向、再找位置」的元認知過程，是前兩作都沒有的全新挑戰層次。

- **創新三：跨詩記憶的網絡效應（Cross-Poem Memory Network）**
  GAME21 和 GAME22 都只考驗一首詩的記憶。本作同時涉及主句詩 + 多首副句詩，玩家必須同時記住 6-9 首詩的片段，並知道它們在哪個字上相交。這種「多詩同框、共字相連」的記憶方式，是花月系列對古典文學涵養要求最高的設計。

### 5. 題目生成與格局建構演算法

#### 1. 主句選取

系統依 `poemMinRating` 從詩庫抽取一首詩，選取其中一行作為主句（五言或七言）。若設定為隨機方向，系統隨機決定本局主句為橫向或縱向。

#### 2. 副句搜尋與定位

對主句的每一個字（設為第 i 個字，放置於軸線位置 i）：
1. 搜尋詩庫中**包含該字**的其他詩句（排除主句所屬的詩）
2. 確認副句中該字的索引位置 k（1-indexed）：
   - 橫向主句（主句在第 5 行）：需要 k ≤ 5 且 (副句長度 - k) ≤ 4，確保副句完全在行 1-9 內（不超出格子）
   - 縱向主句同理
3. 若無完全符合的副句，允許截斷超出邊界的部分（只顯示在格子內的片段）
4. 從候選中優先選取「在語意或意境上與主句相關的副句」（使用詩詞評分系統過濾）

#### 3. 干擾字生成

1. 統計所有有字格子後，計算剩餘空格數量
2. 從詩庫中抽取字元（需符合以下條件）：
   - 不是主句或任何副句中的字
   - 來自評分 ≥ `poemMinRating` 的詩詞（保持文字品質）
   - 優先選擇「視覺上類似主/副句字的字元」（如主句有「月」，干擾字可選「日、星、雲」），最大化混淆效果
3. 填入至多 `decoyCharCount` 個空格（隨機選擇哪些空格填入）

#### 4. 切割與打亂（同 GAME22）

同 GAME22 的隨機多格形切割算法（以難度的 `minPieceArea` 控制切割傾向）與打亂流程。

#### 5. 勝利判定邏輯

```javascript
// 參考 Game9（詩韻鎖扣）與 Game22（詩詞拼圖）的字元讀取寫法
function checkWin(currentGrid, expectedGrid) {
    for (let row = 0; row < 9; row++) {
        for (let col = 0; col < 9; col++) {
            const expected = expectedGrid[row][col]; // 詩句字 | 干擾字 | null（空）
            if (expected === null) continue;         // 空格不檢查
            if (currentGrid[row][col] !== expected) return false; // 逐格字元比對
        }
    }
    return true;
    // 關鍵：不比對「哪片拼圖在哪個位置」，只比對「每格顯示的字是否符合」
    // 處理主/副句疊字：只要對應格子的字元正確即可，不論同字出現幾次
}
```

### 6. 音樂與音效設計

《縱橫集句》的音樂氛圍定位為「書房中整理多卷竹書」——比 GAME22 更厚重、更具儀式感，因為這裡涉及的不只是一首詩，而是多首詩相互輝映的文學空間。

**背景音樂**：低增益傳統器樂（推薦 `melodyPlayer.playFullMelody('踏雪尋梅')` 或 `'望春風'`），比 GAME22 更舒緩、更冥想性的節奏，強化「整理古典文獻」的沉浸感。

**操作音效**（繼承 GAME22）：
- 抓取片：`playOpenItem()`
- 正確放置詩句字：`playSuccessShort()`
- 自動置換觸發：`playGuzheng(index)` 低增益
- 復原操作：`playCloseItem()`

**GAME23 獨特音效：**
- **交叉點正確就位**：`playJoyfulTriple()` 三連音（比單字就位的短音更有分量），因為同時滿足兩首詩的約束是一個特別的成就
- **全解掃場音效**：金波由左上向右下逐格擴散時，音符也由低到高快速掃過（81格、每格間隔 60ms = 4.86秒全程），形成一段快速音階爆發，與視覺波浪完美同步

**學習效果的聲音設計**：交叉點的特殊音效（三連音）在玩家記憶中形成「兩首詩在這個字上相遇了」的聽覺標記，強化交叉字元的雙重歸屬記憶。

### 7. 難度參數的精密控制 (Difficulty Scaling)

| 難度 | 時限 | 詩評下限 | 主句方向 | 提示延遲 | 提示字數 | 格內標示 | 干擾字數 | 最小片格數 |
|------|------|----------|--------|---------|---------|---------|---------|---------|
| 小學 | 200秒 | ★★★★★★ | 橫向固定 | 0秒 | 全部 | ✅ 主句行 + 交叉點◆ | 0 | 4格 |
| 中學 | 160秒 | ★★★★★ | 橫向固定 | 5秒 | 5字 | ✅ 主句行（無◆）| 6 | 2格 |
| 高中 | 120秒 | ★★★★ | 隨機 | 10秒 | 3字 | ❌ | 12 | 1格 |
| 大學 | 90秒 | ★★★ | 隨機 | 20秒 | 2字 | ❌ | 18 | 1格 |
| 研究所 | **60秒** | ★★★ | 隨機 | **永不** | **0字** | ❌ | **24** | 1格 |

**質變點一（中學→高中）**：三重轉變同時發生——`showHintInGrid` 關閉（失去主句行標示）+ 主句方向開始隨機（不再確定是橫的）+ 干擾字從 6 個增加到 12 個。這是整個花月系列中難度梯度最陡的一個跨越，高中玩家突然面對「不知道主句在哪個軸向」與「格子中充滿雜訊字」的雙重挑戰。

**質變點二（大學→研究所）**：提示完全消失 + 干擾字達到 24 個（若主句為五言，則最多 56 個空格中有 24 個填入干擾字，超過 40% 的空格都是噪音）。玩家面對的格子幾乎是一張滿載陌生文字的亂碼表，必須在 60 秒內憑純粹的記憶辨認出主副句的蹤跡，並將拼圖片還原到正確位置。

**研究所的極限設計**：隨機方向（不確定橫直）+ 零提示 + 24 個干擾字 + 最小 1 格片（大量單字碎片）+ 60 秒限時。一個九字的七言主句 + 九條七言副句 = 9 + 9×6 = 63 個詩句字，加上 24 個干擾字，最多可達 87 個有字格子（超出 9×9 = 81 格，因此干擾字數量以剩餘空格為上限自動調整）。玩家在一片「字的叢林」中，憑對中國古典文學的深度記憶，完成一場最複雜的詩詞拼圖——這是「讀萬卷書」的終極體現。

## 第二部分：遊戲心得與分析 (Game Experience & Analysis)

### 1. 遊戲性分析：「這是一場在詩字的叢林中，同時追蹤九首詩蹤跡的多線記憶馬拉松」

《縱橫集句》是花月系列迄今最立體、最深邃的文學記憶挑戰。它不只要求玩家記住「一首詩」，而是要求玩家的記憶同時持有主句詩 + 多首副句詩的片段，並清楚感知它們在哪個字上發生了化學反應。

玩家的認知體驗分為四個層次：

**第一層：識別詩句片段**——拼圖片上有幾個字，玩家先從字面辨認「這是哪首詩的片段」。如果記得「遶床弄青梅」，看到「弄青梅」三個字就能立即定位這片的大致方向。

**第二層：定位交叉點**——「遶床弄青梅」的「床」必須與主句「床前明月光」的「床」在同一格相交。玩家必須在腦海中同時看到兩首詩，找到那個共用字的格子位置。

**第三層：推理軸向**——在高難度下，玩家甚至不知道主句是橫是直。他們必須從多片拼圖的字元分布，推理出整個交叉格局的軸向，再決定每片的正確位置。

**第四層：在噪音中辨真**——干擾字讓格子中充滿形似詩句的文字，玩家的文學直覺必須持續過濾「這個字是詩句的一部分還是噪音？」這種警覺性的維持，是純粹的文學素養考驗。

**頂點時刻**：研究所難度，主句方向未知、零提示、24 個干擾字，60 秒限時。玩家以閃光般的文學直覺逐片還原，最後一片就位，金波從左上以 60ms/格的速度掃過整個 9×9 格子，81 個格子逐一亮起，旋律快速音階爆發——那個「整個詩詞交叉世界完整浮現」的瞬間，是玩家用記憶還給古典文學的最大敬意。

### 2. 精華推廣亮點：足以引爆社群討論的核心魅力

- **「交叉點的雙重歸屬」挑戰的知識感**：
  當玩家把「舉杯邀明月」的「明」字片放進正確位置，它同時滿足了橫向主句「床前明月光」和縱向副句「舉杯邀明月」兩首詩的約束——此刻觸發的三連音和雙色閃爍，讓玩家切身感受到「兩首詩在這裡相遇了」。這種「文學化學反應」的瞬間，在短影音中有極高的傳播潛力。
  標題可以下：**「一個字，同時屬於兩首詩——這是古今最美的文學巧合。」**

- **「干擾字叢林」的策略突破**：
  研究所難度的格子幾乎被文字填滿，玩家面對一片看似都是詩文的混亂拼圖。當高手玩家以極快的速度從「字的叢林」中精準識別詩句片段、忽略干擾字，那種「見山是山、見字是詩」的瀟灑眼力，在旁觀者眼中如同展示一種超能力。
  標題可以下：**「81 個格子，24 個假字，60 秒——他一眼就知道哪些字是詩、哪些是干擾。」**

- **「縱橫知多少」的驗身挑戰**：
  以同一首詩的不同難度（小學橫向無干擾 vs. 研究所隨機方向滿干擾）製作「難度對比」短影音。同一首詩、同一拼圖，小學版溫馨易解，研究所版讓大學教授也皺眉。這種「同題不同世界」的對比，能精準傳達本作的難度縱深。

### 3. 對外推廣短語提案

> *「這不是一首詩的拼圖——這是九首詩同時開始相遇的那張格子。你能不能在九九八十一個字的洪流中，找出所有相交的那些字，讓它們各就各位？」*
> *「小心那些不屬於任何詩的干擾字！它們長得跟詩一樣，就等著你把拼圖放錯——研究所難度下，每三個字就有一個是騙你的假字。」*
> *「一個字同時住在兩首詩裡。當你把它放進正確的格子，你會聽到一個聲音——那是兩首相隔千年的詩，在同一個字上，第一次相遇的聲音。」*

### 4. 總結

《縱橫集句》以「多詩交叉填字 × 拼圖組裝 × 干擾字噪音」的三維設計，達成了花月系列前所未有的文學記憶深度。它繼承了 GAME21 的交叉詩格結構與 GAME22 的拼圖操作體驗，並以「干擾字數量」作為全新的難度調節維度——從 0 個干擾的純粹詩境，到 24 個干擾的文字叢林，提供了完整的難度光譜。主句方向的不確定性讓高階玩家必須從碎片中重建軸向，是花月系列中最具「偵探感」的認知挑戰；而交叉點的雙重歸屬設計，則讓「一字連兩詩」成為整個系列中最具文學美感的遊戲瞬間。在 60 秒內，憑藉對中國古典詩詞的完整記憶，從文字的叢林中重建一幅縱橫交叉的詩詞宇宙——這不是遊戲，這是一場屬於真正讀過詩的人的記憶盛典。
