# 《花月》Game20: 丟三落一 - 遊戲企劃書與遊戲心得

## 第一部分：遊戲企劃書 (Game Design Document)

### 0. 前身遊戲分析：從 Game1《慢思快選》的進化

#### A. 前身遊戲玩法解析（Game1 慢思快選）

《慢思快選》以「字元級填空」為核心：從詩的兩句中隨機遮蔽數個字元（以「◎」替代），玩家從四個選項中選出包含正確字元的完整句子。核心爽點在於「閃光的頓悟感」——在密集的遮罩中依靠殘餘字詞推理出正確答案，如同在霧中辨認人影。

**前身的局限性**：Game1 的答題粒度是「字元」——玩家在選項中真正需要辨別的只有那幾個被遮蔽的字。這意味著即使玩家只認識一兩個遮蔽字，也可能靠排除法過關，對整句詩意的掌握未必深入。

#### B. 前身的介面設計與吸引力

Game1 的 2×2 四宮格按鈕是它最具識別度的視覺語言——整齊對稱、每格足夠大、易於手指點擊。計時器矩形邊框環繞整個答案區的設計，讓「時間消逝」化為視覺上四邊緩慢收縮的壓迫感，比數字倒計時更具戲劇性。

值得在 Game20 繼承的元素：矩形計時框的壓迫感設計、詩詞出處的點擊跳轉功能、答題後的飛星結算動畫。

#### C. 詩詞融合分析：從「字元級」升級為「句子級」

| 問題 | Game1 的答案 | Game20 的升級 |
|------|------------|-------------|
| **什麼被替換** | 句子中的少數字元 | 整整一句詩，完全消失 |
| **語文知識門檻** | 知道那幾個字即可 | 必須理解上下句的語意流動與對仗關係 |
| **詩詞學習效果** | 字序記憶、字形辨識 | 句意記憶、詩篇結構理解、上下文推理能力 |

Game20 的設計理念：**讓玩家無法靠字形配對取巧，必須真正「讀懂」這首詩的脈絡，才能在多個完整句子的選項中選出最合理的那一句。**

#### D. 手機觸控適配：矛盾點與解決方案

- **答案選項為完整詩句**：採縱向單欄排列，每顆按鈕橫向全寬，字級依文字長度動態縮放；按鈕最小高度 44px 以上符合觸控標準。
- **三句格式空間競爭**：題目區字級自適應，隱藏句以灰色虛線佔位欄呈現（不填入「◎」字符雜訊）。
- **多選一時誤觸**：選項間距 4px、按鈕邊框與背景在 hover 時加亮，給予明確的視覺確認。

### 1. 遊戲概述與核心目的

**遊戲名稱**：丟三落一 (Missing from the Sequence)
**遊戲目標**：從詩詞中截取 2 至 3 句，其中一句完全隱藏，玩家須從 4 至 8 個選項（皆為完整詩句）中選出正確的那一句，填回它在詩中消失的位置。
**學習目的**：Game1《慢思快選》考驗的是「字元層級的記憶精度」，而《丟三落一》將挑戰提升至「句子層級的語境感知」。玩家必須真正理解「上一句說了什麼、下一句應該如何接續、中間那句的語氣轉折是否合理」——這正是古典詩詞理解的核心能力：掌握詩篇的**句義流動**與**對仗韻律**。

---

### 2. 遊戲玩法與核心操作

#### A. 遊戲流程

1. **選擇難度** → 系統依難度抽取符合條件的詩詞，根據格式參數決定本局採用哪種出題模式
2. **題目生成**：系統從詩詞中截取連續的 2 或 3 句，隱藏其中 1 句，並依難度對可見句施加局部字元遮蔽
3. **題目呈現**：
   - 題目區顯示可見的 1-2 句（可能含有若干「◎」遮蔽字）
   - 隱藏句以**灰色虛線佔位欄**標示，標明隱藏位置（「猜上句」、「猜下句」或「猜中句」）
4. **作答**：玩家從答案區的 4-8 個選項中點擊自認正確的句子
5. **判定**：
   - **正確**：佔位欄填入正解句，高亮顯示，計分，自動載入下一題
   - **錯誤**：錯誤計數 +1，紅心減少，錯誤選項標紅鎖定
6. **勝利條件**：在時間內選出正確答案
7. **失敗條件**：錯誤次數超過上限 **或** 計時器歸零

**三種出題格式：**

| 格式代號 | 可見句 | 隱藏句位置 | 記憶考驗方向 |
|---------|-------|-----------|------------|
| **A：兩句格式** | 第 2 句 | 第 1 句（上句） | 看後句猜前句：理解對仗關係 |
| **B：三句格式（猜下句）** | 第 1+2 句 | 第 3 句（下一聯首句） | 看前兩句猜接下來：理解詩意發展 |
| **C：三句格式（猜中句）** | 第 3+4 句 | 第 2 句（上一聯末句）| 看後兩句猜中間：逆向推理詩脈，最難 |

#### B. 操作方式

| 平台 | 選擇答案 |
|------|---------|
| **桌機（滑鼠）** | hover 高亮 → click 判定 |
| **手機（觸控）** | pointerdown 高亮 → pointerup 判定 |

**操作限制**：答錯選項標紅後不可再選（鎖定）。

#### C. 即時反饋機制

- **hover / pointerdown**：按鈕邊框金色、輕微放大
- **答對**：按鈕轉深翠綠 → 佔位欄填入正解句同步綠色顯示 → 播放 `playSuccess()` → 自動進入勝利動畫
- **答錯**：按鈕轉朱紅 + `shake` 抖動 → 播放 `playFailure()` → 紅心減少動畫
- **超時 / 失誤過多**：⚠️ **不揭曉正解**（避免劇透打斷學習動機）。題目區與選項區維持原樣但停用，等玩家自行「重來」

#### D. 「重來」與「開新局」差異

| 行為 | 重來（retryGame） | 開新局（startNewGame） |
|------|-------------------|-----------------------|
| 題目 | 保留同一題 | 重新抽取 |
| 選項位置 | **重新洗牌**（避免靠記憶上次答錯按鈕的位置取巧） | 重新生成並洗牌 |
| 分數 | 歸 0 重算 | 歸 0 重算 |
| 紅心 | 重置 | 重置 |

### 3. 介面設計與使用者體驗

#### A. 主介面佈局

```
┌─────────────────────────────────────┐  ← 頂部資訊列
│  分數: 0        [小學][重來][開新局]  │
├─────────────────────────────────────┤  ← 生命值列
│            ♥ ♥ ♥ ♥                  │
├─────────────────────────────────────┤
│  詩名 / 朝代 / 作者（點擊開詩詞頁）  │
│  ┌─────────────────────────────┐   │
│  │  [可見句1]  床前明月◎       │   │  ← 題目區
│  │  ── 灰色虛線佔位欄 ──        │   │
│  │  「↑ 猜前一句 / ↓ 猜下一句」 │   │
│  │  [可見句2（三句格式才有）]    │   │
│  └─────────────────────────────┘   │
│  ┌──────────────────────────────┐  │  ← 答案區
│  │  [SVG Timer Ring 邊框]        │  │
│  │  ┌──────────────────────┐    │  │
│  │  │ 答案選項 1（完整詩句）  │    │  │
│  │  │ … 4-8 顆依難度          │  │
│  │  └──────────────────────┘    │  │
│  └──────────────────────────────┘  │
└─────────────────────────────────────┘
```

**隱藏句佔位欄的語言提示**：
- 格式 A（猜上句）：「↑ 猜前一句」，位置在可見句上方
- 格式 B（猜下句）：「↓ 猜下一句」，位置在兩可見句下方
- 格式 C（猜中句）：「？ 猜中間句」，夾在兩可見句中間

#### B. 視覺回饋系統

| 狀態 | 視覺表現 |
|------|---------|
| 可見句（無遮蔽）| 白色文字 |
| 可見句（遮蔽字 ◎）| 金橙色 `hsl(36, 80%, 55%)` |
| 隱藏句佔位欄 | 灰色虛線框 + 脈動動畫提示 |
| 答案按鈕（normal）| 米色背景、深褐字 |
| 答案按鈕（hover）| 金色邊框、`scale(1.02)` |
| 答對 | 深翠綠背景 |
| 答錯（鎖定）| 朱紅背景 + `shake` 抖動 |
| 失敗時 | 選項全部停用但**不揭曉正解** |

#### C. 計時器呈現

SVG 矩形邊框環繞整個答案區四邊，`stroke-dashoffset` 動畫讓邊框由暗紅漸進至鮮紅後完全消失。

### 4. 遊戲機制的繼承與創新

- **句子級填空取代字元級填空（核心升級）**：Game1 隱藏的是「句中的部分字元」，Game20 隱藏的是「整句詩的存在」，選項全部是完整的詩句。
- **三種出題格式的混合挑戰**：A / B / C 三格式考驗截然不同的認知路徑，防止玩家養成單一解題習慣。
- **可見句的局部遮蔽作為難度調節器**：在「提示句」也施加 ◎ 遮蔽，產生「霧中找霧」的雙重不確定性。
- **選項數量擴張至 8 選 1**：從小學 4 選 1 到研究所 8 選 1，接近真實閱讀理解測驗的資訊密度。
- **每句保底 1 個非遮蔽字**：`applyVisibleMask` 內建 `maxPossible = validIndices.length - 1`，避免整句被掩蓋無線索。
- **失敗不劇透正解**：強化「重來再試」的學習迴圈，玩家可在保留題目的情況下再次思考。

### 5. 題目生成的精密設計

#### 1. 詩句截取邏輯

呼叫共用的 `getSharedRandomPoem()` 取得符合難度評分的詩詞後，依格式決定截取方式：
- **格式 A**：抽取連續 2 句（奇數句+偶數句），偶數句可見、奇數句隱藏
- **格式 B**：抽取連續 3 句，前兩句可見、第 3 句隱藏
- **格式 C**：抽取連續 4 句，後兩句可見、第 2 句隱藏（第 1 句僅作詩意錨點，不顯示）

#### 2. 可見句的字元遮蔽

- 隨機選取指定數量的字元位置替換為「◎」，標點符號不遮蔽
- ⚠️ **保底機制**：每句至少留 1 個非標點字不遮蔽（`maxPossible = validIndices.length - 1`），避免整句被掩蓋

#### 3. 干擾選項的生成策略

選項中除正解句外，其餘均為「完整詩句干擾項」，三段式來源：
- ① 同首詩的其他句子（強干擾，字數需相同）
- ② 同朝代或同作者、字數相同
- ③ 全題庫遍歷補滿（字數相同）
- ④ 最後保底：放寬字數限制

#### 4. 防重複機制

同一局選項彼此不重複（`usedTexts` Set）；題目中已出現的可見句不會出現在選項裡。

### 6. 音樂與音效設計

**背景音樂**：答題進行中不播放背景音樂，強化閱讀詩句時的專注環境。

**操作音效**：
- 開啟遊戲 / 選擇難度：`playConfirmItem()`
- 點擊重來：`playOpenItem()`

**正確/錯誤回饋音**：
- **答對**：`playSuccess()`——向上三連音
- **答錯**：`playFailure()`——低沉下行音 + 按鈕 `shake` 抖動同步

**勝利結算**：通關後觸發 `ScoreManager.playWinAnimation()`，配合全螢幕飛星動畫。

### 7. 難度參數的精密控制 (Difficulty Scaling)

⚠️ 以下為現行 `difficultySettings`（含使用者最新調整）：

| 難度 | 時限 | 詩評下限 | 錯誤容忍 | 出題格式 | 可見句遮蔽字數 | 答案選項數 |
|------|------|----------|----------|---------|--------------|---------|
| 小學   | 30 秒 | ★★★★★★ | 3 次 | A | 0 字 | 4 選 1 |
| 中學   | 25 秒 | ★★★★★ | 3 次 | A or B | 0-2 字 | 5 選 1 |
| 高中   | 20 秒 | ★★★★ | 2 次 | B | 1-4 字 | 6 選 1 |
| 大學   | 15 秒 | ★★★ | 2 次 | B or C | 3-5 字 | 7 選 1 |
| 研究所 | **10 秒** | ★★★ | **1 次** | **B or C** | **4-6 字** | **8 選 1** |

**質變點一（中學→高中）**：固定 B 格式（看前兩句猜下一句），且選項數提升到 6。
**質變點二（高中→大學）**：B 與 C 隨機交替；選項提升到 7。
**研究所的極限設計**：B/C 隨機；8 選 1；10 秒內僅 1 次容錯。

### 8. 計分設定（scoreManager.gameSettings.game20）

```
{ base: 100, heart: 50, time: 3, getPointA: 0 }
```

- `base = 100`：基礎通關分
- `heart = 50`：每顆剩餘紅心 +50 分（重心；獎勵零失誤通關）
- `time = 3`：每秒剩餘 +3 分
- `getPointA = 0`：本遊戲為單題決勝，無連續得分

## 第二部分：遊戲心得與分析 (Game Experience & Analysis)

### 1. 遊戲性分析：「這是一場在詩詞語境中拼回失落拼圖的記憶偵探遊戲」

《丟三落一》是 Game1《慢思快選》的精神繼承者，卻又在體驗深度上完成了一次質的跨越。

Game1 讓玩家感受到「快速認字的爽感」；Game20 讓玩家感受到「真正讀懂一首詩的頓悟感」。當玩家面對格式 C（猜中句）的題目，看著「後兩句」苦思「那一句中間發生了什麼」，這種推理過程幾乎等同於文學批評中的「細讀（Close Reading）」。

心理狀態的三段式變化：
1. **輕鬆掃描**（小學）：看到熟悉的詩句在選項中，快速點擊
2. **謹慎推理**（高中格式 B）：看到前兩句，大腦開始「預測詩的走向」
3. **逆向重建**（研究所格式 B/C）：8 個選項密密麻麻、10 秒倒數、僅 1 次容錯，必須以破碎上下文逆向推理

**失敗的處理哲學**：失敗時系統**不揭曉正解**，題目維持原樣但停用——這是刻意設計，讓玩家在按下「重來」時帶著「我還記得幾個遮蔽字、隱藏句方向、答案大概在某幾個選項裡」的殘餘記憶再次挑戰。失敗即下次成功的階梯，而非終點訊息。

### 2. 精華推廣亮點

- **「格式 C 逆向猜中句」的燒腦挑戰**：看到後兩句、猜中間那句，是反直覺的閱讀順序顛覆。
- **「8 選 1 + 10 秒」的速度極限**：研究所難度的視覺壓迫感極強，最適合社群短影音題材。
- **「可見句也被遮蔽」的雙重不確定性**：高難度下用來推理的可見句自己也有 4-6 字被「◎」遮住。

### 3. 對外推廣短語提案

> *「整整一句詩消失了——你能從 8 個相似的句子裡，找出那句本來就應該在這裡的話嗎？」*
> *「你看到的是後兩句，你要猜的是中間那句——當你覺得自己找到了，看看那 7 個也很像的干擾選項。」*
> *「10 秒、8 選 1、零容錯。這不是在考你背過多少詩，是在考你有沒有真正讀懂它。」*

### 4. 開發實作紀錄與已知優化

開發過程中累積的關鍵修正：

1. **遮蔽保底**（`applyVisibleMask`）：早期版本允許整句全遮蔽，玩家失去任何線索，已加入「至少留 1 字」的保底邏輯。
2. **失敗不揭曉**：早期版本會在失敗時用藍色高亮正解、用綠色填佔位欄，但此設計反而打斷學習迴圈。現改為失敗時題目區與選項全部保留但停用，重來時可再次思考。
3. **重來時選項洗牌**：早期版本重來不洗牌，玩家可記住「上次答錯按鈕的位置」直接排除。已修正為每次 `retryGame` 都呼叫 `shuffleInPlace`。
4. **計分配比**：`heart=50, time=3` — 紅心比重特別高，獎勵零失誤通關玩家。

### 5. 總結

《丟三落一》從 Game1 的「字元層級記憶」推進到「句子層級語境推理」，不只是難度的提升，而是認知維度的轉換——從「這幾個字是什麼」到「這一句詩為什麼是這樣」。三種出題格式的混合運用、可見句局部遮蔽與選項數量的協同調節，加上「失敗不劇透」的學習設計，讓研究所難度成為整個花月系列中最接近「真正理解詩詞」的終極考驗。
