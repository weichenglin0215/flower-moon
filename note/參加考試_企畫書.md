# 參加考試（考棚）企畫書 · v2

> 版本：v2  |  日期：2026-07-01  |  歸屬模組：`exam.js`（新增）、`collection.js`（既有考棚彈窗整合）、`achievement.js`（總覽入口 + 領獎狀）、`fmCollectionSave.js`（新增失敗次數欄位）、`supabaseClient.js`（雲端同步）

---

## 一、核心設計理念

**「積分」是門票，「考試」是試煉，「領獎狀」才是正式冊封。**

從「縣案首」起，玩家不再因積分達標自動升等。取得文位必須依序完成四步驟：

1. 累積積分達標 → 於「成就與紀錄 → 總覽」出現「參加考試」CTA
2. 存夠文錢 → 於「江南小院 → 考棚」報名，付費進入考試
3. 通過考試 → 於「成就與紀錄 → 總覽」出現「領取獎狀」CTA
4. 領取獎狀 → 正式取得該文位，同時給予額外積分＋文錢獎勵

失敗不可恥，是**光榮的積累**：雲端記錄每個文位的參加次數與失敗次數，玩家可對朋友驕傲地說「這是我第 137 次挑戰進士，終於中了」。

---

## 二、規則總則

| 情境 | 顯示文位 | 需通過考試？ | 需領獎狀？ |
|------|---------|-------------|-----------|
| 積分 < 縣案首（80,000） | 依積分自動升等（書僮～童生） | ❌ | ❌ |
| 積分 ≥ 縣案首但未通過考試 | 顯示「童生（可考縣案首）」 | ✅ | — |
| 已通過考試但未領獎狀 | 仍顯示上一個已領文位 | ✅ 已完成 | ⏳ 待領 |
| 已通過考試且已領獎狀 | 顯示該文位 | ✅ | ✅ |

**核心原則：**
- 每個文位僅需通過一次；已取得者不可重玩（灰化）
- 考試遊戲本身**不給積分、不給文錢、不觸發詩詞紀錄**；獎勵集中在「領取獎狀」時發放
- **失敗次數是玩家的驕傲勳章**，永久保留、雲端同步

## 三、考試遊戲規格（`exam.js`）

### 3.1 遊戲流程
- **題數**：連續 10 題
- **共用心（Heart）**：全 10 題共用一組心，數量依難度不同；用完即為「落榜」
- **答對節奏**：答對 → 播放倒數框動畫 → **立刻**跳出下一題
- **答錯**：扣一顆心
- **時間**：每題有獨立倒數，時間到視同答錯
- **通關條件**：10 題全結束且心 > 0

### 3.2 題型
類似「丟三落一」game20：每題呈現一句缺字的詩，選出正確漏字。題庫依考試等級 `poolMinRating` / `poolMaxRating` 過濾。

### 3.3 難度分級 (`difficultySettings`)

11 個等級各自對應一個 `difficultySettings` 物件（新增 `poolMaxRating`，避免出現難度過低的詩句）：

```javascript
difficultySettings: {
    '縣案首': { hearts: 5, secPerQ: 12, poolMinRating: 6, poolMaxRating: 7, ... },
    '府案首': { hearts: 5, secPerQ: 11, poolMinRating: 6, poolMaxRating: 7, ... },
    '文童':   { hearts: 4, secPerQ: 10, poolMinRating: 5, poolMaxRating: 7, ... },
    '秀才':   { hearts: 4, secPerQ:  9, poolMinRating: 5, poolMaxRating: 6, ... },
    '舉人':   { hearts: 3, secPerQ:  9, poolMinRating: 4, poolMaxRating: 6, ... },
    '貢士':   { hearts: 3, secPerQ:  8, poolMinRating: 4, poolMaxRating: 5, ... },
    '進士':   { hearts: 3, secPerQ:  7, poolMinRating: 3, poolMaxRating: 5, ... },
    '探花':   { hearts: 2, secPerQ:  7, poolMinRating: 3, poolMaxRating: 4, ... },
    '榜眼':   { hearts: 2, secPerQ:  6, poolMinRating: 2, poolMaxRating: 4, ... },
    '狀元':   { hearts: 2, secPerQ:  6, poolMinRating: 2, poolMaxRating: 3, ... },
    '大儒':   { hearts: 1, secPerQ:  5, poolMinRating: 1, poolMaxRating: 2, ... }
}
```

隨等級提升：**心數減少、答題時間縮短、詩詞評分門檻提高、缺字選項難度提高**。

### 3.4 結局分支

#### A. 通過（10 題結束時心 > 0）
1. 播放 `SoundManager.playJoyfulTriple()`
2. 顯示「金榜題名」動畫彈窗（**非** 獎狀動畫，僅告知考試通過），文字如：
   - 「金榜題名！第 XX 次挑戰終得中式！」
   - 「請至『成就與紀錄』領取獎狀，正式冊封文位。」
   - 按鈕：「返回江南小院」
3. 寫入 **`fmCollectionSave.data.ranks.passed`**（陣列 push 該文位名，供 UI 判定「可領獎狀」）
4. 寫入 **`fmCollectionSave.data.examStats[<文位>].passCount++`**
5. 寫入 `game_logs`（`isWin: true`），記錄參加並通過
6. **不觸發** `AchievementDialog.showCert`（獎狀動畫延至玩家至成就頁領取時再播放）
7. **不寫入** `playerData.achievements.claimed`（正式冊封由玩家在成就頁「領取獎狀」時才發生）
8. 點擊「返回」→ 關閉考試 → 回到「江南小院」

#### B. 落榜（心 = 0 或 10 題未完成）
1. 播放 `SoundManager.playSadTriple()`
2. 彈出 `GameMessage`（失敗態）
   - 標題：隨機勵志語（例：「柳暗花明又一村」、「捲土重來未可知」）
   - 說明：「本次未第。這是您第 XX 次挑戰『XX』，天將降大任，此不足懼。學海無涯，再接再厲。」
   - 按鈕：「確認」
3. 寫入 **`fmCollectionSave.data.examStats[<文位>].failCount++`**
4. 寫入 `game_logs`（`isWin: false`），保留失敗紀錄
5. 立即同步 `examStats` 至雲端（`supabaseClient.saveGameToCloud`）
6. 點擊「確認」→ 關閉考試 → 回到「江南小院」
7. **不扣心（不影響其他遊戲）**、**入場費已於進入前扣過**

### 3.5 UI 佈局
沿用 game20 的 500×850 舞台結構，替換以下元素：
- **頂端 diff-tag**：顯示「第 XX 次挑戰 · 縣案首」等，凸顯玩家是第幾次參加該文位的考試，配色使用 `collection.js` 的文位色系
- **題號指示器**：`3 / 10`（畫面上方）
- **心的圖示**：10 題共用，橫排顯示於題號右側
- **主要顯示區**：詩句 + 缺字選項
- **底部倒數框**：每題獨立

### 3.6 明確不做
- **不計分**、**不呼叫 `saveScore`**
- **不記錄詩詞遊玩次數**（不呼叫 `recordPoemPlay`）
- **不觸發關卡挑戰進度**（不呼叫 `completeLevel`）
- **不顯示得分表演動畫**（無星星飛舞、無難度倍率捲動）
- **不在通過時彈獎狀**（獎狀留給玩家去成就殿堂領取）

## 四、四步入口設計（依規則順序排列）

### Step 1：「成就與紀錄 → 總覽」— 積分達標 → 顯示「參加考試」

**顯示條件**：
```
nextRank.name ∈ EXAM_RANKS_ORDER
且 totalScore >= nextRank.minScore
且 nextRank.name ∉ fmCollectionSave.data.ranks.passed
```

**位置**：`#achNextRankInfo` 區塊內，於下一階級名稱旁新增：

```html
<button id="achBtnGoExam" class="ach-btn-small ach-exam-cta">
    參加考試（第 <span class="exam-attempt-no">N</span> 次挑戰）
</button>
```

其中 `N = examStats[nextRank.name].failCount + examStats[nextRank.name].passCount + 1`。

**點擊行為**：
```javascript
btnGoExam.onclick = () => {
    if (window.SoundManager) window.SoundManager.playConfirmItem();
    this.hide();
    if (window.CollectionDialog) window.CollectionDialog.show();
    if (window.Collection && typeof window.Collection.openExam === 'function') {
        setTimeout(() => window.Collection.openExam(), 300);
    }
};
```

**視覺提示**：按鈕加上與「領取獎狀」相同的呼吸動畫紅圈（`.claim-pending::after`）。

---

### Step 2：「江南小院 → 考棚」— 積分＋文錢皆足 → 進入考試

**位置**：`collection.js` 現有的 `openExam()` 彈窗。

**顯示內容**：
- 目前積分
- 下一文位名稱、積分門檻
- **本次為第 N 次挑戰**（`examStats[rank].failCount + passCount + 1`）
- 歷史紀錄：`通過 P 次 / 失敗 F 次`（低調呈現於角落）
- 入場費（文錢）
- 若積分/文錢皆足 → 顯示「報考 XX（扣 N 文錢）」按鈕
- 否則 → 灰化按鈕並提示「積分不足」或「盤纏不足」

**`takeExam(rank)` 改寫：**
```javascript
takeExam: function (rank) {
    const fee = EXAM_FEES[rank.name];
    if (this.data.silver < fee) { this.showToast('盤纏不足'); return; }
    this.data.silver -= fee;
    FMCollectionSave.save(this.data);
    this.hidePopup();
    this.hide();

    if (window.Exam && typeof window.Exam.start === 'function') {
        window.Exam.start(rank, {
            onPass: () => {
                // Exam 已完成：寫 ranks.passed、examStats.passCount++、game_log
                //             並顯示金榜題名彈窗
                if (window.CollectionDialog) window.CollectionDialog.show();
            },
            onFail: () => {
                // Exam 已完成：寫 examStats.failCount++、game_log、雲端同步
                if (window.CollectionDialog) window.CollectionDialog.show();
            }
        });
    }
}
```

---

### Step 3：「成就與紀錄 → 總覽」— 通過考試後 → 顯示「領取獎狀」

**顯示條件**：
```
rank ∈ fmCollectionSave.data.ranks.passed
且 rank ∉ playerData.achievements.claimed（即 rank_<文位名> 不在其中）
```

**位置**：同 Step 1 的位置（`#achNextRankInfo` 區塊），按鈕文字與行為切換：

- 按鈕文字：`領取獎狀`（配上呼吸紅圈 `.claim-pending`）
- 點擊 → 呼叫 `AchievementDialog.claimAchievementReward('rank_<文位名>', certImg, certText)`
  - 沿用既有領獎流程：加入 `claimed` → `totalScore += 10000` → `silver += 100` → 播放獎狀動畫

---

### Step 4：領取獎狀 → 正式冊封文位

由既有 `claimAchievementReward` 完成：
- 寫入 `playerData.achievements.claimed`（`rank_<文位名>`）
- 積分 +10,000 / 文錢 +100
- 播放獎狀動畫（`showCert(imgUrl, text, true, 10000, 100)`）
- **從此** `getEffectiveRank(data)` 才會回傳該文位

## 五、資料模型異動

### 5.1 `fmCollectionSave.data.ranks.passed` — 沿用既有欄位
陣列，記錄玩家通過考試的文位名稱。已存在。

### 5.2 `fmCollectionSave.data.examStats` — **新增**（本次要求核心）

```javascript
examStats: {
    '縣案首': { passCount: 0, failCount: 0, lastAttemptTs: 0 },
    '府案首': { passCount: 0, failCount: 0, lastAttemptTs: 0 },
    '文童':   { passCount: 0, failCount: 0, lastAttemptTs: 0 },
    '秀才':   { passCount: 0, failCount: 0, lastAttemptTs: 0 },
    '舉人':   { passCount: 0, failCount: 0, lastAttemptTs: 0 },
    '貢士':   { passCount: 0, failCount: 0, lastAttemptTs: 0 },
    '進士':   { passCount: 0, failCount: 0, lastAttemptTs: 0 },
    '探花':   { passCount: 0, failCount: 0, lastAttemptTs: 0 },
    '榜眼':   { passCount: 0, failCount: 0, lastAttemptTs: 0 },
    '狀元':   { passCount: 0, failCount: 0, lastAttemptTs: 0 },
    '大儒':   { passCount: 0, failCount: 0, lastAttemptTs: 0 }
}
```

**用途：**
- CTA 顯示「第 N 次挑戰」
- 江南小院考棚顯示歷史成績
- 未來可作為勳章基礎：「破釜沉舟（失敗 10 次終得中）」、「一鼓作氣（一次通過）」

**遷移：** `FMCollectionSave.migrate()` 補上預設空物件。

### 5.3 雲端同步（`supabaseClient.js`）

`examStats` 是江南小院 (`fmCollectionSave`) 的資料，同步策略沿用該系統：
- 每次寫入後立即呼叫 `saveGameToCloud`（或收集系統自己的雲端同步 API）
- 玩家登入時 `syncOnStartup` 從雲端讀回 `examStats`，覆蓋本機
- 若雲端資料不存在 → 使用 `FMCollectionSave.defaultData().examStats` 初始化

**Supabase 資料表欄位建議：**
- `player_saves.exam_stats` (jsonb) — 儲存整個 `examStats` 物件
- 或若走既有 `collection_saves` 存檔表 → 加入 `exam_stats` 欄位

### 5.4 `playerData.achievements.claimed` — 沿用；規則變更
- **通過考試時「不寫」** `rank_<文位名>`（原 v1 設計錯誤，v2 修正）
- **領取獎狀時才寫入**，觸發積分+10,000、文錢+100
- 避免重複觸發 CTA 按鈕的判定改為：檢查 `ranks.passed` 與 `claimed` 兩者的差集

### 5.5 「文位顯示」判定 — `getEffectiveRank(playerData)`

**新增函式** `ScoreManager.getEffectiveRank(data)`：
- 書僮～童生階段（積分 < 80,000）：以積分推算
- 縣案首以上：找出 `claimed` 中所有 `rank_<XXX>` 對應的文位，回傳排序最高者
- 若 claimed 內尚無任何文位獎狀：回傳「童生」（即便積分達到縣案首也如此）

**替換點：**
- `renderData` 的 `achRankView` 顯示
- 群英榜、江南小院 HUD、任何顯示文位的位置
- **保留** `getCurrentRank(score)` 供「積分是否達標」的邏輯判定使用

- **格式 A**（2 句）：顯示第 2 句，猜第 1 句

- **格式 C**（4 句）：顯示第 3+4 句，猜夾在中間的第 2 句

隨等級提升，可見句被遮蔽的字數增加、格式從單純的 A 演進到 B/C 逆推。實作 `_applyVisibleMask()` 為 game20 版本的精簡：遮罩位置採「前 N 或後 N」隨機（比 game20 依詩句字數的多分支更清爽）。
