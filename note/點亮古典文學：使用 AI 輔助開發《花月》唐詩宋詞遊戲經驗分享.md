## 講稿內容

### 1. 開場 (5分鐘)

大家好！今天非常高興能站在這裡，和各位分享我與 AI 協作開發的專案《花月》——一款以唐詩宋詞為主題的網頁遊戲。這是我第一次使用 AI 作為輔助來撰寫遊戲程式碼。對許多剛接觸程式設計或是想要利用 AI 提升效率的朋友來說，這段旅程充滿了驚喜，當然也有不少挑戰。在接下來的一小時裡，我將帶大家走過這幾個月的心路歷程，希望能為各位的 AI 開發之路提供一些實質的幫助。

### 2. 為何選擇唐詩宋詞？(10分鐘)

首先，大家可能會好奇，為什麼要挑選「唐詩宋詞」作為遊戲題材？
在資訊爆炸的時代，我們每天接觸大量的碎片化訊息。我希望能夠打造一款遊戲，不僅具備娛樂性，還能讓玩家在遊玩的過程中，不自覺地感受到東方古典文學的美和意境。唐詩宋詞的韻律、對仗、意象，本身就是一種極高密度的資訊壓縮。透過遊戲的互動性，我們可以將死板的文字轉化為生動的體驗，讓年輕一代或是偶爾想放鬆的玩家，能在這片刻的寧靜中找回對古典文學的感動。當然，這對程式邏輯和資料結構來說，也是一個非常有趣的挑戰。

### 3. 遊戲機制的轉化與創新 (10分鐘)

在《花月》中，大家可能會發現，部分遊戲類型似曾相識。沒錯，我們借鑑了一些經典的機台遊戲或是大家熟悉的益智遊戲機制（比如：翻牌記憶、消除遊戲、接水管等概念）。

**但重點在於：如何與唐詩宋詞「高度結合」？**
如果只是單純把詩詞的文字貼在上面，那只是一款換皮遊戲。我們的做法是將「詩詞的結構」作為核心機制。例如：
- **字的順序：** 根據五言絕句或七言律詩的平仄與結構設計消除或排列的規則。
- **意境的結合：** 將詩中的「春花」、「秋月」等意象具象化為遊戲內的視覺特效或過關條件。
- **難度的控制：** 利用詩詞的冷熱門程度或是字詞的生僻程度，來設計遊戲的解謎難度，讓玩家在卡關時也能反覆咀嚼詩句的含義。

### 4. 與 AI 協作：溝通、撰寫與維護 (15分鐘)

接下來進入重頭戲：我們是如何與 AI 溝通，寫出高效率且容易維護的程式碼的？

**以中文語言跟 AI 溝通的藝術**
很多人以為要寫好程式一定要用英文，但在這幾個月的開發中，我發現只要中文表達足夠精確，AI 是能夠完全理解並產出高品質程式碼的。
- **關鍵一：釐清「需求」而非直接給「解法」。** 新手常犯的錯誤是叫 AI「在這裡加一個 if」，而比較好的做法是告訴 AI「當玩家點擊錯誤詩詞時，我希望字體震動並扣分。請幫我設計一個高效的函式」。
- **關鍵二：分階段實作。** 不要一次塞給 AI 一整個系統的要求。先請 AI 規劃資料結構，確認沒問題後，再寫邏輯運算，最後再處理 UI 渲染。這能大幅降低 AI 產生幻覺（Hallucination）或雜亂程式碼的機率。
- **關鍵三：要求註解與模組化。** 在提示詞中加入：「請為每個重要函式加上中文註解，並盡可能將重複邏輯抽離為獨立模組。」這是保持程式碼容易維護的關鍵。

回顧這幾個月的對話重點，我們的對話模式已經從早期的「請幫我寫...」進化到「這是目前的架構，如果要增加某個功能，你會建議怎麼實作？」AI 現在已經是我強大的共創夥伴。

### 5. Agent Skills 的強大威力 (10分鐘)

要讓 AI 成為真正的專案夥伴，除了即時對話，更需要長期記憶與規範。這就不得不提「Agent Skills」對我們的巨大幫助。

**什麼是 Agent Skills？**
簡單來說，這就是我們給 AI 訂立的「專案開發手冊」。
在使用 AI 輔助開發的過程中，我們曾經遇到一個問題：AI 每次產生的介面風格或變數命名習慣都不一樣。為了解決這個問題，我們開始撰寫並善用 Skills 文件。

舉例來說：
1. **`資料類型介面設計規範.md`**： 這份文件定義了遊戲中所有與詩詞關聯的 JSON 資料格式、命名傳統以及存取資料時必須使用的防呆機制。有了它，AI 在存取詩詞資料時就不會隨便創造不存在的屬性。
2. **`遊戲類型介面設計規範.md`**： 這份文件規範了所有新遊戲模組必須實作的生命週期函式（如 `init()`, `start()`, `pause()`, `gameOver()`），並且規定了按鈕的 CSS 命名空間，絕對避免樣式污染。

**讓 AI 自己寫 Skills**
最有趣的是，我們並不需要自己從頭把這些規範打出來。當我們與 AI 討論出一個很棒的架構後，我會對 AI 說：「請根據我們剛剛確定的設計，幫我撰寫（或更新）`遊戲類型介面設計規範.md`。」讓 AI 自行維護這些知識庫，這使得後續的溝通成本大幅降低。

### 6. 避坑指南與開發規範 (5分鐘)

開發過程中，卡關是家常便飯。我們經常陷入某些特定的錯誤狀態，例如：
- Canvas 渲染的效能瓶頸
- 事件監聽器未正確解除導致的 Memory Leak
- 非同步音效載入與播放的延遲問題

為了解決這些問題，我們建立了一份 **`花月開發常見錯誤與解法.md`**。這份文件就像是一本錯題本。只要踩過一次坑，AI 查出了解決方案，我們就會把這個解法存進這個文件中。未來當類似的錯誤再度發生時，AI 會優先參考這個「錯題本」，不僅讓錯誤迎刃而解，也統一了我們除錯的思維模式與後續程式的撰寫規範。

### 7. 美學與聽覺饗宴 (3分鐘)

**畫面構成與美術風格**
《花月》的美術風格力求乾淨、典雅。我們大量使用了網頁原生的 CSS 技術（如柔和的漸層、毛玻璃效果、書法字體的適配），而不是塞滿大量圖片。這不僅符合前端網頁設計的現代美學，也讓詩文成為畫面的真正主角。

**音效設計與實際運用**
音效是遊戲靈魂的一部分。我們的音效設計結合了東方樂器（如古箏、笛子）的音色。在先前的開發中，我們花了很多心力與 AI 討論如何利用 JavaScript 完美控制背景音樂的淡入淡出、以及過關音效與玩家點擊節奏的配合，這讓這款網頁遊戲具備了接近原生 App 的沈浸感。

### 8. 部署與結語 (2分鐘)

**輕鬆發布至 GitHub Pages**
以網頁作為平台的優勢在於，我們不需要經過繁瑣的 App Store 審核。利用 GitHub Pages，一鍵 Push 即可發布更新，讓所有玩家隨時玩到最新的內容。與 AI 協作的優勢，在這裡也展露無遺——AI 甚至能幫忙撰寫自動部署的腳本（Action scripts），大幅降低了開發營運的門檻。

**總結**
這幾個月下來，AI 對我來說不再只是一個會寫 Code 的機器器，它更像是一位不知疲倦、隨時可被指導卻又能反過來給予專業建議的資深工程師。只要我們敢於去發明規則、善用 Skills 將知識沉澱，即使是第一次寫遊戲的新手，也能端出一盤令人驚艷的好菜。

謝謝大家！希望我的分享能給大家帶來一些啟發，也期待看到大家使用 AI 誕生更多精彩的作品。如果大家有任何問題，歡迎提問交流！
