# 🎮 遊戲類型介面設計規範 (Game Interface Specification)

本規範定義了「花月」專案中所有遊戲類型頁面（如慢思快選、飛花令等）的介面設計、欄位分配與命名規則，旨在確保未來新增遊戲時具有高度的一致性與視覺品質。

## 0.1 ⚠️ 共用主題系統（strongly required for all new games）

自 V0.32 起，所有遊戲的「外殼」（overlay／頂部資訊列／控制鈕／難度標籤／生命值列／計時框／結算彈窗）**必須改用共用主題 CSS**，不得再於各自 `gameX.css` 中重複定義這些樣式。共用主題分兩套，依遊戲底色擇一套用：

| 主題檔 | Class 前綴 | 適用情境 | 目前範例遊戲 |
|--------|-----------|---------|-------------|
| `theme_xuanzhi.css` | `.fm-` | 淺色宣紙底（多數 DOM 類遊戲） | game1, game13 等 |
| `theme_dark.css` | `.fmd-` | 深色墨底（三消／連線類，需鮮豔字塊在深底跳出） | game24 等（game25+ 新增時比照套用） |

`index.html` 的 `<link>` 順序固定為：`screen_adaptive.css` → `theme_xuanzhi.css` → `theme_dark.css` → 其他共用 CSS → 各 `gameN.css`（依編號排序）。務必維持此順序，讓 `gameN.css` 能以 `var(--fm-xxx, 後備值)` 覆寫/延伸主題變數。

**兩套主題不可混用於同一遊戲**；`.fm-theme-dark` class 是 `theme_xuanzhi.css` 內建的「暗色皮膚」保留掛鉤（掛在 `.fm-overlay` 上即可切換同一組 `--fm-*` 變數為墨黑色），若宣紙系遊戲想要暗色版本，優先使用這個掛鉤，而非改套 `.fmd-`。

### 0.1.2 使用方式

* `createDOM()` 中，overlay 的 `className` 應**同時保留私有 class 與共用 class**，例如：
  ```javascript
  div.className = 'game13-overlay fm-overlay hidden';
  ```
  私有 `gameX-overlay` class 作為未來擴充或選擇器掛鉤保留，實際外觀樣式交給 `fm-overlay`。dark 系統（如 game24）目前僅掛 `fmd-overlay` 不重複保留私有 class，兩者皆屬可接受寫法，新遊戲建議統一保留私有 class 以利除錯。
* 頭部/控制鈕/難度標籤/生命值列/詩詞資訊/計時框等「外殼」DOM，一律直接套用上表對應的 `fm-*`／`fmd-*` class，**不再**另外定義 `gameX-header`、`gameX-score-board`、`gameX-controls`、`gameX-nav-btn` 等私有樣式。
* 遊戲核心玩法區（題目、答案網格、棋盤格子、粒子特效等）維持第 3.1 節規則，使用 `gameX-` 前綴私有 class，只有這部分才寫在各自 `gameX.css`。
* 顏色一律透過 `var(--fm-xxx, 後備值)` 取用共用變數（見 `theme_xuanzhi.css` 開頭定義），不要在 `gameX.css` 內寫死色碼，以確保未來共用主題調色時全站同步更新。
* 舊有遊戲（尚未套用本節規範者）不強制立即遷移，但新增或大改版遊戲時應優先套用；逐步套用進度可參考 §0 一覽表與各 `gameX.css` 內的遷移註解（如「已抽至 theme_xuanzhi.css 的 .fm-* 共用區塊」）。

## 1. 介面設計與風格
*   **視窗適配系統整合**：所有遊戲介面必須位於 `screen_adaptive.css` 定義的 `#stage` 容器內，並遵循 `screen_adaptive.js` 計算出的縮放比例。確保這些覆蓋層在任何螢幕尺寸下都能等比例縮放並置中顯示。確保在不同螢幕下維持一致的遊戲比例。
*   **視覺風格**：採用 **中國傳統美學、傳統配色、優雅、溫潤，文人雅士風格** 結合暗色調設計。
*   **背景設計**：使用徑向漸層 (`radial-gradient`)，色彩範圍從中心深藍 (`hsl(210, 30%, 25%)` 等) 到邊緣純黑。
*   **容器比例**：所有遊戲或資料類型介面必須位於 `#stage` 內。
                確保在 `gameX.js` 中建立容器時，其類別包含 `gameX-overlay`，例如：
                div.className = 'gameX-overlay hidden';
                並掛載於 `document.getElementById('stage')` 之下。
*   **邊框設計**：使用微弱的白色半透明邊框 (`hsla(0, 0%, 100%, 0.20)`) 來定義區域。
*   **物件尺寸與移動速度**：所有物件尺寸與移動速度，都必須以邏輯像素 (`px`) 為單位。由於舞台容器整體被縮放，內部的物件只需維持固定的邏輯尺寸，預設 1u=16px。
請參考 `game6.js` 的 `setupCanvas` 函數範例：
        ```javascript
        setupCanvas: function () {
            const area = document.querySelector('.game6-area');
            // 邏輯畫布大小固定為容器偏移大小（在 500x850 舞台內）
            this.canvas.width = area.offsetWidth;
            this.canvas.height = area.offsetHeight;

            // 使用固定的 px 作為基本單位，配合 stage 的 transform: scale() 達成跨裝置適配
            this.u = 16; 
            // 初始化玩家尺寸
            this.player.width = this.ui.playerWidth * this.u;
            this.player.height = this.ui.playerHeight * this.u;
            this.player.x = this.canvas.width / 2;
        }
        ```
*   **字體尺寸**：一律使用 `px` 單位，避免受系統字體縮放影響。
*   **顏色**：使用 hsl 色彩模式，基準為 hsl(0, 60%, 24%)。
*   **透明度**：使用 hsla 色彩模式，基準為 hsla(0, 0%, 100%, 0.80)。

## 2. 欄位分配 (Layout)
一個標準的遊戲介面應包含以下垂直排列的區域。**外殼部分（1、2）一律套用 §0.1 的 `fm-*`／`fmd-*` 共用 class，不再另建私有樣式**；下方括號內的 `gameX-` 名稱僅供 id 命名與私有掛鉤參考：
1.  **頂部資訊列 (`fm-header` / `fmd-header`)**：
    *   **左側**：分數看板 (`fm-scoreboard` / `fmd-score-board`)。
    *   **右側**：控制按鈕組 (`fm-controls` / `fmd-controls`)，依序包含「難度標籤」按鈕 (`fm-difficulty-tag` / `fmd-difficulty-tag`)、「重來」按鈕與「開新局」按鈕 (`fm-nav-btn` 或裸 `nav-btn`，見 §0.1.1)。
    *   **備註**：控制按鈕組的位置與順序已統一，難度選擇器置於左側第一個。
2.  **生命值列 (`fm-sub-header` / `fmd-sub-header`)**：
    *   居中顯示紅心圖標 (`fm-hearts` 容器 + `fm-heart` 單顆)，用於代表剩餘錯誤次數；靠右可放 `fm-poem-info` / `fmd-poem-info` 詩詞出處連結。
3.  **核心遊戲區 (`gameX-area`)**：
    *   **問題區域 (`gameX-question-area`)**：顯示詩詞題目、提示字或關鍵字選擇器。
    題目字數若超過7個字，需自動調整字體尺寸，避免超出界面寬度，請參考GAME1.js的 adjustFontSize 函數。
    *   **詩詞資訊 (`gameX-poem-info`)**：標註詩名(顯示長度限制15個中文字)、朝代、作者，具備連結樣式，點擊可開啟詩詞資料集。
4.  **答案區域 (`gameX-answer-area`)**：
    *   包含**計時方框/圓環 (`gameX-timer-ring`)**（使用 SVG 動態描繪）。
    *   **選項網格 (`gameX-answer-grid`)**：放置選擇按鈕。
5.  **遊戲結束訊息 (`gameX-message`)**：
    *   遊戲結束時顯示，包含標題、內容和綠色按鈕。

## 3. 命名規則
*   **ID 命名**：統一以 `gameX-` 為前綴（X 為遊戲編號），例如 `game1-container`, `game2-score`。
*   **CSS 類別**：遵循相同的 `gameX-` 前綴，通用組件如 `nav-btn`, `heart` 則維持跨遊戲統一。
*   **JS 物件**：將遊戲邏輯封裝在 `window.GameX` 物件中，並暴露 `show()`, `stopGame()` 等生命週期函數。
*   **檔案命名**：統一以 `gameX-` 為前綴（X 為遊戲編號），例如 `game1.css`, `game1.js`。
*   **各遊戲難度設定的變數名稱**：如果有使用到符合功能的變數請依下方統一命名
    *   `timeLimit`: 時間限制（秒）—— 已棄用於大多數遊戲，改用 `timeLimitRate`
    *   `timeLimitRate`: 每字時間倍率（秒/字），實際時限 = 執行期間實際題目字數 × timeLimitRate（⚠️ 必須在取詩/生題函式之後才能計算，不可用固定設定值如 `questionCount` 代替實際字數）
    *   `poemMinRating`: 最低詩詞評分
    *   `maxMistakeCount`: 最多錯誤次數
    *   `answerAtLine`: 答案出現在第幾行，0=第一行或第二行，1=第一行，2=第二行，3=第一行和第二行
    *   `minLines`: 最少詩句數
    *   `maxLines`: 最多詩句數
    *   `minChars`: 最少總字數
    *   `maxChars`: 最多總字數
    *   `minMaskCount`: 最少遮罩字數
    *   `maxMaskCount`: 最多遮罩字數
    *   `sentenceMinRating`: 最低詩句評分（game3 專用）
    *   `incrementSpeed`: 遊戲速度遞增量
    *   `maxSpeed`: 最高速度上限
    *   `timeMutiply`: 時間倍率（注意：此處為程式既有拼法，非 multiply）

### 3.1 ⚠️ CSS class 命名必須加模組前綴（強制要求）
**所有遊戲模組的專屬 CSS class，必須加上 `gameX-` 前綴。絕不使用通用名稱（如 `.poem-line`、`.title`、`.body`）作為某一遊戲的私有 class。**

例外允許跨遊戲共用的通用 class：`.hearts`、`.heart`、`.nav-btn`、`.hidden`、`.flying-star`（均由 `scoreManager.js` 或 `screen_adaptive.css` 定義）。

### 3.2 URL 參數嚴格比對機制
所有 `gameX.js` 結尾自動啟動的判斷，必須使用 `URLSearchParams` 精確比對，防止字串部分吻合（例如 `game=1` 與 `game=10`）造成的錯誤加載：
```javascript
if (new URLSearchParams(window.location.search).get('game') === 'X') {
    setTimeout(() => {
        if (window.GameX) window.GameX.show();
        window.history.replaceState({}, document.title, window.location.pathname);
    }, 50);
}
```

## 4. 動畫與交互設計
*   **計時器**：使用 SVG 矩形路徑 (`#gameX-timer-path`)，透過 `stroke-dashoffset` 實現邊框倒數動畫。
*   **錯誤反饋**：點選錯誤選項時，必須觸發 `shake` (抖動) 動畫。
*   **過關動畫**：獲勝時呼叫 `ScoreManager.playWinAnimation` 顯示得分飄浮與計分跳動效果。
*   **按鈕效果**：Hover 時應有背景色亮化或邊框變色。
    box-shadow: 0 0px 8px rgba(255, 255, 255, 0.9);
    filter: brightness(1.5);
*   **挑戰成功防呆機制**：一旦觸發通關 (Win)，必須**立刻**禁用 (`disabled = true`) 「重來」與「開新局」按鍵，防止玩家在動畫期間連點刷分。

## 5. 字體規劃與色彩習慣
*   **主字體**：`'Noto Serif TC', serif` (思源宋體)。
*   **動態縮放**：腳本中必須包含 `adjustFontSize` 邏輯，根據文字長度自動調整 `px` 大小。
*   **詩詞名稱長度限制**：詩詞名稱 poem-info只能顯示前12個字，若超過12個字，則以「...」替代。
        //詩詞名稱只能顯示前12個字
        let title = this.currentPoem.title;
        if (title.length > 12) {
            title = title.substring(0, 10) + "...";
        }
        const infoText = `${title} / ${this.currentPoem.dynasty} / ${this.currentPoem.author}`;
        const infoEl = document.getElementById('gameX-poem-info');
        infoEl.textContent = infoText;
        infoEl.dataset.poemId = this.currentPoem.id;
        //this.adjustFontSize(infoEl, infoText.length, 20, 1.0); //取消，若詩詞名稱過長，改以刪減詩詞名稱字數替代

*   **強調/關鍵字**：`hsl(36, 80%, 50%)` (金橙色)；
*   **紅心/損血**：`hsl(4, 90%, 60%)` (鮮紅色)；
*   **正確反饋**：`hsl(145, 68%, 25%)` (深翠綠色)；
*   **錯誤反饋**：`hsl(0, 70%, 25%)` (朱紅色)。

## 6. 共用題庫選取 (getSharedRandomPoem)

```javascript
// 完整簽名（在 script.js 中定義）
getSharedRandomPoem(
    minRating,        // 最低詩詞評分
    minLines,         // 最少行數（演算法強制從奇數句起始，保持兩句一聯完整性）
    maxLines,         // 最多行數
    minChars,         // 最少總字數
    maxChars,         // 最多總字數
    keyword = "",     // 篩選關鍵字（選填，空字串表示不限制）
    seed = null,      // 隨機種子/關卡序號（傳入則確定性選取，null 則純隨機）
    gameKey = ""      // 遊戲ID，融入種子確保不同遊戲同關卡產生不同題目
)
```

**回傳值：**
```javascript
{
    poem: 原始詩詞物件,           // 包含 id, title, dynasty, author, content 等欄位
    lines: ["乾淨句子1", ...],    // 已去除標點符號的字串陣列
    startIndex: 起始索引           // 於原始 poem.content 中的起始行號
}
// 無符合結果時回傳 null（內建降級保護，會自動放寬 minRating 重試）
```

**重要原則**：
- 全程禁止以 `rating` 排序選取，確保隨機性與題庫多樣性。
- 演算法限制必須從奇數句（index 0, 2, 4...）開始，保持「兩句一聯」語意完整。
- 有 `seed` 時為確定性選取，相同參數永遠回傳同一首詩（適合挑戰模式）。

## 7. 共通性元件規格與範例 (Common Component Specifications)

### 7.1 HTML 結構模板（宣紙 `.fm-` 版，DOM 類遊戲預設採用）
外殼（1、2）套用 `fm-*` 共用 class；核心玩法區（3、4）維持 `gameX-` 私有 class；結算彈窗直接使用共用的 `.common-game-message`（定義於 `theme_xuanzhi.css`，不再各自寫 `gameX-message`）。
```html
<div id="gameX-container" class="gameX-overlay fm-overlay hidden">
    <!-- 1. 資訊列 -->
    <div class="fm-header">
        <div class="fm-scoreboard">分數: <span id="gameX-score">0</span></div>
        <div class="fm-controls">
            <!-- 難度標籤統一放在最左側 -->
            <button class="fm-difficulty-tag" id="gameX-diff-tag">小學</button>
            <button id="gameX-retryGame-btn" class="fm-nav-btn">重來</button>
```

### 7.3 JS 邏輯模板

**更新倒數方框 (Timer Ring Update)**
```javascript
updateTimerRing: function (ratio) {
    const rect = document.getElementById('gameX-timer-path');
    const container = document.getElementById('gameX-answer-grid-container');
    if (!rect || !container) return;

    const w = container.offsetWidth;
    const h = container.offsetHeight;
    
    const svg = document.getElementById('gameX-timer-ring');
    svg.setAttribute('width', w);
    svg.setAttribute('height', h);

    const rw = Math.max(0, w - 10);
    const rh = Math.max(0, h - 10);

    rect.setAttribute('width', rw);
    rect.setAttribute('height', rh);

    const perimeter = (rw + rh) * 2;
    rect.style.strokeDasharray = perimeter;
    rect.style.strokeDashoffset = perimeter * (1 - ratio); // ratio 從 1 至 0
}
```

**勝利或失敗判定**
取消 gamewin()與gamelose()，改用 `gameOver(win, reason)`

**防刷分與結算防呆**
```javascript
gameOver: function (win, reason) {
    this.isActive = false;
    this.isWin = win;
    
    if (win) {
        document.getElementById('gameX-retryGame-btn').disabled = true;
        document.getElementById('gameX-newGame-btn').disabled = true;
    } else {
        document.getElementById('gameX-retryGame-btn').disabled = false;
        document.getElementById('gameX-newGame-btn').disabled = false;
    }
    
    const msgDiv = document.getElementById('gameX-message');
    msgDiv.classList.remove('hidden');
}
```

## 8. 音樂音效共用模組（SoundManager）

所有音效皆由 `script.js` 的 `SoundManager` 模組提供，無需各遊戲自行建立 AudioContext。

```javascript
// --- 古箏與鋼琴單音 ---
soundManager.playGuzheng(index, gainMultiply = 1)     // 撥放古箏音階（index 為音階索引）
soundManager.playMelodyNote(noteNumber, gainMultiply = 1) // 撥放自然大調單音

// --- 基礎互動音效 ---
soundManager.playHit(index, timeLength)     // 撞擊聲
soundManager.playSuccess()                  // 正確回饋（A4 → A5，較長）
soundManager.playSuccessShort()             // 正確短音（適合連續點擊遊戲）
soundManager.playFailure()                  // 錯誤回饋（A3 → A2）
soundManager.playCountdown()               // 倒數321音效（G3 → A3 → A4）

// --- 物品操作音效 ---
soundManager.playOpenItem()                // 開啟項目（E4）
soundManager.playCloseItem()               // 關閉項目（C4）
soundManager.playConfirmItem()             // 確認項目（A4）

// --- 敵人/破壞音效 ---
soundManager.playBreakEnemy()             // 敵人被破壞（兩聲向上跳動）

// --- 情緒音效 ---
soundManager.playJoyfulTriple()           // 喜悅三連音（C5 E5 G5）
soundManager.playJoyfulTripleSlow()       // 喜悅三連音（慢速）
soundManager.playSadTriple()              // 悲傷三連音（A4 → E3 → C3）

// --- 樂曲播放系統 ---
melodyPlayer.setMelody(melodyName)        // 設定當前樂曲（不立即播放）
melodyPlayer.playNextNote()              // 播放樂曲的下一個音符（手動步進）
melodyPlayer.playFullMelody(melodyName)  // 連續自動播放整首樂曲
```

**內建樂曲庫**（傳入 `melodyName` 字串）：
`小星星`、`生日快樂歌`、`青青校樹`、`送別`、`茉莉花`、`踏雪尋梅`、`春神來了`、`望春風`
