# 📜 資料類型介面設計規範 (Data Interface Specification)

本規範定義了「花月」專案中所有資料展示類型頁面（如成就紀錄、名人列傳、詩詞資料集）的介面設計規範，確保資料呈現清晰、層次分明且操作流暢。

## 1. 介面設計與風格
*   **視覺風格**：採半透明 Overlay 疊加模式，強調資訊的閱讀性。
*   **佈局結構**：多採用「兩欄式」或「分頁標籤式」佈局。
*   **捲動體驗**：內容區塊必須支援 `momentum scrolling` (慣性捲動)，並在 JS 中實作滑鼠拖拽與觸控移動的相容邏輯。
*   **玻璃材質**：背景通常使用 `backdrop-filter: blur()` 或深色半透明背景，強化層次感。

## 2. 欄位分配 (Layout)
一個標準的資料介面通常包含以下組件：
1.  **標題與關閉列**：包含頁面標題及右上角的關閉按鈕 (`&times;` 或 `.page-close-btn`)。
2.  **分頁標籤 (`tabs`)**：如成就系統中的「總覽」、「紀錄」、「成就」。
3.  **過濾與搜尋區 (Header)**：
    *   下拉選單 (`select`)：用於朝代、姓氏或難度篩選。
    *   搜尋框 (`input type="text"`)：提供關鍵字快速檢索。
4.  **內容主區塊 (Body)**：
    *   **左側/清單**：放置條目清單 (Authors, Badges, etc.)，具備「選中」狀態 (`.selected`)。
    *   **右側/詳情**：顯示選定項目的詳細內容、作品集或獎狀預覽。

## 3. 命名規則
*   **ID 命名**：採用語義化命名，如 `achievementOverlay`, `authorBioPage`, `poemDialog`。
*   **組件類別**：通用的資料顯示類別包括：
    *   `.ach-panel`, `.bio-content` (面板主體)
    *   `.filter-group` (過濾器容器)
    *   `.author-item`, `.work-item` (條目)
*   **函數命名**：統一使用 `processData`, `applyFilters`, `updateList`, `show`, `hide` 等。

## 4. 動畫與交互設計
*   **進入與退出**：配合 `hidden` 類別切換及 `active` 類別觸發動態效果。
*   **獲獎特效**：獎狀 (Certificate) 出現時應有由小變大的縮放動畫與文字漸顯效果。
*   **點擊反饋**：列表項目在點擊時應有背景色變換或縮放效果，引導使用者當前選擇位置。
*   **拖曳手感**：支援鼠標拖拉慣性，增強移動端與 PC 端的介面流暢度。

## 5. 字體規劃
*   **主體標題**：使用粗體 `1.2rem` 以上，展現權威感。
*   **條目名稱**：`1.1rem`，保持可讀性。
*   **次要資訊/數值**：`0.8rem` - `0.9rem`，適度調降透明度 (`opacity: 0.8`) 以區分層級。
*   **詩詞本文**：維持經典的直排或橫排宋體美感。

## 6. 色彩習慣用法
*   **分頁欄色彩**：活躍選項通常使用亮色底線或色塊區分。
*   **強調數值**：使用金黃色或亮白色突顯累積總分、排行等。
*   **獎狀風格**：依據品階設定不同背景（九品至一品、聖旨級別）。
*   **未選中項目**：背景透白或半透明灰，文字色度較低。

---
> [!IMPORTANT]
> **開發提醒**：在製作新的資料面板時，請務必先瞭解 `responsive.js` 與 `responsive.css` 針對螢幕高度的計算邏輯。資料類介面往往含有長列表，必須確保在各種解析度下都能正確觸發垂直滾動條，且關閉按鈕不會超出可視區域。
