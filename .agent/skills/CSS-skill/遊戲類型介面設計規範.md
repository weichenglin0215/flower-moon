# 🎮 遊戲類型介面設計規範 (Game Interface Specification)

本規範定義了「花月」專案中所有遊戲類型頁面（如慢思快選、飛花令等）的介面設計、欄位分配與命名規則，旨在確保未來新增遊戲時具有高度的一致性與視覺品質。

## 1. 介面設計與風格
*   **響應式佈局系統整合**：所有資料介面必須使用 `responsive.css` 中的 `aspect-5-8` 類別，responsive.js 計算出的動態尺寸 (--responsive-width, --responsive-height)。確保這些覆蓋層在任何螢幕尺寸下都能置中並正確縮放。確保在不同螢幕下維持一致的遊戲比例。
*   **視覺風格**：採用 **中國傳統美學、傳統配色、優雅、溫潤，文人雅士風格** 結合暗色調設計。
*   **背景設計**：使用徑向漸層 (`radial-gradient`)，色彩範圍從中心深藍 (`hsl(210, 30%, 25%)`) 到邊緣純黑。
*   **容器比例**：所有遊戲 Overlay 必須帶有 `aspect-5-8` 類別，確保在不同螢幕下維持一致的遊戲比例。
*   **邊框設計**：使用微弱的白色半透明邊框 (`hsla(0, 0%, 100%, 0.20)`) 來定義區域。
*   **字體尺寸**：使用rem單位，基準為0.3rem=10px。
*   **顏色**：使用 hsl 色彩模式，基準為 hsl(0, 60%, 24%)。
*   **透明度**：使用 hsla 色彩模式，基準為 hsla(0, 0%, 100%, 0.80)。

## 2. 欄位分配 (Layout)
一個標準的遊戲介面應包含以下垂直排列的區域：
1.  **頂部資訊列 (`gameX-header`)**：
    *   **左側**：分數看板 (`gameX-score-board`)。
    *   **右側**：控制按鈕組 (`gameX-controls`)，至少包含「重來」按鈕retryGame()與「開新局」按鈕 startNewGame()。
    *   **用色與形狀**：
        *   按鍵顏色使用 hsl(0, 60%, 24%)。
        *   padding、margin、border-radius、font-size請根據game4進行調整。
2.  **生命值列 (`gameX-sub-header`)**：
    *   居中顯示紅心圖標 (`hearts`)，用於代表剩餘錯誤次數。
3.  **核心遊戲區 (`gameX-area`)**：
    *   **難度標籤 (`gameX-difficulty-tag`)**：顯示當前遊戲難度。
    *   **問題區域 (`gameX-question-area`)**：顯示詩詞題目、提示字或關鍵字選擇器。
    *   **詩詞資訊 (`gameX-poem-info`)**：標註詩名、朝代、作者，且通常具備連結樣式，可以開啟"詩詞資料集"顯示該詩的細節。
4.  **答案區域 (`gameX-answer-area`)**：
    *   包含**計時圓環 (`gameX-timer-ring`)**。
    *   **選項網格 (`gameX-answer-grid`)**：放置選擇按鈕。

## 3. 命名規則
*   **ID 命名**：統一以 `gameX-` 為前綴（X 為遊戲編號），例如 `game1-container`, `game2-score`。
*   **CSS 類別**：遵循相同的 `gameX-` 前綴，通用組件如 `nav-btn`, `heart` 則維持跨遊戲統一。
*   **JS 物件**：將遊戲邏輯封裝在 `window.GameX` 物件中，並暴露 `show()`, `stopGame()` 等生命週期函數。
*   **檔案命名**：統一以 `gameX-` 為前綴（X 為遊戲編號），例如 `game1.css`, `game1.js`。

## 4. 動畫與交互設計
*   **計時器**：使用 SVG 矩形路徑 (`#gameX-timer-path`)，透過 `stroke-dashoffset` 實現邊框倒數動畫。
*   **錯誤反饋**：點選錯誤選項時，必須觸發 `shake` (抖動) 動畫。
*   **過關動畫**：獲勝時呼叫 `ScoreManager.playWinAnimation` 顯示得分飄浮與計分跳動效果。
*   **按鈕效果**：Hover 時應有背景色亮化或邊框變色。
*   **挑戰成功的分數計算動畫**：先表演Heart動畫，再表演剩餘時間轉換成黃色星星的動畫，最後再表演得分乘倍數效果。
*   以上效果請參考game6與scoreManager的程式寫法。

## 5. 字體規劃
*   **主字體**：`'Noto Serif TC', serif` (思源宋體)，展現古典文學質感。
*   **動態縮放**：遊戲腳本中必須包含 `adjustFontSize` 邏輯，根據文字長度自動調整 `rem` 大小，防止版型崩潰。
    *   問題文字預設：`2.5rem` (加粗)。
    *   選項按鈕文字預設：`2.0rem`。
    *   資訊文字預設：`1.0rem`。
*   **字體尺寸**：使用rem單位，基準為0.3rem=10px。

## 6. 色彩習慣用法
*   **基礎文字色**：`hsla(0, 0%, 20%, 1.00)` (質樸的深灰色)。
*   **深底背景的文字色**：`hsl(40, 50%, 80%)` (溫潤的古籍奶油色)。
*   **強調/關鍵字**：`hsl(36, 80%, 50%)` (金橙色)。
*   **紅心/損血**：`hsl(4, 90%, 60%)` (鮮紅色)。
*   **正確反饋**：`hsl(145, 68%, 25%)` (深翠綠色)。
*   **錯誤反饋**：`hsl(0, 70%, 25%)` (朱紅色)。
*   **計時路徑**：`hsl(0, 80%, 36%)` (胭脂紅)。
*   **顏色**：使用 hsl 色彩模式，基準為 hsl(0, 60%, 24%)。
*   **透明度**：使用 hsla 色彩模式，基準為 hsla(0, 0%, 100%, 0.80)。

## 7. 詩詞題目挑選規則
本文件定義程式於詩詞資料庫中選取題目之邏輯。為確保出題多樣性，禁止在選取階段使用評分排序，須統一採隨機選取。
已經有共用程式碼，請直接使用function getSharedRandomPoem(minRating, minLines, maxLines, minChars, maxChars)來選取詩詞。
請參考 game8 的 selectRandomPoem: function () 的寫法。
* **A. 詩詞篩選階段**
    * **條件：** 篩選出所有 `rating` 大於等於 `difficultySettings.stars` 之詩詞。
    * **執行：** 從符合條件的清單中**隨機**選取一首詩詞。
    * **注意：** 嚴禁以 `rating` 排序後取最高分，以避免題目集中於特定高分詩詞。

* **B. 起始詩句篩選階段**
    * **條件：** 於選定之詩詞中，篩選出符合以下條件的索引（index）：
        * 索引為**奇數**（第 1, 3, 5... 句）。
        * 以奇數句開始挑選是符合中國詩詞的使用習慣。
        * 切勿從偶數句(第 2, 4, 6... 句)開始挑選，嚴重影響玩家對詩詞的認知。
        * 該句之 `line_ratings` 大於等於 `difficultySettings.stars`。
        * 後續剩餘行數必須大於等於 `difficultySettings.minLines`（確保題目詩句數量足夠）。
    * **執行：** 從符合條件的句子索引清單中**隨機**選取一筆作為起始句。
    * **注意：** 嚴禁以 `line_ratings` 排序後取最高分。以避免題目集中於特定高分詩句。

* **C. 題目組建階段**
    * **執行：** 以步驟 2 選定之索引為起點，往後連續擷取大於等於 `difficultySettings.minLines` 數量的句子作為最終題目。
    * **字數上限條件：** 擷取到的句子的總字數必須小於等於 `difficultySettings.maxChars`。
    * **條件：** 選取的內容必須為該詩詞中原始排序的連續片段，不得跳句。

## 8. 其他整體性要求
*   **遊戲挑戰成功時，重來按鈕應禁用**：避免玩家在挑戰成功畫面點擊重來。
