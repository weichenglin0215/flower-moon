# 📜 資料類型介面設計規範 (Data Interface Specification)

本規範定義了「花月」專案中所有資料展示類型頁面（如成就紀錄、名人列傳、詩詞資料集）的介面設計規範，確保資料呈現清晰、層次分明且操作流暢。

## 1. 介面設計與風格
A.  **響應式佈局系統整合**：所有資料介面必須使用 `responsive.css` 中的 `aspect-5-8` 類別，responsive.js 計算出的動態尺寸 (--responsive-width, --responsive-height)。確保這些覆蓋層在任何螢幕尺寸下都能置中並正確縮放。確保在不同螢幕下維持一致的遊戲比例。
B.  **視覺風格**：採 Overlay 疊加模式，強調資訊的閱讀性。
C.  **佈局結構**：多採用「兩欄式」或「分頁標籤式」佈局。
D.  **捲動體驗**：內容區塊(尤其是多個項目的列表)必須能夠使用手指上下滑動來捲動觀看內容，且支援 `momentum scrolling` (慣性捲動)，並在 JS 中實作滑鼠拖拽與觸控移動的相容邏輯。可以參考author_bio.js的 `function setupMomentumScroll(scrollContainer)`
E.  **水墨宣紙渲染風格**：背景通常使用 `backdrop-filter: blur()` 或深色半透明背景，強化層次感。
F.  **字體尺寸**：使用rem單位，基準為0.3rem=10px。
G.  **顏色**：使用 hsl 色彩模式，基準為 hsl(0, 60%, 24%)。
H.  **透明度**：使用 hsla 色彩模式，基準為 hsla(0, 0%, 100%, 0.80)。


## 2. 欄位分配 (Layout)
一個標準的資料介面通常包含以下組件：
A.  **標題與關閉列**：包含頁面標題及右上角的關閉按鈕 (`&times;` 或 `.page-close-btn`)。
B.  **分頁標籤 (`tabs`)**：如成就系統中的「總覽」、「紀錄」、「成就」。
C.  **過濾與搜尋區 (Header)**：
    *   下拉選單 (`select`)：用於朝代、姓氏或難度篩選。
    *   搜尋框 (`input type="text"`)：提供關鍵字快速檢索。
D.  **內容主區塊 (Body)**：
    *   **左側/清單**：放置條目清單 (Authors, Badges, etc.)，具備「選中」狀態 (`.selected`)。
    *   **右側/詳情**：顯示選定項目的詳細內容、作品集或獎狀預覽。
E.  **尺寸**：使用rem單位，基準為0.3rem=10px。

## 3. 命名規則
A.  **ID 命名**：採用語義化命名，如 `achievementOverlay`, `authorBioPage`, `poemDialog`。
B.  **組件類別**：通用的資料顯示類別包括：
    *   `.ach-panel`, `.bio-content` (面板主體)
    *   `.filter-group` (過濾器容器)
    *   `.author-item`, `.work-item` (條目)
C.  **函數命名**：統一使用 `processData`, `applyFilters`, `updateList`, `show`, `hide` 等。

## 3.1 ⚠️ CSS class 命名必須加模組前綴（強制要求）
所有資料類型模組（`achievement.js`、`author_bio.js`、`poem_dialog.js`、`difficulty-selector.js`、`introCard.js`）的 CSS class 名稱，**必須**加上該模組的騶一前綴，絕不使用通用素數名稱：

- **正確**：`.ach-panel`、`.bio-header`、`.pd-line-content`、`.cards-poem-line`
- **錯誤（會導致污染）**：`.poem-line`、`.title`、`.header`、`.body`

> 案例：2026-03 發現 `cards.css` 的 `.poem-line:first-child { transform: translateY(-1.8rem); }` 和 `calendar.css` 的 `.poem-line` 充突，導致日曆介面第一行詩句大幅偏移。修復方式：將 `cards.css` 中的 `.poem-line` 改名為 `.cards-poem-line`。

## 3.2 每個 JS 模組必須加入 `loadCSS` 防護
資料類型模組通常被其他頁面動態導入，無法依賴 HTML `<link>` 標籤的霧入機制來載入 CSS。必須在 `init()` 裡自行載入：

```javascript
// 在 init() 開頭加入（以 achievement.js 為例）
if (!document.getElementById('achievement-css')) {
    const link = document.createElement('link');
    link.id = 'achievement-css';
    link.rel = 'stylesheet';
    link.href = 'achievement.css';
    document.head.appendChild(link);
}
```

此機制確保：
1. 即使 HTML 未明確引入 CSS，模組也能自行裝載所需樣式
2. 利用唯一 `id` 標記防止重複載入
3. 明確封裝「哪個 JS 責任載入哪個 CSS」的从屬關係

## 4. 動畫與交互設計
A.  **進入與退出**：配合 `hidden` 類別切換及 `active` 類別觸發動態效果。
B.  **獲獎特效**：獎狀 (Certificate) 出現時應有由小變大的縮放動畫與文字漸顯效果。
C.  **點擊反饋**：列表項目在點擊時應有背景色變換或縮放效果，引導使用者當前選擇位置。
D.  **拖曳手感**：支援鼠標拖拉慣性，增強移動端與 PC 端的介面流暢度。

## 5. 字體規劃
A.  **主體標題**：使用粗體 `1.2rem` 以上，展現權威感。
B.  **條目名稱**：`1.1rem`，保持可讀性。
C.  **次要資訊/數值**：`0.8rem` ~ `0.9rem`，適度調降透明度 (`opacity: 0.8`) 以區分層級。
D.  **詩詞本文**：維持經典的直排或橫排宋體美感。
E.  **字體尺寸**：使用rem單位，基準為0.3rem=10px。

## 6. 色彩習慣用法
A.  **分頁欄色彩**：活躍選項通常使用亮色底線或色塊區分。
B.  **強調數值**：使用金黃色或亮白色突顯累積總分、排行等。
C.  **獎狀風格**：依據品階設定不同背景（九品至一品、聖旨級別）。
D.  **未選中項目**：背景透白或半透明灰，文字色度較低。
E.  **顏色**：使用 hsl 色彩模式，基準為 hsl(0, 60%, 24%)。
F.  **透明度**：使用 hsla 色彩模式，基準為 hsla(0, 0%, 100%, 0.80)。
---
> [!IMPORTANT]
> **開發提醒**：在製作新的資料面板時，請務必先瞭解 `responsive.js` 與 `responsive.css` 針對螢幕高度的計算邏輯，能同時符合PC與手機上瀏覽器的規範，避免在手機瀏覽器上介面錯位或尺寸不符。資料類介面往往含有長列表，必須確保在各種解析度下都能正確觸發垂直滾動條，且關閉按鈕不會超出可視區域。
